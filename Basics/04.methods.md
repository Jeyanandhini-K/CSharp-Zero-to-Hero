# Methods and Functions

A method is a reusable block of code that performs a specific task. Methods are the building blocks of every C# program.

WHAT IS A METHOD?

A method is a function - a container for code that solves one specific problem.

Basic Method:

int Add(int a, int b) {
    return a + b;
}

int result = Add(5, 3);
Console.WriteLine(result);  // 8

METHOD STRUCTURE

[AccessModifier] [ReturnType] MethodName([Parameters]) {
    // Method body
    return [value];  // If return type is not void
}

Parts:
- AccessModifier: public, private, protected (we'll learn in OOP)
- ReturnType: What method returns (int, string, void, bool, etc.)
- MethodName: Name (use PascalCase: CalculateTotal)
- Parameters: Input values in parentheses
- Return Statement: return gives back a value

SIMPLE METHODS - NO PARAMETERS, NO RETURN

These methods just do something without giving back data.

Print a Message:

void Greet() {
    Console.WriteLine("Hello, World!");
}

Greet();  // Output: Hello, World!

Print Multiple Lines:

void PrintWelcome() {
    Console.WriteLine("Welcome!");
    Console.WriteLine("Let's learn C#");
}

PrintWelcome();
// Output:
// Welcome!
// Let's learn C#

METHODS WITH PARAMETERS

Parameters are inputs to the method.

Single Parameter:

void Greet(string name) {
    Console.WriteLine($"Hello, {name}!");
}

Greet("Ali");      // Output: Hello, Ali!
Greet("Fatima");   // Output: Hello, Fatima!

Multiple Parameters:

void PrintStudent(string name, int age, double gpa) {
    Console.WriteLine($"Name: {name}");
    Console.WriteLine($"Age: {age}");
    Console.WriteLine($"GPA: {gpa}");
}

PrintStudent("Ahmed", 20, 3.8);
// Output:
// Name: Ahmed
// Age: 20
// GPA: 3.8

Parameter Validation:

void PrintAge(int age) {
    if (age < 0 || age > 120) {
        Console.WriteLine("Invalid age");
        return;  // Exit method early
    }
    Console.WriteLine($"Age: {age}");
}

PrintAge(25);   // Age: 25
PrintAge(150);  // Invalid age

METHODS WITH RETURN VALUES

These methods give back a value.

Return Single Value:

int Add(int a, int b) {
    return a + b;
}

int sum = Add(10, 5);
Console.WriteLine(sum);  // 15

Return String:

string GetFullName(string firstName, string lastName) {
    return firstName + " " + lastName;
}

string name = GetFullName("Ali", "Khan");
Console.WriteLine(name);  // Ali Khan

Return Boolean:

bool IsAdult(int age) {
    return age >= 18;
}

if (IsAdult(25)) {
    Console.WriteLine("You can vote");
}
// Output: You can vote

Return with Multiple Cases:

string GetGrade(int score) {
    if (score >= 90) return "A";
    else if (score >= 80) return "B";
    else if (score >= 70) return "C";
    else return "F";
}

Console.WriteLine(GetGrade(85));  // B

METHOD OVERLOADING

Multiple methods with same name but different parameters.

Add Two Integers:

int Add(int a, int b) {
    return a + b;
}

Add Three Integers:

int Add(int a, int b, int c) {
    return a + b + c;
}

Add Two Doubles:

double Add(double a, double b) {
    return a + b;
}

Using Overloaded Methods:

Console.WriteLine(Add(5, 3));           // Calls first: 8
Console.WriteLine(Add(5, 3, 2));        // Calls second: 10
Console.WriteLine(Add(5.5, 3.2));       // Calls third: 8.7

Compiler figures out which version to use based on parameters.

Overloading with Different Parameter Types:

void Display(int num) {
    Console.WriteLine($"Integer: {num}");
}

void Display(string text) {
    Console.WriteLine($"String: {text}");
}

void Display(double decimal) {
    Console.WriteLine($"Double: {decimal}");
}

Display(42);        // Integer: 42
Display("Hello");   // String: Hello
Display(3.14);      // Double: 3.14

DEFAULT PARAMETERS

Set a default value if caller doesn't provide one.

Simple Example:

void Greet(string name = "Guest") {
    Console.WriteLine($"Hello, {name}!");
}

Greet();           // Output: Hello, Guest!
Greet("Ahmed");    // Output: Hello, Ahmed!

Create Account:

void CreateAccount(string username, string role = "User") {
    Console.WriteLine($"Creating account: {username}");
    Console.WriteLine($"Role: {role}");
}

CreateAccount("ali_khan");                    // Role: User (default)
CreateAccount("admin_user", "Administrator"); // Role: Administrator

Multiple Default Parameters:

void PrintMessage(string message, int count = 1, bool caps = false) {
    string text = caps ? message.ToUpper() : message;
    for (int i = 0; i < count; i++) {
        Console.WriteLine(text);
    }
}

PrintMessage("Hello");              // Prints once
PrintMessage("Hello", 3);           // Prints 3 times
PrintMessage("Hello", 2, true);     // Prints 2 times in CAPS

PARAMS KEYWORD - VARIABLE ARGUMENTS

Accept any number of arguments.

Sum Example:

int Sum(params int[] numbers) {
    int total = 0;
    foreach (int num in numbers) {
        total += num;
    }
    return total;
}

Console.WriteLine(Sum(5));          // 5
Console.WriteLine(Sum(5, 10));      // 15
Console.WriteLine(Sum(5, 10, 3, 8));    // 26

Print Names:

void PrintNames(params string[] names) {
    foreach (string name in names) {
        Console.WriteLine(name);
    }
}

PrintNames("Ali");
PrintNames("Ali", "Ahmed");
PrintNames("Ali", "Ahmed", "Hassan", "Fatima");

PRACTICAL EXAMPLES

Calculate Grade:

string CalculateGrade(double percentage) {
    if (percentage >= 90) return "A";
    if (percentage >= 80) return "B";
    if (percentage >= 70) return "C";
    if (percentage >= 60) return "D";
    return "F";
}

double score = 85;
string grade = CalculateGrade(score);
Console.WriteLine($"Score: {score}%, Grade: {grade}");
// Output: Score: 85%, Grade: B

Validate Email:

bool IsValidEmail(string email) {
    return email.Contains("@") && email.Contains(".");
}

string email = "user@example.com";
if (IsValidEmail(email)) {
    Console.WriteLine("Email is valid");
}
// Output: Email is valid

Apply Discount:

double ApplyDiscount(double price, double discountPercent = 10) {
    return price - (price * discountPercent / 100);
}

Console.WriteLine(ApplyDiscount(100));       // 90 (10% off)
Console.WriteLine(ApplyDiscount(100, 20));   // 80 (20% off)

Check Password Strength:

bool IsStrongPassword(string password) {
    bool hasLength = password.Length >= 8;
    bool hasUppercase = password.Any(char.IsUpper);
    bool hasLowercase = password.Any(char.IsLower);
    bool hasDigit = password.Any(char.IsDigit);
    
    return hasLength && hasUppercase && hasLowercase && hasDigit;
}

Console.WriteLine(IsStrongPassword("MyPass123"));  // true
Console.WriteLine(IsStrongPassword("weak"));       // false

Calculate BMI:

double CalculateBMI(double weight, double height) {
    return weight / (height * height);
}

string GetBMICategory(double bmi) {
    if (bmi < 18.5) return "Underweight";
    if (bmi < 25) return "Normal weight";
    if (bmi < 30) return "Overweight";
    return "Obese";
}

double bmi = CalculateBMI(70, 1.75);
Console.WriteLine($"BMI: {bmi:F2}");
Console.WriteLine($"Category: {GetBMICategory(bmi)}");

NAMING CONVENTIONS

DO:
- Use PascalCase: CalculateTotal(), GetUserName(), IsValid()
- Use descriptive names: ValidateEmail() (good) vs Check() (bad)
- Use verb-noun: GetData(), CalculateSum(), PrintMessage()
- Start with action verb: IsActive(), HasPermission(), CreateUser()

DONT:
- Use abbreviations: Calc() instead use Calculate()
- Make names too long: CalculateTotalPriceWithTaxAndDiscount() is too much
- Use single letters (except i, j for loops)
- Use method1, method2 naming

METHOD SIGNATURE

The method signature includes return type, method name, and parameters.

public string GetFullName(string firstName, string lastName)
// public = access modifier
// string = return type
// GetFullName = method name
// (string firstName, string lastName) = parameters

Understanding signatures is crucial for overloading, implementing interfaces, and calling methods correctly.

RETURN EARLY PATTERN

Exit method early instead of nested if-else.

Bad - Deep Nesting:

string GetStatus(int age) {
    if (age >= 0) {
        if (age < 18) {
            return "Minor";
        } else {
            return "Adult";
        }
    }
    return "Invalid";
}

Good - Return Early:

string GetStatus(int age) {
    if (age < 0) return "Invalid";
    if (age < 18) return "Minor";
    return "Adult";
}

INTERVIEW QUESTIONS

Q: Difference between void and return?
A: void = no return value. return = gives back a value to caller.

Q: Can you have two methods with same name?
A: Yes, with method overloading. Different parameters required.

Q: When use ref or out?
A: ref to modify original variable. out to return multiple values.

Q: What does params do?
A: Accepts variable number of arguments of same type.

Q: Why use default parameters?
A: Make methods flexible. Caller can skip optional parameters.

Q: What's method overloading used for?
A: Shows polymorphism understanding. Code reusability.

Q: How validate inputs in method?
A: Check parameters at start. Throw exceptions if invalid.

Q: When use return early pattern?
A: Avoid deep nesting. Makes code cleaner and more readable.

KEY TAKEAWAYS

- Method = reusable code block for one task
- Parameters = inputs. Return values = outputs.
- void = no return value
- return = gives back value
- Method overloading = same name, different parameters
- Default parameters = optional inputs
- params = variable arguments
- Use PascalCase for method names
- Return early = avoid nesting
- Methods make code organized and maintainable
- Always validate inputs
- Use meaningful names: CalculateTotal() not Calc()
