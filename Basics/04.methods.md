# 🔧 Methods in C#

---

## 🧠 What is a Method?

A **method** is a reusable block of code that does one specific job. Instead of writing the same code again and again, you write it once and call it whenever you need it.

```csharp
// ❌ Without methods — repeated code
Console.WriteLine("Hello, Asha!");
Console.WriteLine("Hello, Ravi!");
Console.WriteLine("Hello, Priya!");

// ✅ With a method — write once, use many times
void Greet(string name) => Console.WriteLine($"Hello, {name}!");

Greet("Asha");
Greet("Ravi");
Greet("Priya");
```

## 🏗️ Method Structure

```csharp
// accessModifier returnType MethodName(parameters)
public int Add(int a, int b)
{
    return a + b;
}

// Parts:
// public     — who can call this
// int        — what it returns
// Add        — method name
// (int a, int b) — parameters (input)
// return a + b   — return value (output)
```

## 🔄 Return Types

```csharp
// void — returns nothing
void PrintHello() => Console.WriteLine("Hello!");

// int — returns a number
int Square(int n) => n * n;

// string — returns text
string Greet(string name) => $"Hello, {name}!";

// bool — returns true/false
bool IsEven(int n) => n % 2 == 0;

// Multiple values — return a tuple
(int Min, int Max) FindRange(int[] nums)
    => (nums.Min(), nums.Max());

var (min, max) = FindRange(new[] { 3, 1, 5, 2 });
Console.WriteLine($"Min: {min}, Max: {max}");   // Min: 1, Max: 5
```

## 🎯 Method Overloading

Same name, different parameters — compiler picks the right one:

```csharp
// Three versions of Add
int    Add(int a, int b)          => a + b;
double Add(double a, double b)    => a + b;
int    Add(int a, int b, int c)   => a + b + c;

Console.WriteLine(Add(1, 2));         // 3   — int version
Console.WriteLine(Add(1.5, 2.5));     // 4.0 — double version
Console.WriteLine(Add(1, 2, 3));      // 6   — three-param version
```

## 🎁 Optional Parameters & Named Arguments

```csharp
// Optional — has a default value
void CreateUser(string name, int age = 18, bool isActive = true)
{
    Console.WriteLine($"{name}, {age}, Active: {isActive}");
}

CreateUser("Asha");              // Asha, 18, Active: True
CreateUser("Ravi", 25);          // Ravi, 25, Active: True
CreateUser("Priya", 30, false);  // Priya, 30, Active: False

// Named arguments — pass in any order
CreateUser(age: 22, name: "Kiran", isActive: true);
```

## 📤 ref, out, in Parameters

```csharp
// ref — pass by reference (read AND write)
void Double(ref int n) { n *= 2; }
int x = 5;
Double(ref x);
Console.WriteLine(x);   // 10

// out — must be assigned inside (write only)
bool TryDivide(int a, int b, out int result)
{
    if (b == 0) { result = 0; return false; }
    result = a / b;
    return true;
}
if (TryDivide(10, 2, out int val))
    Console.WriteLine(val);   // 5

// in — read-only reference (performance for large structs)
void Print(in int n) { Console.WriteLine(n); }
```

## 🌊 params — Variable Arguments

```csharp
int Sum(params int[] numbers)
{
    return numbers.Sum();
}

Console.WriteLine(Sum(1, 2, 3));         // 6
Console.WriteLine(Sum(1, 2, 3, 4, 5));  // 15
Console.WriteLine(Sum());                // 0

string JoinAll(string separator, params string[] words)
    => string.Join(separator, words);

Console.WriteLine(JoinAll(", ", "Asha", "Ravi", "Priya")); // Asha, Ravi, Priya
```

## ➡️ Expression Body Methods

```csharp
// Full method
public int Add(int a, int b)
{
    return a + b;
}

// Expression body — same thing, one line ✅
public int Add(int a, int b) => a + b;
public string Greet(string name) => $"Hello, {name}!";
public bool IsAdult(int age) => age >= 18;
public double Area(double r) => Math.PI * r * r;
```

## 🔁 Recursion

A method that calls itself:

```csharp
// Factorial: 5! = 5 × 4 × 3 × 2 × 1 = 120
int Factorial(int n)
{
    if (n <= 1) return 1;        // base case — stops recursion
    return n * Factorial(n - 1); // recursive call
}
Console.WriteLine(Factorial(5));   // 120

// How it works:
// Factorial(5) = 5 × Factorial(4)
// Factorial(4) = 4 × Factorial(3)
// Factorial(3) = 3 × Factorial(2)
// Factorial(2) = 2 × Factorial(1)
// Factorial(1) = 1  ← base case
```

## 🌍 Real-World Example

```csharp
public class OrderCalculator
{
    // Calculate subtotal
    public decimal GetSubtotal(List<decimal> prices)
        => prices.Sum();

    // Apply discount
    public decimal ApplyDiscount(decimal total, decimal discountPercent)
        => total - (total * discountPercent / 100);

    // Calculate tax
    public decimal AddTax(decimal amount, decimal taxPercent = 18)
        => amount + (amount * taxPercent / 100);

    // Full order total
    public decimal GetFinalTotal(List<decimal> prices, decimal discount = 0)
    {
        decimal subtotal    = GetSubtotal(prices);
        decimal discounted  = ApplyDiscount(subtotal, discount);
        decimal withTax     = AddTax(discounted);
        return Math.Round(withTax, 2);
    }
}

var calc   = new OrderCalculator();
var prices = new List<decimal> { 999m, 599m, 149m };
Console.WriteLine(calc.GetFinalTotal(prices, discount: 10));
```

## 🎤 Interview Q&A

**Q1: What is method overloading?**
A: Same method name with different parameter types or counts. Compiler picks the right version.
```csharp
void Print(int n)    => Console.WriteLine(n);
void Print(string s) => Console.WriteLine(s);
```

**Q2: What is the difference between `ref` and `out`?**
A: `ref` = must be initialized before passing, can read and write. `out` = doesn't need init, must be assigned inside.
```csharp
void Double(ref int n) { n *= 2; }    // reads and writes
bool Try(out int n)    { n = 5; return true; }  // must assign
```

**Q3: What is recursion?**
A: A method that calls itself. Must have a base case to stop — otherwise infinite loop and stack overflow.

**Q4: What is a void method?**
A: A method that returns nothing. Used for side effects — printing, saving, sending emails.

**Q5: What are optional parameters?**
A: Parameters with default values — caller can skip them.
```csharp
void Greet(string name, string greeting = "Hello")
    => Console.WriteLine($"{greeting}, {name}!");
Greet("Asha");           // Hello, Asha!
Greet("Asha", "Hi");     // Hi, Asha!
```

## 🚀 Key Takeaways

✅ Methods = reusable blocks of code — write once, call many times  
✅ Return type defines what comes out — `void` for nothing  
✅ Overloading = same name, different params  
✅ `params` = variable number of arguments  
✅ `ref` reads & writes · `out` must be assigned inside  
✅ Optional params have defaults · named args pass in any order  
✅ Expression body `=>` = concise one-liners  
✅ Recursion needs a base case — or you get a stack overflow!
