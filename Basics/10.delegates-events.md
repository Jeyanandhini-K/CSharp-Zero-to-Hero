# 📢 Delegates and Events in C#

## 🧠 What is a Delegate?

A **delegate** is a **type-safe function pointer** or **reference type that holds a method**. It allows you to pass methods as parameters and store them in variables.

---

## 📌 Basic Delegate

```csharp
// Define delegate type
public delegate void Greet(string name);

// Method matching delegate signature
public static void SayHello(string name)
{
    Console.WriteLine($"Hello, {name}!");
}

// Use delegate
Greet greeting = SayHello;
greeting("Ali");  // Hello, Ali!
```

---

## 🔄 Multiple Methods in Delegate

You can assign multiple methods to one delegate (multicast).

```csharp
public delegate void Notify();

public static void EmailNotification()
{
    Console.WriteLine("Sending email...");
}

public static void SMSNotification()
{
    Console.WriteLine("Sending SMS...");
}

public static void LogNotification()
{
    Console.WriteLine("Logging notification...");
}

// Combine methods
Notify notify = EmailNotification;
notify += SMSNotification;
notify += LogNotification;

notify();
// Output:
// Sending email...
// Sending SMS...
// Logging notification...
```

---

## 🎯 Built-in Delegates: Action & Func

Instead of defining custom delegates, use built-in ones.

### Action (void return)

```csharp
// Action with no parameters
Action greet = () => Console.WriteLine("Hello!");
greet();

// Action with parameters
Action<string> print = (text) => Console.WriteLine(text);
print("World");

// Action with multiple parameters
Action<int, int> add = (a, b) => Console.WriteLine(a + b);
add(5, 10);  // 15
```

### Func (has return value)

```csharp
// Func<input, output>
Func<int, int, int> multiply = (a, b) => a * b;
int result = multiply(5, 10);
Console.WriteLine(result);  // 50

Func<string, int> getLength = (str) => str.Length;
Console.WriteLine(getLength("Hello"));  // 5

Func<int, string> convertToString = (num) => num.ToString();
Console.WriteLine(convertToString(42));  // "42"
```

## 📢 What is an Event?

An **event** is a **wrapper around a delegate** that allows objects to **notify** other objects when something happens. It provides encapsulation.

---

## 📌 Basic Event

```csharp
public class Button
{
    // Define delegate
    public delegate void ClickDelegate();

    // Define event
    public event ClickDelegate OnClick;

    public void Click()
    {
        Console.WriteLine("Button clicked!");
        OnClick?.Invoke();  // Fire event if handlers exist
    }
}

class Program
{
    static void Main()
    {
        Button btn = new Button();
        
        // Subscribe to event
        btn.OnClick += () => Console.WriteLine("Handler 1 executed");
        btn.OnClick += () => Console.WriteLine("Handler 2 executed");

        btn.Click();
        // Output:
        // Button clicked!
        // Handler 1 executed
        // Handler 2 executed
    }
}
```

---

## 📊 Real-World Example: Temperature Monitor

```csharp
public class TemperatureMonitor
{
    private int temperature;

    // Event delegate
    public delegate void TemperatureChangedDelegate(int newTemp);

    // Event
    public event TemperatureChangedDelegate OnTemperatureWarning;

    public int Temperature
    {
        get { return temperature; }
        set
        {
            temperature = value;
            if (temperature > 30)
            {
                OnTemperatureWarning?.Invoke(temperature);
            }
        }
    }
}

public class AlertSystem
{
    public void SendAlert(int temp)
    {
        Console.WriteLine($"⚠️ WARNING: Temperature is {temp}°C - Too hot!");
    }
}

public class LogSystem
{
    public void LogWarning(int temp)
    {
        Console.WriteLine($"📝 LOGGED: High temperature {temp}°C recorded");
    }
}

// Usage
TemperatureMonitor monitor = new TemperatureMonitor();
AlertSystem alert = new AlertSystem();
LogSystem log = new LogSystem();

monitor.OnTemperatureWarning += alert.SendAlert;
monitor.OnTemperatureWarning += log.LogWarning;

monitor.Temperature = 25;   // No event
monitor.Temperature = 35;   // Triggers event
// Output:
// ⚠️ WARNING: Temperature is 35°C - Too hot!
// 📝 LOGGED: High temperature 35°C recorded
```

## 🔌 EventHandler Pattern (Standard)

C# has a standard event pattern using `EventHandler`.

```csharp
public class Button
{
    // Standard event with EventHandler
    public event EventHandler OnClick;

    public void Click()
    {
        Console.WriteLine("Button clicked!");
        OnClick?.Invoke(this, EventArgs.Empty);
    }
}

class Program
{
    static void HandleClick(object sender, EventArgs e)
    {
        Console.WriteLine("Button was clicked!");
    }

    static void Main()
    {
        Button btn = new Button();
        btn.OnClick += HandleClick;
        btn.Click();
    }
}
```

## 🎯 Real-World: Button Click Event

```csharp
public class CustomButton
{
    public event EventHandler<EventArgs> Clicked;

    public void Click()
    {
        Clicked?.Invoke(this, EventArgs.Empty);
    }
}

class Program
{
    static void Main()
    {
        CustomButton btn = new CustomButton();

        btn.Clicked += (sender, e) => Console.WriteLine("Button 1 handled");
        btn.Clicked += (sender, e) => Console.WriteLine("Button 2 handled");

        btn.Click();
        // Button 1 handled
        // Button 2 handled
    }
}
```

## 📋 Summary Table

| Concept | Purpose | Example |
|---|---|---|
| **Delegate** | Function pointer/reference | `public delegate void Notify();` |
| **Action** | Void method reference | `Action<string> print = (x) => {};` |
| **Func** | Method with return value | `Func<int,int> add = (a,b) => a+b;` |
| **Event** | Notification mechanism | `public event EventHandler OnClick;` |
| **+=** | Subscribe to event | `btn.OnClick += handler;` |
| **-=** | Unsubscribe from event | `btn.OnClick -= handler;` |
| **?.Invoke()** | Fire event safely | `OnClick?.Invoke();` |

## 🎤 Interview Q&A

### Q1: What is a delegate?
**A:** Type-safe function pointer. Holds reference to a method.

```csharp
public delegate void MyDelegate();
MyDelegate d = SomeMethod;
d();
```

---

### Q2: What is the difference between delegate and event?
**A:**
- **Delegate** = Function pointer
- **Event** = Encapsulated delegate (can only subscribe/unsubscribe)

```csharp
public delegate void MyDelegate();
public event MyDelegate MyEvent;  // Event wraps delegate
```

---

### Q3: What is Action?
**A:** Built-in delegate for void methods.

```csharp
Action<string> print = (s) => Console.WriteLine(s);
print("Hello");
```

---

### Q4: What is Func?
**A:** Built-in delegate that returns a value.

```csharp
Func<int, int, int> add = (a, b) => a + b;
int result = add(5, 10);  // 15
```

---

### Q5: How do you subscribe to an event?
**A:** Use `+=` operator.

```csharp
button.OnClick += () => Console.WriteLine("Clicked");
```

---

### Q6: How do you unsubscribe from an event?
**A:** Use `-=` operator.

```csharp
button.OnClick -= handler;
```

---

### Q7: What is multicast delegate?
**A:** Delegate with multiple methods assigned.

```csharp
Action notify = Method1;
notify += Method2;
notify += Method3;
notify();  // Calls all three
```

---

### Q8: How do you safely fire an event?
**A:** Use `?.Invoke()` to check if handlers exist.

```csharp
OnClick?.Invoke();  // Only invokes if OnClick is not null
```

---

### Q9: What is EventHandler?
**A:** Standard delegate pattern for events.

```csharp
public event EventHandler OnClick;
OnClick?.Invoke(this, EventArgs.Empty);
```

---

### Q10: When should you use events?
**A:** When you want to notify multiple objects about something happening.

```csharp
// Button notifies listeners when clicked
// Temperature monitor notifies listeners when too hot
// Database notifies listeners when data changes
```

---

## 🚀 Key Takeaways

✅ Delegate = Function pointer  
✅ Event = Encapsulated delegate  
✅ Action = Void delegate  
✅ Func = Return value delegate  
✅ += to subscribe, -= to unsubscribe  
✅ ?.Invoke() to safely fire event  
✅ Events enable loose coupling
