# 🎯 Pattern Matching & Tuples in C#

---

## 📦 What is a Tuple?

A **tuple** bundles multiple values together — no class needed.

```csharp
var point = (3, 4);
Console.WriteLine(point.Item1);   // 3
Console.WriteLine(point.Item2);   // 4
```

### ✏️ Named Tuples (much more readable)

```csharp
var person = (Name: "Asha", Age: 30);
Console.WriteLine(person.Name);   // Asha
Console.WriteLine(person.Age);    // 30
```

---

## 🔄 Returning Multiple Values

Before tuples → messy `out` parameters or a whole class.
With tuples → clean and simple ✅

```csharp
(int Min, int Max) FindRange(int[] numbers)
{
    return (numbers.Min(), numbers.Max());
}

var range = FindRange(new[] { 4, 1, 9, 2 });
Console.WriteLine($"{range.Min} to {range.Max}");   // 1 to 9
```

### 💥 Deconstruction — unpack in one line

```csharp
var (low, high) = FindRange(new[] { 4, 1, 9, 2 });
Console.WriteLine(low);    // 1
Console.WriteLine(high);   // 9
```

> 💡 Use tuples for **small, temporary groupings**. If data lives long or needs behavior → use a `class` or `record`.

## 🔍 What is Pattern Matching?

**Pattern matching** tests a value's type, value, or shape — and extracts data at the same time. Replaces messy `if/else` chains.

---

## ✅ The `is` Pattern

Check type AND get a typed variable — in one shot:

```csharp
object obj = "hello";

if (obj is string s)
{
    Console.WriteLine(s.Length);   // 5
}
```

Most common use — null check:

```csharp
if (value is not null)
{
    // safe to use!
}
```

---

## ⚡ Switch Expression (modern `switch`)

Returns a value. No `break`. No `case`. Just clean `=>` arms:

```csharp
string Describe(int n) => n switch
{
    0   => "zero",
    > 0 => "positive",
    < 0 => "negative",
};
```

---

## 📐 Relational & Logical Patterns

Use `<`, `>`, `<=`, `>=` and combine with `and` / `or` / `not`:

```csharp
string Grade(int score) => score switch
{
    >= 90 => "A",
    >= 80 => "B",
    >= 70 => "C",
    >= 60 => "D",
    _     => "F",       // _ = catch-all (like default)
};

bool IsWeekend(int day)      => day is 6 or 7;
bool IsValidPercent(int p)   => p is >= 0 and <= 100;
```

---

## 🏷️ Type Patterns in Switch

Branch on the actual type of an object:

```csharp
decimal Area(object shape) => shape switch
{
    Circle c    => (decimal)(Math.PI * c.Radius * c.Radius),
    Rectangle r => r.Width * r.Height,
    _           => throw new ArgumentException("Unknown shape"),
};
```

---

## 🏠 Property Patterns

Match on property values directly:

```csharp
string Shipping(Order o) => o switch
{
    { Total: > 100 }   => "Free shipping",
    { Country: "US" }  => "Standard",
    _                  => "International",
};
```

---

## 🎮 Tuples + Patterns + `when`

Combine for powerful rule tables:

```csharp
string RockPaperScissors(string a, string b) => (a, b) switch
{
    ("rock",    "scissors") => "Player 1 wins",
    ("scissors","paper")    => "Player 1 wins",
    ("paper",   "rock")     => "Player 1 wins",
    _ when a == b           => "Tie",
    _                       => "Player 2 wins",
};
```

## 📋 Summary Table

| Feature | Syntax | What it does |
|---|---|---|
| Named tuple | `(Name: "Asha", Age: 30)` | Bundle values with names |
| Deconstruct | `var (a, b) = tuple;` | Unpack tuple into variables |
| `is` pattern | `obj is string s` | Check type + extract |
| Switch expression | `x switch { ... }` | Clean value-returning switch |
| Relational | `>= 90` | Compare values in patterns |
| Logical | `and` / `or` / `not` | Combine patterns |
| Property | `{ Name: "Asha" }` | Match on properties |
| Catch-all | `_` | Matches anything (like `default`) |
| `when` guard | `_ when x == y` | Extra condition on a pattern |

## 🎤 Interview Q&A

**Q1: What is a tuple?**
A: A lightweight way to group multiple values without creating a class.
```csharp
var p = (Name: "Asha", Age: 30);
Console.WriteLine(p.Name);   // Asha
```

**Q2: How do you return multiple values from a method?**
A: Use a tuple as the return type.
```csharp
(int Min, int Max) FindRange(int[] nums) => (nums.Min(), nums.Max());
```

**Q3: What is pattern matching?**
A: Testing a value's type, value, or shape and extracting data — all in one expression.

**Q4: What does `_` mean in a switch expression?**
A: It's the catch-all — matches anything that didn't match above. Like `default` in classic switch.
```csharp
string result = x switch { 1 => "one", _ => "other" };
```

**Q5: What is a property pattern?**
A: Matching on a property value inside `{ }`.
```csharp
string msg = order switch { { Total: > 100 } => "Free!", _ => "Paid" };
```

**Q6: What is a `when` guard?**
A: An extra condition added to a pattern arm.
```csharp
_ when a == b => "Tie"
```

**Q7: What's the difference between `switch` statement and `switch` expression?**
A: Statement uses `case/break` and doesn't return a value. Expression uses `=>`, returns a value, and is much shorter.

**Q8: Can you combine `and` / `or` in patterns?**
A: Yes!
```csharp
bool valid = score is >= 0 and <= 100;
bool weekend = day is 6 or 7;
```

## 🚀 Key Takeaways

✅ Tuples bundle values — no class needed  
✅ Deconstruct tuples into separate variables  
✅ `switch` expressions return values — cleaner than `if/else`  
✅ `_` = catch-all in patterns  
✅ Combine `and` / `or` / `not` for expressive logic  
✅ Property patterns match on object properties  
✅ `when` adds extra conditions to pattern arms
