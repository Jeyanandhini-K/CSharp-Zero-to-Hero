# 🚫 Null Handling in C#

## 🧠 What is Null?

`null` means "no value" or "nothing". It represents the absence of an object.

```csharp
string name = null;  // No value assigned
int? age = null;     // Nullable int

Console.WriteLine(name);  // (blank)
Console.WriteLine(age);   // (blank)
```

---

## ⚠️ Null Reference Exception

The most common runtime error: trying to use null value.

```csharp
string text = null;
Console.WriteLine(text.Length);  // ❌ NullReferenceException!

int? num = null;
Console.WriteLine(num + 5);      // ❌ Error (can't do math with null)
```

---

## 🛡️ Nullable Types

A type that can hold `null` value. Use `?` suffix.

```csharp
// Non-nullable (cannot be null)
int age = 0;         // Must have a value
string name = "";    // Must have a value

// Nullable (can be null)
int? age = null;     // Can be null
string? name = null; // Can be null
double? price = null;
bool? isActive = null;
```

---

## 🔍 Null Coalescing Operator `??`

Provide a default value if variable is null.

```csharp
string? name = null;
string displayName = name ?? "Anonymous";

Console.WriteLine(displayName);  // Anonymous

int? age = 25;
int displayAge = age ?? 0;

Console.WriteLine(displayAge);   // 25
```

---

## 🔗 Null Coalescing Assignment `??=`

Assign a value only if variable is null.

```csharp
string? message = null;
message ??= "Default message";
Console.WriteLine(message);  // Default message

message ??= "Another message";
Console.WriteLine(message);  // Default message (not changed)
```

---

## 🛡️ Null-Conditional Operator `?.`

Safely call methods/properties on nullable objects.

```csharp
string? text = null;

// ❌ Error
// Console.WriteLine(text.Length);

// ✅ Safe
int? length = text?.Length;
Console.WriteLine(length);  // (null)

// Safe method call
text?.ToUpper();  // Doesn't execute if text is null
```

---

## 🔗 Null-Conditional with Coalescing

Combine `?.` and `??` for full safety.

```csharp
string? text = null;
int length = text?.Length ?? 0;

Console.WriteLine(length);  // 0 (null, so use default)

text = "Hello";
length = text?.Length ?? 0;

Console.WriteLine(length);  // 5 (has value)
```

## 📝 Real-World Example: User Profile

```csharp
public class User
{
    public string? Name { get; set; }
    public string? Email { get; set; }
    public int? Age { get; set; }
    public Profile? Profile { get; set; }
}

public class Profile
{
    public string? Bio { get; set; }
    public string? Website { get; set; }
}

// Safe navigation
User? user = GetUser();

string displayName = user?.Name ?? "Unknown";
int userAge = user?.Age ?? 0;
string bio = user?.Profile?.Bio ?? "No bio";

Console.WriteLine(displayName);  // Safe
Console.WriteLine(userAge);      // Safe
Console.WriteLine(bio);          // Safe - even if Profile is null
```

---

## ✅ Null Checks Before Use

Always check for null before using.

```csharp
public class DataProcessor
{
    public void ProcessUser(User? user)
    {
        // Method 1: if statement
        if (user != null)
        {
            Console.WriteLine(user.Name);
        }

        // Method 2: null coalescing
        string name = user?.Name ?? "Unknown";
        Console.WriteLine(name);

        // Method 3: null-forgiving operator ! (use carefully)
        Console.WriteLine(user!.Name);  // Tells compiler: "Trust me, it's not null"
    }
}
```

## 🎤 Interview Q&A

### Q1: What does null mean?
**A:** No value assigned to a variable. Represents absence of object.

```csharp
string text = null;  // No value
```

---

### Q2: What is NullReferenceException?
**A:** Runtime error thrown when you use a null value.

```csharp
string text = null;
Console.WriteLine(text.Length);  // ❌ NullReferenceException
```

---

### Q3: What are nullable types?
**A:** Types that can hold null value. Use `?` suffix.

```csharp
int? age = null;         // Can be null
string? name = null;     // Can be null
```

---

### Q4: What is the null coalescing operator?
**A:** `??` provides default value if left side is null.

```csharp
string? name = null;
string display = name ?? "Anonymous";  // Anonymous
```

---

### Q5: What is the null-conditional operator?
**A:** `?.` safely accesses members without throwing error if null.

```csharp
string? text = null;
int? length = text?.Length;  // null, not error
```

---

### Q6: How do you combine null-conditional with coalescing?
**A:** Use `?.` then `??` together.

```csharp
string? text = null;
int length = text?.Length ?? 0;  // 0 if null
```

---

### Q7: What is `??=` operator?
**A:** Assign value only if variable is null.

```csharp
string? message = null;
message ??= "Default";  // Assigns
message ??= "New";      // Doesn't assign
```

---

### Q8: When should you use nullable reference types?
**A:** When a variable might legitimately be null.

```csharp
string? optionalEmail = null;   // Might not have email
string requiredName = "Ali";    // Must have name
```

---

### Q9: What is the null-forgiving operator `!`?
**A:** Tells compiler "Trust me, this isn't null". Use carefully!

```csharp
string? text = null;
string safe = text!.ToUpper();  // ❌ Still crashes at runtime!
```

---

### Q10: How do you safely check for null?
**A:** Use `if (value != null)` or null coalescing.

```csharp
// Method 1
if (user != null)
{
    Console.WriteLine(user.Name);
}

// Method 2
string name = user?.Name ?? "Unknown";
```

## 🚀 Key Takeaways

✅ `null` = No value assigned  
✅ `?` = Nullable type  
✅ `??` = Default if null  
✅ `?.` = Safe property/method access  
✅ `??=` = Assign only if null  
✅ Always check for null before using  
✅ Prevents NullReferenceException
