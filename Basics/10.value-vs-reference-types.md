# 🧠 Value vs Reference Types in C#

---

## 🔑 The Big Idea

Every type in C# is either a **value type** or a **reference type**. This decides what happens when you assign or pass it — and it's one of the most important concepts in C#.

```csharp
// Value type — COPIES the value
int a = 10;
int b = a;
b = 99;
Console.WriteLine(a);   // 10 — unchanged ✅

// Reference type — SHARES the object
var list1 = new List<int> { 1, 2, 3 };
var list2 = list1;
list2.Add(99);
Console.WriteLine(list1.Count);   // 4 — also changed! ⚠️
```

---

## 📦 Value Types

Value types store data **directly** — on the stack. Assignment makes a **copy**.

```csharp
// All of these are value types
int    age    = 25;
double price  = 9.99;
bool   active = true;
char   grade  = 'A';

// Structs are value types
DateTime now   = DateTime.Now;
Guid     id    = Guid.NewGuid();

// Enums are value types
DayOfWeek day = DayOfWeek.Monday;

// Custom struct — value type you define
struct Point { public int X; public int Y; }
Point p1 = new Point { X = 1, Y = 2 };
Point p2 = p1;   // COPY
p2.X = 99;
Console.WriteLine(p1.X);   // 1 — unchanged ✅
```

---

## 🔗 Reference Types

Reference types store a **reference** (pointer) to data on the heap. Assignment shares the reference.

```csharp
// All of these are reference types
string   name  = "Asha";
int[]    nums  = { 1, 2, 3 };
object   obj   = new object();

// Classes are reference types
public class Person { public string Name { get; set; } = ""; }

Person person1 = new Person { Name = "Asha" };
Person person2 = person1;   // SAME OBJECT — not a copy!
person2.Name = "Ravi";
Console.WriteLine(person1.Name);   // "Ravi" — both changed! ⚠️
```

---

## 🗺️ Memory: Stack vs Heap

```
STACK (value types)          HEAP (reference types)
┌─────────────┐              ┌──────────────────────┐
│  a = 10     │              │  Person { Name=Asha } │
│  b = 10     │  ←copy       │         ↑            │
│  p1 = {1,2} │              │  person1 ──────────── │
│  p2 = {1,2} │  ←copy       │  person2 ──────────── │
└─────────────┘              └──────────────────────┘

Value types: each variable has its OWN copy of the data
Reference types: variables point to the SAME object
```

---

## 🔡 String — The Special Case

`string` is a reference type — but behaves like a value type because it's **immutable**:

```csharp
string s1 = "hello";
string s2 = s1;      // shares the reference
s2 = "world";        // creates a NEW string — doesn't modify s1!
Console.WriteLine(s1);   // "hello" — unchanged ✅

// Each operation creates a NEW string
string s = "Hello";
s = s + " World";   // new string object — old one discarded
s = s.ToUpper();    // another new string
```

## 🔄 Passing to Methods

```csharp
// Value type — method gets a COPY
void DoubleIt(int n) { n *= 2; }
int x = 5;
DoubleIt(x);
Console.WriteLine(x);   // 5 — unchanged ✅ (method had a copy)

// Reference type — method gets the SAME object
void AddItem(List<int> list) { list.Add(99); }
var myList = new List<int> { 1, 2, 3 };
AddItem(myList);
Console.WriteLine(myList.Count);   // 4 — changed! ⚠️

// Force value type to be shared — use 'ref'
void DoubleRef(ref int n) { n *= 2; }
int y = 5;
DoubleRef(ref y);
Console.WriteLine(y);   // 10 ✅
```

## 📋 Summary Table

| | Value Type | Reference Type |
|---|---|---|
| Stored on | Stack | Heap |
| Assignment | Copies the value | Copies the reference |
| Examples | `int`, `bool`, `struct`, `enum` | `class`, `string`, `array`, `List` |
| Default value | `0`, `false`, etc. | `null` |
| Nullable | needs `?` suffix | nullable by default |
| Performance | Faster (stack) | Slightly slower (heap) |

## 🎤 Interview Q&A

**Q1: What is the difference between value types and reference types?**
A: Value types copy data on assignment (independent). Reference types copy the reference (shared object).
```csharp
int a = 5; int b = a; b = 10; // a = 5 still
var p1 = new Person(); var p2 = p1; p2.Name = "X"; // p1.Name = "X" too!
```

**Q2: Where are value and reference types stored?**
A: Value types on the stack. Reference types on the heap (with a reference on the stack).

**Q3: Is `string` a value or reference type?**
A: Reference type — but immutable. Any "change" creates a new string object, so it behaves like a value type.

**Q4: What is boxing and unboxing?**
A: Boxing = wrapping a value type in an object (heap allocation). Unboxing = extracting it back.
```csharp
int n = 42;
object boxed = n;       // boxing   — value type → reference type
int unboxed = (int)boxed; // unboxing — reference type → value type
```

**Q5: How do you pass a value type by reference?**
A: Use the `ref` keyword.
```csharp
void Double(ref int n) { n *= 2; }
int x = 5; Double(ref x); // x = 10
```

---

## 🚀 Key Takeaways

✅ Value types = **copied** on assignment — independent  
✅ Reference types = **shared** on assignment — same object  
✅ Value types on **stack** · Reference types on **heap**  
✅ `string` = reference type but **immutable** — behaves like value type  
✅ Methods get copies of value types — changes don't affect caller  
✅ Methods get references to reference types — changes DO affect caller  
✅ Use `ref` to pass value types by reference
