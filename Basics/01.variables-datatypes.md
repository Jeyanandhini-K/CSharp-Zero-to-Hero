## рҹ§  What is a Variable?

A variable is a **named container** that stores a value. Think of it like a labeled box вҖ” the label is the name, what's inside is the value.

```csharp
int    age    = 25;
string name   = "Asha";
double salary = 50000.50;
bool   active = true;
```

## рҹ”ў Value Types вҖ” Integers

```csharp
byte   b  = 255;           // 0 to 255
short  s  = 32000;         // -32,768 to 32,767
int    i  = 2147483647;    // most common вң…
long   l  = 9999999999L;   // very large numbers

// Useful int methods
int max = int.MaxValue;    // 2,147,483,647
int min = int.MinValue;    // -2,147,483,648
int abs = Math.Abs(-42);   // 42
```

---

## рҹ”Ј Value Types вҖ” Decimals

```csharp
float   f = 3.14f;          // 7 digits precision  (suffix f)
double  d = 3.14159265358;  // 15 digits precision вң… most common
decimal m = 99.99m;         // 28 digits вҖ” use for MONEY (suffix m)

// вқҢ Never use float/double for money
double price = 0.1 + 0.2;
Console.WriteLine(price);   // 0.30000000000000004 рҹҳұ

// вң… Use decimal for money
decimal price = 0.1m + 0.2m;
Console.WriteLine(price);   // 0.3 вң…
```

## рҹ”Ө String & Char

```csharp
// string вҖ” text (reference type)
string name     = "Asha";
string greeting = "Hello, " + name;        // concatenation
string inter    = $"Hello, {name}!";       // interpolation вң…
string multi    = "Line1\nLine2";          // newline

// char вҖ” single character (value type)
char grade  = 'A';
char symbol = '@';

// Useful string methods
Console.WriteLine("hello".ToUpper());       // HELLO
Console.WriteLine("HELLO".ToLower());       // hello
Console.WriteLine("  hello  ".Trim());      // hello
Console.WriteLine("hello".Length);          // 5
Console.WriteLine("hello".Contains("ell")); // True
Console.WriteLine("hello".Replace("l","r")); // herro
```

## рҹҺҜ var вҖ” Let Compiler Infer the Type

```csharp

var name    = "Asha";       // string
var price   = 9.99m;        // decimal
var list    = new List<int>();

// вқҢ var when type is unclear

// var is NOT dynamic вҖ” type is fixed at compile time

x = "hello";   // вқҢ compile error вҖ” x is int!
```

## рҹ”„ Type Conversion

```csharp
// Implicit вҖ” safe, automatic
int    i = 10;
double d = i;   // int вҶ’ double вң… no data loss

// Explicit (cast) вҖ” might lose data
double pi     = 3.14;
int    approx = (int)pi;   // 3 вҖ” decimal lost!

// Convert class
string numStr = "42";
int    num    = Convert.ToInt32(numStr);
double dbl    = Convert.ToDouble("3.14");
string str    = Convert.ToString(42);

// Safe parse вҖ” no exception if invalid
bool ok = int.TryParse("abc", out int result);
Console.WriteLine(ok);      // False
Console.WriteLine(result);  // 0 вҖ” default
```

## рҹҺӨ Interview Q&A

**Q1: What is the difference between `int` and `long`?**
A: `int` = 32-bit (Вұ2.1 billion). `long` = 64-bit (much larger). Use `long` when values exceed int range.
```csharp
int  population = 1_400_000_000;   // India's population вҖ” fits in int
long distance   = 9_460_730_472_580_800L; // light year in meters вҖ” needs long
```

**Q2: Why use `decimal` instead of `double` for money?**
A: `double` has floating-point precision errors. `decimal` is exact to 28 digits.
```csharp
Console.WriteLine(0.1 + 0.2);    // 0.30000000000000004 вқҢ
Console.WriteLine(0.1m + 0.2m);  // 0.3 вң…
```

**Q3: What is `var` and is it the same as `dynamic`?**
A: `var` = compile-time type inference вҖ” type is fixed. `dynamic` = runtime type вҖ” can change. `var` is safe; `dynamic` bypasses type checking.

**Q4: What is the difference between `Convert.ToInt32` and `int.Parse`?**
A: Both convert string to int. `Convert.ToInt32(null)` returns 0; `int.Parse(null)` throws exception. `int.TryParse` is safest вҖ” no exception.

**Q5: What is implicit vs explicit conversion?**
A: Implicit = automatic, safe (intвҶ’double). Explicit = manual cast, might lose data (doubleвҶ’int).
```csharp
int i = 10; double d = i;    // implicit вң…
double pi = 3.14; int n = (int)pi; // explicit вҖ” 3
```

## рҹҡҖ Key Takeaways

вң… Variables are named containers for values  
вң… Use `int` for whole numbers, `double` for decimals, `decimal` for money  
вң… Use `string` for text, `char` for single characters, `bool` for true/false  
вң… `var` = compiler infers type вҖ” still strongly typed  
вң… Use `decimal` (not `double`) for financial calculations  
вң… Use `int.TryParse` for safe string-to-number conversion  
вң… Explicit cast `(int)` can lose data вҖ” use carefully
