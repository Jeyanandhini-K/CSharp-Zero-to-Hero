# ⚠️ Exception Handling in C#

---

## 🧠 What is an Exception?

An **exception** is an error that happens while your program is running. Without handling it, your app crashes. With handling — you catch it, deal with it, and keep going.

```csharp
// ❌ No exception handling — app crashes!
int[] numbers = { 1, 2, 3 };
Console.WriteLine(numbers[10]);   // 💥 IndexOutOfRangeException!

// ✅ With exception handling — graceful recovery
try
{
    Console.WriteLine(numbers[10]);
}
catch (IndexOutOfRangeException ex)
{
    Console.WriteLine($"Error: {ex.Message}");   // Error: Index was outside...
}
```

## 🛡️ try / catch / finally

```csharp
try
{
    // Code that might throw
    int result = 10 / 0;
}
catch (DivideByZeroException ex)
{
    // Handle the specific exception
    Console.WriteLine($"Can't divide by zero: {ex.Message}");
}
catch (Exception ex)
{
    // Catch-all — for any other exception
    Console.WriteLine($"Something went wrong: {ex.Message}");
}
finally
{
    // ALWAYS runs — even if exception, even if return ✅
    Console.WriteLine("Cleanup done!");
}
```

## 🎯 Common Exceptions

```csharp
// NullReferenceException — using null
string? name = null;
Console.WriteLine(name.Length);   // 💥 NullReferenceException
// Fix: null check first
if (name is not null) Console.WriteLine(name.Length);

// IndexOutOfRangeException — bad array index
int[] arr = { 1, 2, 3 };
Console.WriteLine(arr[5]);   // 💥 IndexOutOfRangeException
// Fix: check length
if (arr.Length > 5) Console.WriteLine(arr[5]);

// FormatException — wrong string format
int n = int.Parse("abc");   // 💥 FormatException
// Fix: use TryParse
bool ok = int.TryParse("abc", out int result);

// DivideByZeroException
int x = 10 / 0;   // 💥 DivideByZeroException
// Fix: check before dividing
if (b != 0) Console.WriteLine(a / b);

// OverflowException
checked { int max = int.MaxValue + 1; }   // 💥 OverflowException

// FileNotFoundException
File.ReadAllText("missing.txt");   // 💥 FileNotFoundException
```

## 🎭 Multiple catch Blocks

```csharp
// Catch specific exceptions first, general last
try
{
    string text = File.ReadAllText("data.txt");
    int    num  = int.Parse(text);
    int    result = 100 / num;
    Console.WriteLine(result);
}
catch (FileNotFoundException ex)
{
    Console.WriteLine($"File not found: {ex.FileName}");
}
catch (FormatException ex)
{
    Console.WriteLine($"File content is not a number: {ex.Message}");
}
catch (DivideByZeroException)
{
    Console.WriteLine("File contains zero — can't divide!");
}
catch (Exception ex)
{
    Console.WriteLine($"Unexpected error: {ex.Message}");
}
finally
{
    Console.WriteLine("Done processing file.");
}
```

## 🚀 throw — Raise an Exception

```csharp
// Throw built-in exceptions
public void SetAge(int age)
{
    if (age < 0 || age > 150)
        throw new ArgumentOutOfRangeException(nameof(age), "Age must be 0–150");
    Age = age;
}

// Re-throw — preserve original stack trace
try { DoSomething(); }
catch (Exception ex)
{
    _logger.LogError(ex, "Error in DoSomething");
    throw;   // ✅ throw — preserves stack trace
    // throw ex;  ❌ resets stack trace — loses info!
}
```

## 🏗️ Custom Exceptions

```csharp
// Create your own exception
public class InsufficientFundsException : Exception
{
    public decimal Amount    { get; }
    public decimal Balance   { get; }

    public InsufficientFundsException(decimal amount, decimal balance)
        : base($"Cannot withdraw ₹{amount}. Balance is only ₹{balance}.")
    {
        Amount  = amount;
        Balance = balance;
    }
}

// Use it
public class BankAccount
{
    private decimal _balance = 1000m;

    public void Withdraw(decimal amount)
    {
        if (amount > _balance)
            throw new InsufficientFundsException(amount, _balance);

        _balance -= amount;
        Console.WriteLine($"Withdrawn ₹{amount}. New balance: ₹{_balance}");
    }
}

// Catch it
try { account.Withdraw(5000m); }
catch (InsufficientFundsException ex)
{
    Console.WriteLine(ex.Message);
    Console.WriteLine($"You need ₹{ex.Amount - ex.Balance} more.");
}
```

## ⚡ when — Exception Filters

```csharp
// Handle exception only when condition is true
try
{
    int result = GetResult();
}
catch (Exception ex) when (ex.Message.Contains("timeout"))
{
    Console.WriteLine("Request timed out — retry");
}
catch (Exception ex) when (ex.Message.Contains("not found"))
{
    Console.WriteLine("Resource not found");
}
catch (Exception ex)
{
    Console.WriteLine($"Other error: {ex.Message}");
}
```

## 🌍 Real-World Example: File Processor

```csharp
public class FileProcessor
{
    public async Task<string> ReadAndProcessAsync(string path)
    {
        try
        {
            if (!File.Exists(path))
                throw new FileNotFoundException($"File not found: {path}");

            string content = await File.ReadAllTextAsync(path);

            if (string.IsNullOrWhiteSpace(content))
                throw new InvalidOperationException("File is empty");

            return content.ToUpper();
        }
        catch (FileNotFoundException ex)
        {
            Console.WriteLine($"❌ {ex.Message}");
            return string.Empty;
        }
        catch (UnauthorizedAccessException)
        {
            Console.WriteLine("❌ No permission to read file");
            return string.Empty;
        }
        catch (Exception ex)
        {
            Console.WriteLine($"❌ Unexpected: {ex.Message}");
            throw;   // re-throw — let caller handle it
        }
        finally
        {
            Console.WriteLine("✅ File processing complete");
        }
    }
}
```

## 📋 Summary Table

| Keyword | Purpose |
|---|---|
| `try` | Wrap code that might throw |
| `catch` | Handle specific exception |
| `finally` | Always runs — cleanup |
| `throw` | Raise an exception |
| `throw;` | Re-throw, preserve stack trace |
| `throw ex;` | Re-throw, LOSES stack trace ❌ |
| `when` | Exception filter — conditional catch |
| Custom exception | Inherit from `Exception` |

## 🎤 Interview Q&A

**Q1: What is an exception?**
A: A runtime error that interrupts normal program flow. Caught with `try/catch` to handle gracefully.

**Q2: What is the purpose of `finally`?**
A: Always runs — whether exception occurred or not. Used for cleanup (close files, release connections).
```csharp
try { File.Open("f.txt"); }
catch { Console.WriteLine("Error"); }
finally { Console.WriteLine("Always runs"); }   // ✅
```

**Q3: What is the difference between `throw` and `throw ex`?**
A: `throw` preserves original stack trace. `throw ex` resets it — you lose where the error started.
```csharp
catch (Exception ex)
{
    throw;    // ✅ original stack trace preserved
    throw ex; // ❌ stack trace reset — bad!
}
```

**Q4: When should you create a custom exception?**
A: When built-in exceptions don't describe your specific error. Custom exceptions carry extra context.
```csharp
throw new InsufficientFundsException(amount, balance);
```

**Q5: What is the order of catch blocks?**
A: Most specific first, most general last. If `Exception` comes first, it catches everything and specific blocks never run.
```csharp
catch (FileNotFoundException ex) { }   // specific first ✅
catch (IOException ex)           { }   // more general
catch (Exception ex)             { }   // catch-all last
```

**Q6: What is `when` in exception handling?**
A: A filter — only catches if condition is true. Multiple catches for same type with different conditions.
```csharp
catch (Exception ex) when (ex.Message.Contains("timeout")) { }
```

**Q7: What is NullReferenceException and how to avoid it?**
A: Thrown when you access a member of a null object. Avoid with null checks or `?.` operator.
```csharp
string? name = null;
Console.WriteLine(name?.Length ?? 0);   // 0 — no exception ✅
```

## 🚀 Key Takeaways

✅ `try/catch/finally` — handle errors gracefully  
✅ Catch specific exceptions first, `Exception` last  
✅ `finally` always runs — use for cleanup  
✅ `throw;` preserves stack trace · `throw ex;` loses it  
✅ Use `TryParse` instead of `Parse` for safe conversion  
✅ Custom exceptions = inherit from `Exception`  
✅ `when` clause = conditional exception catching  
✅ Never swallow exceptions silently — always log them
