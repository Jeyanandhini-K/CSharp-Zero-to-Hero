# Exception Handling

Exceptions are errors that occur at runtime. Exception handling lets your program handle errors gracefully instead of crashing.

WHAT IS AN EXCEPTION?

An exception is an unexpected event that disrupts normal program flow.

Examples of Exceptions:

int[] numbers = { 1, 2, 3 };
Console.WriteLine(numbers[10]);  // IndexOutOfRangeException (index doesn't exist)

int result = 10 / 0;  // DivideByZeroException (can't divide by zero)

string text = null;
Console.WriteLine(text.Length);  // NullReferenceException (null has no properties)

int number = int.Parse("abc");  // FormatException (can't convert "abc" to int)

Without handling these, your program crashes. Exception handling catches and handles errors.

TRY-CATCH BLOCK

The try-catch block catches errors and handles them gracefully.

Basic Syntax:

try {
    // Code that might throw exception
}
catch (ExceptionType e) {
    // Handle the exception
    Console.WriteLine(e.Message);
}

Simple Example:

try {
    int[] numbers = { 1, 2, 3 };
    Console.WriteLine(numbers[10]);  // This throws exception
}
catch (IndexOutOfRangeException e) {
    Console.WriteLine("Error: Index out of range!");
    Console.WriteLine($"Details: {e.Message}");
}

Console.WriteLine("Program continues...");

// Output:
// Error: Index out of range!
// Details: Index was outside the bounds of the array.
// Program continues...

CATCH MULTIPLE EXCEPTION TYPES

try {
    Console.WriteLine("Enter a number:");
    int number = int.Parse(Console.ReadLine());
    
    int result = 100 / number;
    Console.WriteLine($"Result: {result}");
}
catch (FormatException e) {
    Console.WriteLine("Error: Please enter a valid number!");
}
catch (DivideByZeroException e) {
    Console.WriteLine("Error: Cannot divide by zero!");
}
catch (Exception e) {
    Console.WriteLine($"Unexpected error: {e.Message}");
}

IMPORTANT: Catch specific exceptions first, generic ones last!

EXCEPTION PROPERTIES

try {
    int x = int.Parse("abc");
}
catch (FormatException e) {
    Console.WriteLine($"Message: {e.Message}");
    Console.WriteLine($"Type: {e.GetType()}");
    Console.WriteLine($"Stack Trace: {e.StackTrace}");
}

TRY-FINALLY BLOCK

Finally runs no matter what - for cleanup code.

Syntax:

try {
    // Code that might throw exception
}
catch (Exception e) {
    // Handle exception
}
finally {
    // Always runs (cleanup code)
}

File Cleanup Example:

StreamReader reader = null;

try {
    reader = new StreamReader("file.txt");
    string content = reader.ReadToEnd();
    Console.WriteLine(content);
}
catch (FileNotFoundException e) {
    Console.WriteLine("File not found!");
}
finally {
    reader?.Close();  // Always runs - cleanup resources
    Console.WriteLine("File closed.");
}

TRY-CATCH-FINALLY ALL TOGETHER

try {
    Console.WriteLine("Attempting operation...");
    int result = 10 / int.Parse(Console.ReadLine());
    Console.WriteLine($"Result: {result}");
}
catch (DivideByZeroException e) {
    Console.WriteLine("Cannot divide by zero!");
}
catch (FormatException e) {
    Console.WriteLine("Invalid input!");
}
finally {
    Console.WriteLine("Operation complete.");
}

THROW CUSTOM EXCEPTIONS

Create your own exception errors.

Syntax:

throw new ExceptionType("Error message");

Age Validation:

void ValidateAge(int age) {
    if (age < 0 || age > 120) {
        throw new ArgumentException("Age must be between 0 and 120");
    }
    Console.WriteLine($"Valid age: {age}");
}

try {
    ValidateAge(25);   // OK
    ValidateAge(150);  // Throws exception
}
catch (ArgumentException e) {
    Console.WriteLine($"Error: {e.Message}");
}

// Output:
// Valid age: 25
// Error: Age must be between 0 and 120

Insufficient Balance:

void WithdrawMoney(decimal balance, decimal amount) {
    if (amount > balance) {
        throw new InvalidOperationException("Insufficient balance!");
    }
    Console.WriteLine($"Withdrew: ${amount}");
    Console.WriteLine($"Remaining: ${balance - amount}");
}

try {
    WithdrawMoney(100, 50);   // OK
    WithdrawMoney(100, 200);  // Throws exception
}
catch (InvalidOperationException e) {
    Console.WriteLine($"Transaction failed: {e.Message}");
}

// Output:
// Withdrew: $50
// Remaining: $50
// Transaction failed: Insufficient balance!

COMMON EXCEPTIONS

NullReferenceException - Accessing property of null object
IndexOutOfRangeException - Accessing array index that doesn't exist
DivideByZeroException - Dividing by zero
FormatException - Invalid conversion (parse "abc" as int)
ArgumentException - Invalid argument passed to method
FileNotFoundException - File doesn't exist
InvalidOperationException - Operation not valid in current state
OverflowException - Result too large for data type

PRACTICAL EXAMPLES

Safe Integer Parsing:

int SafeParseInt(string input) {
    try {
        return int.Parse(input);
    }
    catch (FormatException) {
        Console.WriteLine($"'{input}' is not a valid number");
        return 0;  // Default value
    }
}

int num1 = SafeParseInt("42");      // 42
int num2 = SafeParseInt("abc");     // Error message, returns 0

ATM Withdrawal:

class BankAccount {
    public decimal Balance { get; private set; } = 1000;
    
    public void Withdraw(decimal amount) {
        if (amount <= 0) {
            throw new ArgumentException("Amount must be positive");
        }
        
        if (amount > Balance) {
            throw new InvalidOperationException("Insufficient funds");
        }
        
        Balance -= amount;
        Console.WriteLine($"Withdrew: ${amount}. Balance: ${Balance}");
    }
}

try {
    BankAccount account = new BankAccount();
    account.Withdraw(100);   // OK
    account.Withdraw(2000);  // Throws exception
}
catch (InvalidOperationException e) {
    Console.WriteLine($"Error: {e.Message}");
}
catch (ArgumentException e) {
    Console.WriteLine($"Error: {e.Message}");
}

// Output:
// Withdrew: $100. Balance: $900
// Error: Insufficient funds

Student Grade Validator:

void AssignGrade(int score) {
    try {
        if (score < 0 || score > 100) {
            throw new ArgumentException("Score must be 0-100");
        }
        
        string grade = score >= 90 ? "A" : 
                      score >= 80 ? "B" : 
                      score >= 70 ? "C" : 
                      score >= 60 ? "D" : "F";
        
        Console.WriteLine($"Score: {score}, Grade: {grade}");
    }
    catch (ArgumentException e) {
        Console.WriteLine($"Invalid score: {e.Message}");
    }
}

AssignGrade(85);   // Score: 85, Grade: B
AssignGrade(150);  // Invalid score: Score must be 0-100

Login Validation:

void Login(string username, string password) {
    try {
        if (string.IsNullOrEmpty(username)) {
            throw new ArgumentException("Username cannot be empty");
        }
        
        if (password.Length < 8) {
            throw new ArgumentException("Password must be at least 8 characters");
        }
        
        Console.WriteLine($"Login successful for {username}");
    }
    catch (ArgumentException e) {
        Console.WriteLine($"Login failed: {e.Message}");
    }
}

Login("ali", "pass");       // Login failed: Password must be at least 8 characters
Login("", "password123");   // Login failed: Username cannot be empty
Login("ali", "password123"); // Login successful for ali

BEST PRACTICES

DO:
- Catch specific exceptions (not just Exception)
- Use finally for cleanup
- Throw meaningful exceptions with clear messages
- Log exceptions for debugging
- Validate input early

DONT:
- Silently ignore exceptions
- Catch Exception as first catch (use as last resort)
- Use exceptions for normal program flow
- Throw generic Exception without message
- Catch and do nothing

INTERVIEW QUESTIONS

Q: Difference between try-catch and try-finally?
A: catch handles exceptions. finally always runs for cleanup.

Q: When throw custom exception?
A: When validating business logic (age, balance, etc.).

Q: What happens if no exception in try block?
A: catch is skipped, finally still runs.

Q: Can catch multiple exception types?
A: Yes, use multiple catch blocks. Specific first, generic last.

Q: Difference between IndexOutOfRangeException and NullReferenceException?
A: Index = accessing invalid array index. Null = accessing property of null object.

Q: What's the purpose of finally block?
A: Always runs for cleanup - closing files, releasing resources.

Q: When use try-catch vs return early?
A: try-catch for exceptional cases. Return early for normal validation.

Q: How to avoid NullReferenceException?
A: Check for null before using: if (obj != null) or use null-coalescing operator ??.

KEY TAKEAWAYS

- try = code that might error
- catch = handle specific errors
- finally = cleanup code (always runs)
- throw = create custom errors
- Always catch specific exceptions first
- Never silently ignore exceptions
- Use meaningful error messages
- Validate inputs early in methods
- Log exceptions for debugging
- Exception handling prevents crashes
- Errors are not failures - they're opportunities to recover
