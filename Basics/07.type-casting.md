# 🔄 Type Casting in C#

## 🧠 What is Type Casting?

**Type casting** is converting one data type to another. There are two types:
- **Implicit** = Automatic, safe conversion (smaller → larger)
- **Explicit** = Manual, requires cast operator (larger → smaller, risky)

---

## 📌 Implicit Casting

Automatic conversion from smaller to larger type. No data loss.

```csharp
int intValue = 42;
double doubleValue = intValue;  // Automatic, safe

Console.WriteLine(doubleValue);  // 42

// Implicit casting order
byte → short → int → long → float → double
```

**Examples:**

```csharp
int num = 100;
long bigNum = num;              // int → long ✅

int age = 25;
double salary = age;            // int → double ✅

char letter = 'A';
int ascii = letter;             // char → int ✅
```

---

## 🔄 Explicit Casting

Manual conversion using cast operator `()`. May cause data loss.

```csharp
double doubleValue = 42.7;
int intValue = (int)doubleValue;  // Explicit cast required

Console.WriteLine(intValue);  // 42 (decimal part lost)
```

**Examples:**

```csharp
double price = 99.99;
int roundedPrice = (int)price;      // 99 (loses decimal)

long bigNumber = 1000000;
int smallNumber = (int)bigNumber;   // OK if within range

long veryBig = 5000000000;
int overflow = (int)veryBig;        // Data loss! Unpredictable result
```

---

## 🛡️ Safe Casting with `checked`

Check for overflow errors.

```csharp
long veryBig = 5000000000;

try
{
    int result = checked((int)veryBig);  // Throws exception if overflow
}
catch (OverflowException ex)
{
    Console.WriteLine("Value too large to convert");
}
```

## 🔢 String Casting

Converting between strings and numbers.

### String to Number

```csharp
// Method 1: Direct cast (risky, throws exception)
string numStr = "42";
int num = int.Parse(numStr);

// Method 2: TryParse (safe, returns bool)
string input = "42";
if (int.TryParse(input, out int result))
{
    Console.WriteLine(result);  // 42
}
else
{
    Console.WriteLine("Invalid number");
}

// Method 3: Convert class
string value = "3.14";
double doubleValue = Convert.ToDouble(value);
```

### Number to String

```csharp
int number = 42;
string str = number.ToString();

double price = 19.99;
string priceStr = price.ToString();

// Format while converting
int age = 25;
string formatted = age.ToString("D3");  // "025"
```

## 🔗 Reference Type Casting

Casting between base and derived classes.

```csharp
public class Animal { }
public class Dog : Animal { }

// Upcast (derived → base) - safe, implicit
Dog dog = new Dog();
Animal animal = dog;  // Implicit ✅

// Downcast (base → derived) - risky, explicit
Animal baseAnimal = new Dog();
Dog derivedDog = (Dog)baseAnimal;  // Explicit ✅

// Invalid downcast
Animal cat = new Cat();
Dog wrongCast = (Dog)cat;  // Runtime error ❌
```

---

## 🔍 Safe Casting with `as` and `is`

### Using `is` (check type)

```csharp
Animal animal = new Dog();

if (animal is Dog)
{
    Console.WriteLine("It's a Dog");
}

if (animal is Cat)
{
    Console.WriteLine("It's a Cat");
}
```

### Using `as` (safe cast)

```csharp
Animal animal = new Dog();

Dog dog = animal as Dog;
if (dog != null)
{
    dog.Bark();  // Safe to use
}

Cat cat = animal as Cat;
if (cat == null)
{
    Console.WriteLine("Not a Cat");
}
```

## 📋 Summary Table

| Type | Syntax | Safe? | Use When |
|---|---|---|---|
| **Implicit** | `type variable = source;` | ✅ Yes | Small → Large |
| **Explicit** | `type variable = (type)source;` | ⚠️ Check | Large → Small |
| **TryParse** | `type.TryParse(str, out var)` | ✅ Yes | String to number |
| **Convert** | `Convert.ToType(source)` | ✅ Yes | Flexible conversion |
| **as** | `variable as Type` | ✅ Yes | Reference types |
| **is** | `variable is Type` | ✅ Yes | Type checking |

## 🎤 Interview Q&A

### Q1: What is type casting?
**A:** Converting one data type to another.

```csharp
int num = 42;
double doubleNum = num;  // Implicit
int backToInt = (int)doubleNum;  // Explicit
```

---

### Q2: What is implicit casting?
**A:** Automatic conversion from smaller to larger type. No data loss.

```csharp
int value = 100;
long bigValue = value;  // Implicit ✅
```

---

### Q3: What is explicit casting?
**A:** Manual conversion using `()`. May lose data.

```csharp
double price = 19.99;
int rounded = (int)price;  // Explicit, loses decimal
```

---

### Q4: When should you use TryParse instead of Parse?
**A:** TryParse is safer. Parse throws exception if conversion fails.

```csharp
// TryParse - safe
if (int.TryParse("123", out int num))
{
    // Success
}

// Parse - throws exception
int num2 = int.Parse("abc");  // Throws FormatException
```

---

### Q5: What happens when you cast a large number to smaller type?
**A:** Data loss or unpredictable result (overflow).

```csharp
long big = 10000000;
int small = (int)big;  // May overflow

// Use checked to catch error
try
{
    int result = checked((int)big);
}
catch (OverflowException)
{
    Console.WriteLine("Overflow!");
}
```

---

### Q6: What is the difference between Parse and Convert?
**A:**
- **Parse** = Specific type, throws exception on failure
- **Convert** = Multiple types, returns default on failure

```csharp
int.Parse("123");              // Returns 123
int.Parse("abc");              // Throws exception

Convert.ToInt32("123");        // Returns 123
Convert.ToInt32("abc");        // Returns 0
```

---

### Q7: What is the `as` operator?
**A:** Safe casting for reference types. Returns null if cast fails.

```csharp
Animal animal = new Dog();

Dog dog = animal as Dog;
if (dog != null)  // Check before using
{
    dog.Bark();
}
```

---

### Q8: What is the `is` operator?
**A:** Checks if variable is a specific type.

```csharp
Animal animal = new Dog();

if (animal is Dog)
{
    Console.WriteLine("It's a Dog");
}
```

---

### Q9: Can you cast reference types without errors?
**A:** Use `as` or `is` to check first.

```csharp
// ❌ Risky
Dog dog = (Dog)animal;

// ✅ Safe
Dog dog = animal as Dog;
if (dog != null) { }
```

---

### Q10: How do you convert string to number safely?
**A:** Use `TryParse` or `Convert` with error handling.

```csharp
// TryParse
if (int.TryParse(input, out int num))
{
    // Use num
}

// Convert with try-catch
try
{
    double price = Convert.ToDouble(input);
}
catch (FormatException)
{
    Console.WriteLine("Invalid format");
}
```

---

## 🚀 Key Takeaways

✅ Implicit = Automatic, safe (small → large)  
✅ Explicit = Manual, risky (large → small)  
✅ Use TryParse for string to number  
✅ Use `as` for safe reference type casting  
✅ Use `is` to check types  
✅ Use `checked` to detect overflow  
✅ Always validate before casting
