# 🔷 Working with JSON in C#

---

## 🧠 What is JSON?

**JSON** (JavaScript Object Notation) is the universal language of data. APIs, config files, databases — they all speak JSON.

```json
{
  "name": "Asha",
  "age": 30,
  "isActive": true,
  "skills": ["C#", "SQL", "Azure"]
}
```

C# reads and writes JSON using **`System.Text.Json`** — built into .NET, fast, and no extra packages needed.

---

## 🔑 Two Key Operations

| Operation | What it does | Method |
|---|---|---|
| **Serialize** | C# object → JSON string | `JsonSerializer.Serialize()` |
| **Deserialize** | JSON string → C# object | `JsonSerializer.Deserialize<T>()` |

---

## ✅ Serialize — Object to JSON

```csharp
using System.Text.Json;

public record Person(string Name, int Age, bool IsActive);

var person = new Person("Asha", 30, true);

// Basic serialize
string json = JsonSerializer.Serialize(person);
Console.WriteLine(json);
// {"Name":"Asha","Age":30,"IsActive":true}

// Pretty printed
var options = new JsonSerializerOptions { WriteIndented = true };
string pretty = JsonSerializer.Serialize(person, options);
Console.WriteLine(pretty);
// {
//   "Name": "Asha",
//   "Age": 30,
//   "IsActive": true
// }
```

---

## ✅ Deserialize — JSON to Object

```csharp
string json = """{"Name":"Asha","Age":30,"IsActive":true}""";

Person? person = JsonSerializer.Deserialize<Person>(json);

Console.WriteLine(person?.Name);      // Asha
Console.WriteLine(person?.Age);       // 30
Console.WriteLine(person?.IsActive);  // True
```

---

## ⚙️ JsonSerializerOptions — Customize Behavior

```csharp
var options = new JsonSerializerOptions
{
    WriteIndented        = true,               // pretty print
    PropertyNamingPolicy = JsonNamingPolicy.CamelCase, // Name → name
    PropertyNameCaseInsensitive = true,        // ignore case when deserializing
    DefaultIgnoreCondition = JsonIgnoreCondition.WhenWritingNull, // skip nulls
};

var person = new Person("Asha", 30, true);
string json = JsonSerializer.Serialize(person, options);
// {
//   "name": "Asha",     ← camelCase!
//   "age": 30,
//   "isActive": true
// }
```

## 🏷️ Attributes — Control JSON Output

```csharp
using System.Text.Json.Serialization;

public class Product
{
    [JsonPropertyName("product_name")]   // custom JSON key name
    public string Name { get; set; }

    [JsonIgnore]                         // skip this property entirely
    public string InternalCode { get; set; }

    [JsonPropertyOrder(1)]               // control output order
    public decimal Price { get; set; }

    public string? Description { get; set; }
}

var p = new Product { Name = "Laptop", Price = 999.99m, InternalCode = "XYZ" };
Console.WriteLine(JsonSerializer.Serialize(p));
// {"Price":999.99,"product_name":"Laptop"}
// InternalCode is ignored ✅
```

## 📁 Read & Write JSON Files

```csharp
// ✅ Save object to JSON file
async Task SaveAsync<T>(string path, T data)
{
    var options = new JsonSerializerOptions { WriteIndented = true };
    string json = JsonSerializer.Serialize(data, options);
    await File.WriteAllTextAsync(path, json);
}

// ✅ Load object from JSON file
async Task<T?> LoadAsync<T>(string path)
{
    if (!File.Exists(path)) return default;
    string json = await File.ReadAllTextAsync(path);
    return JsonSerializer.Deserialize<T>(json);
}

// Usage
var user = new Person("Asha", 30, true);
await SaveAsync("user.json", user);

Person? loaded = await LoadAsync<Person>("user.json");
Console.WriteLine(loaded?.Name);   // Asha
```

## 📋 JSON Collections

```csharp
// Serialize a list
var people = new List<Person>
{
    new("Asha",  30, true),
    new("Ravi",  25, true),
    new("Priya", 28, false),
};

string json = JsonSerializer.Serialize(people, new JsonSerializerOptions { WriteIndented = true });
// [
//   { "Name": "Asha",  "Age": 30, "IsActive": true  },
//   { "Name": "Ravi",  "Age": 25, "IsActive": true  },
//   { "Name": "Priya", "Age": 28, "IsActive": false }
// ]

// Deserialize back
List<Person>? loaded = JsonSerializer.Deserialize<List<Person>>(json);
Console.WriteLine(loaded?.Count);   // 3
```

## 🌍 Real-World: Calling a REST API

```csharp
public record WeatherResponse(string City, double Temperature, string Condition);

public class WeatherService
{
    private static readonly HttpClient _client = new();

    public async Task<WeatherResponse?> GetWeatherAsync(string city)
    {
        string url = $"https://api.example.com/weather/{city}";
        string json = await _client.GetStringAsync(url);

        var options = new JsonSerializerOptions
        {
            PropertyNameCaseInsensitive = true
        };

        return JsonSerializer.Deserialize<WeatherResponse>(json, options);
    }
}

// Usage
var service = new WeatherService();
var weather = await service.GetWeatherAsync("Chennai");
Console.WriteLine($"{weather?.City}: {weather?.Temperature}°C, {weather?.Condition}");
```

## 🚨 Common Mistakes

```csharp
// ❌ Mistake 1: Case mismatch — JSON has "name" but class has "Name"
// Fix: use PropertyNameCaseInsensitive = true in options

// ❌ Mistake 2: Forgetting ? — Deserialize can return null
Person person = JsonSerializer.Deserialize<Person>(json);   // ⚠️ nullable warning
Person? person = JsonSerializer.Deserialize<Person>(json);  // ✅

// ❌ Mistake 3: Not handling malformed JSON
try
{
    var result = JsonSerializer.Deserialize<Person>(badJson);
}
catch (JsonException ex)
{
    Console.WriteLine($"Invalid JSON: {ex.Message}");
}

// ❌ Mistake 4: Serializing with default options in APIs
// Always use camelCase for APIs — JavaScript expects it
var options = new JsonSerializerOptions { PropertyNamingPolicy = JsonNamingPolicy.CamelCase };
```

## 📋 Summary Table

| Task | Code |
|---|---|
| Object → JSON string | `JsonSerializer.Serialize(obj)` |
| JSON string → Object | `JsonSerializer.Deserialize<T>(json)` |
| Pretty print | `WriteIndented = true` |
| camelCase keys | `PropertyNamingPolicy = CamelCase` |
| Ignore case | `PropertyNameCaseInsensitive = true` |
| Skip nulls | `DefaultIgnoreCondition = WhenWritingNull` |
| Custom key name | `[JsonPropertyName("key")]` |
| Skip a property | `[JsonIgnore]` |
| Save to file | `File.WriteAllTextAsync` + Serialize |
| Load from file | `File.ReadAllTextAsync` + Deserialize |

## 🎤 Interview Q&A

**Q1: What is JSON serialization?**
A: Converting a C# object into a JSON string.
```csharp
string json = JsonSerializer.Serialize(myObject);
```

**Q2: What is JSON deserialization?**
A: Converting a JSON string back into a C# object.
```csharp
MyClass obj = JsonSerializer.Deserialize<MyClass>(json);
```

**Q3: What namespace handles JSON in .NET?**
A: `System.Text.Json` — built into .NET, no extra packages needed.

**Q4: How do you pretty-print JSON output?**
A: Set `WriteIndented = true` in `JsonSerializerOptions`.

**Q5: How do you map a JSON key to a different C# property name?**
A: Use `[JsonPropertyName("key_name")]` attribute.
```csharp
[JsonPropertyName("full_name")]
public string Name { get; set; }
```

**Q6: How do you skip a property during serialization?**
A: Use `[JsonIgnore]`.
```csharp
[JsonIgnore]
public string Password { get; set; }
```

**Q7: Why use `PropertyNameCaseInsensitive = true`?**
A: JSON APIs often return camelCase (`"name"`) but C# properties are PascalCase (`Name`). This option makes deserialization work regardless of casing.

**Q8: Can `Deserialize` return null?**
A: Yes — always use `T?` and null-check the result.

**Q9: What exception does invalid JSON throw?**
A: `JsonException` — always wrap deserialization in try/catch when input is untrusted.

**Q10: What's the difference between `System.Text.Json` and `Newtonsoft.Json`?**
A: `System.Text.Json` is built into .NET — fast and allocation-efficient. `Newtonsoft.Json` (Json.NET) is a NuGet package with more features and easier configuration. For new projects, prefer `System.Text.Json`.

## 🚀 Key Takeaways

✅ `JsonSerializer.Serialize` → C# object to JSON string  
✅ `JsonSerializer.Deserialize<T>` → JSON string to C# object  
✅ `System.Text.Json` is built-in — no NuGet needed  
✅ Use `JsonSerializerOptions` to customize behavior  
✅ `WriteIndented = true` for readable output  
✅ `CamelCase` naming policy for APIs  
✅ `[JsonPropertyName]` to rename, `[JsonIgnore]` to skip  
✅ Always handle `JsonException` for untrusted input  
✅ Deserialize returns nullable — always null-check
