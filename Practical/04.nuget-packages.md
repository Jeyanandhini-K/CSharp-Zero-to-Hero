# 📦 NuGet Packages in C#

---

## 🧠 What is NuGet?

**NuGet** is the package manager for .NET — like npm for JavaScript or pip for Python. It gives you access to **hundreds of thousands of free libraries** built by Microsoft and the community.

Instead of building everything from scratch, you grab a package:

```bash
# One command — and you have a world-class library
dotnet add package Newtonsoft.Json
```

> 💡 NuGet packages are hosted at **nuget.org** — over 300,000 packages available.

---

## 🔑 The Three Ways to Add Packages

### 1️⃣ .NET CLI (recommended ✅)

```bash
# Add a package
dotnet add package PackageName

# Add a specific version
dotnet add package PackageName --version 13.0.3

# Remove a package
dotnet remove package PackageName

# List installed packages
dotnet list package

# Restore all packages (after cloning a repo)
dotnet restore
```

### 2️⃣ Visual Studio Package Manager

Right-click project → **Manage NuGet Packages** → Search → Install

### 3️⃣ Edit `.csproj` Directly

```xml
<ItemGroup>
  <PackageReference Include="Newtonsoft.Json" Version="13.0.3" />
  <PackageReference Include="Serilog" Version="3.1.1" />
</ItemGroup>
```

Then run `dotnet restore`.

---

## 📋 What Happens When You Add a Package?

```bash
dotnet add package Serilog
```

1. NuGet downloads the package from nuget.org
2. Adds it to your `.csproj`
3. Saves it in your local cache (`~/.nuget/packages`)
4. You can `using` it immediately

> 💡 The `packages` folder is **never committed to Git** — `dotnet restore` recreates it from `.csproj` on any machine.

## 🔢 Understanding Version Numbers

NuGet uses **Semantic Versioning** (SemVer): `MAJOR.MINOR.PATCH`

```
13.0.3
│   │  └── PATCH — bug fixes only (safe to update)
│   └────── MINOR — new features, backwards compatible (usually safe)
└────────── MAJOR — breaking changes (read release notes first!)
```

```xml
<!-- In .csproj -->
<PackageReference Include="Newtonsoft.Json" Version="13.0.3" />   <!-- exact -->
<PackageReference Include="Newtonsoft.Json" Version="13.*" />      <!-- latest 13.x -->
```

> 💡 For production apps, pin the exact version. For personal projects, `*` is fine.

## 🔄 Updating Packages

```bash
# See what's outdated
dotnet list package --outdated

# Update a specific package
dotnet add package PackageName --version 14.0.0

# Update all packages (use a tool)
dotnet tool install --global dotnet-outdated-tool
dotnet outdated --upgrade
```

## 🌍 Real-World Example: Serilog + CsvHelper

```csharp
// Add packages first:
// dotnet add package Serilog Serilog.Sinks.Console CsvHelper

using Serilog;
using CsvHelper;
using System.Globalization;

// Setup Serilog
Log.Logger = new LoggerConfiguration()
    .WriteTo.Console()
    .CreateLogger();

// Read CSV with CsvHelper
public record Student(string Name, int Grade, double Score);

using var reader = new StreamReader("students.csv");
using var csv = new CsvReader(reader, CultureInfo.InvariantCulture);
var students = csv.GetRecords<Student>().ToList();

Log.Information("Loaded {Count} students", students.Count);

foreach (var student in students.Where(s => s.Score >= 90))
{
    Log.Information("Top student: {Name} — {Score}", student.Name, student.Score);
}
```

## 🔐 Private NuGet Feeds

Companies often host private packages (internal libraries). You add them via `NuGet.config`:

```xml
<?xml version="1.0" encoding="utf-8"?>
<configuration>
  <packageSources>
    <add key="nuget.org" value="https://api.nuget.org/v3/index.json" />
    <add key="MyCompany" value="https://pkgs.dev.azure.com/mycompany/_packaging/feed/nuget/v3/index.json" />
  </packageSources>
</configuration>
```

## 🎤 Interview Q&A

**Q1: What is NuGet?**
A: The package manager for .NET. It lets you add third-party libraries to your project with one command.

**Q2: How do you add a package via CLI?**
A: `dotnet add package PackageName`

**Q3: Where is package info stored?**
A: In your `.csproj` file as `<PackageReference>` elements.

**Q4: What does `dotnet restore` do?**
A: Downloads all packages listed in `.csproj`. Run this after cloning a repo.

**Q5: Should the packages folder be in Git?**
A: No — add it to `.gitignore`. `dotnet restore` recreates it.

**Q6: What is SemVer?**
A: Semantic Versioning — `MAJOR.MINOR.PATCH`. Major = breaking changes, Minor = new features, Patch = bug fixes.

**Q7: What is the difference between Newtonsoft.Json and System.Text.Json?**
A: `System.Text.Json` is built into .NET — fast, no install needed. `Newtonsoft.Json` is a NuGet package with more features and easier configuration. Both are widely used.

**Q8: Name 5 essential NuGet packages.**
A: Serilog (logging), Newtonsoft.Json (JSON), AutoMapper (object mapping), FluentValidation (validation), Moq (testing mocks).

**Q9: How do you see which packages are outdated?**
A: `dotnet list package --outdated`

**Q10: What is a global tool in NuGet?**
A: A CLI tool installed globally on your machine.
```bash
dotnet tool install --global dotnet-ef   # Entity Framework CLI
```
