# 🎁 Generics in C#

## 🧠 What are Generics?

**Generics** allow you to write **reusable code** that works with any data type. Instead of writing the same logic for int, string, double separately, write once with `<T>` (type parameter).

---

## 📌 Basic Generic Class

```csharp
// Generic class
public class Container<T>
{
    private T value;

    public void Set(T item)
    {
        value = item;
    }

    public T Get()
    {
        return value;
    }
}

// Use with different types
Container<int> intContainer = new Container<int>();
intContainer.Set(42);
Console.WriteLine(intContainer.Get());  // 42

Container<string> stringContainer = new Container<string>();
stringContainer.Set("Hello");
Console.WriteLine(stringContainer.Get());  // Hello

Container<double> doubleContainer = new Container<double>();
doubleContainer.Set(3.14);
Console.WriteLine(doubleContainer.Get());  // 3.14
```

## 🔄 Generic Method

```csharp
public class Utilities
{
    // Generic method
    public static void Print<T>(T value)
    {
        Console.WriteLine($"Value: {value}, Type: {typeof(T).Name}");
    }

    public static T GetDefault<T>()
    {
        return default(T);  // null for ref types, 0 for value types
    }
}

Utilities.Print<int>(42);           // Value: 42, Type: Int32
Utilities.Print<string>("Ali");     // Value: Ali, Type: String
Utilities.Print(3.14);              // Type inferred
```

## 🔗 Generic with Multiple Type Parameters

```csharp
public class Pair<T1, T2>
{
    public T1 First { get; set; }
    public T2 Second { get; set; }

    public void Display()
    {
        Console.WriteLine($"First: {First}, Second: {Second}");
    }
}

Pair<string, int> pair = new Pair<string, int>
{
    First = "Age",
    Second = 25
};
pair.Display();  // First: Age, Second: 25
```

## 🛡️ Generic Constraints

Restrict what types can be used with generic.

### Constraint: `where T : class`
```csharp
public class Repository<T> where T : class
{
    // T must be a reference type
    public void Store(T item) { }
}

Repository<string> repo1 = new Repository<string>();  // ✅ OK
// Repository<int> repo2 = new Repository<int>();    // ❌ Error
```

### Constraint: `where T : struct`
```csharp
public class ValueContainer<T> where T : struct
{
    // T must be a value type
    public void Store(T item) { }
}

ValueContainer<int> c1 = new ValueContainer<int>();      // ✅ OK
// ValueContainer<string> c2 = new ValueContainer<string>(); // ❌ Error
```

### Constraint: `where T : BaseClass`
```csharp
public class Animal { }
public class Dog : Animal { }

public class AnimalRepository<T> where T : Animal
{
    // T must be Animal or derived from Animal
}

AnimalRepository<Dog> repo = new AnimalRepository<Dog>();  // ✅ OK
// AnimalRepository<string> repo = new AnimalRepository<string>(); // ❌ Error
```

### Constraint: `where T : IInterface`
```csharp
public interface IComparable
{
    int CompareTo(object obj);
}

public class Sorter<T> where T : IComparable
{
    // T must implement IComparable
}
```

### Constraint: `where T : new()`
```csharp
public class Factory<T> where T : new()
{
    // T must have parameterless constructor
    public T Create()
    {
        return new T();
    }
}

public class MyClass
{
    public MyClass() { }  // Parameterless constructor
}

Factory<MyClass> factory = new Factory<MyClass>();
MyClass instance = factory.Create();  // ✅ OK
```

## 📊 Real-World Example: Generic Repository

```csharp
public class Repository<T> where T : class
{
    private List<T> items = new List<T>();

    public void Add(T item)
    {
        items.Add(item);
    }

    public T GetById(int index)
    {
        return items[index];
    }

    public List<T> GetAll()
    {
        return new List<T>(items);
    }

    public void Remove(T item)
    {
        items.Remove(item);
    }
}

public class User
{
    public int Id { get; set; }
    public string Name { get; set; }
}

public class Product
{
    public int Id { get; set; }
    public string Name { get; set; }
}

// Usage - same code for different types
Repository<User> userRepo = new Repository<User>();
userRepo.Add(new User { Id = 1, Name = "Ali" });
userRepo.Add(new User { Id = 2, Name = "Fatima" });

Repository<Product> productRepo = new Repository<Product>();
productRepo.Add(new Product { Id = 1, Name = "Laptop" });

Console.WriteLine(userRepo.GetAll().Count);      // 2
Console.WriteLine(productRepo.GetAll().Count);   // 1
```

## 🎯 Why Use Generics?

1. **Type Safety** - Catch errors at compile time
2. **Code Reuse** - Write once, use with many types
3. **Performance** - No boxing/unboxing
4. **Readability** - Clear intent

```csharp
// ❌ Without generics (old way)
ArrayList list = new ArrayList();
list.Add(5);
list.Add("text");           // Compiles, but wrong!
int num = (int)list[0];     // Unsafe cast
string text = (string)list[1];

// ✅ With generics (modern way)
List<int> intList = new List<int>();
intList.Add(5);
// intList.Add("text");     // Compile error - prevents bug!
int num2 = intList[0];      // Safe, no cast needed
```

## 🎤 Interview Q&A

### Q1: What are generics?
**A:** Allow writing reusable code for any data type using `<T>`.

```csharp
public class Box<T>
{
    public T item { get; set; }
}

Box<int> b1 = new Box<int>();
Box<string> b2 = new Box<string>();
```

---

### Q2: Why use generics?
**A:** Type safety, reusability, performance, no boxing/unboxing.

---

### Q3: What is a generic constraint?
**A:** Restricts what types can be used with generic.

```csharp
public class Repository<T> where T : class { }
```

---

### Q4: What does `where T : class` mean?
**A:** T must be a reference type.

```csharp
Repository<string> r1 = new();  // ✅
// Repository<int> r2 = new();  // ❌
```

---

### Q5: What does `where T : struct` mean?
**A:** T must be a value type.

```csharp
Container<int> c1 = new();      // ✅
// Container<string> c2 = new(); // ❌
```

---

### Q6: What does `where T : new()` mean?
**A:** T must have a parameterless constructor.

---

### Q7: Can you have multiple constraints?
**A:** Yes! Combine with `and`.

```csharp
public class MyClass<T> where T : class, new()
{
    // T must be reference type AND have no-arg constructor
}
```

---

### Q8: What is the difference between generic class and generic method?
**A:**
- **Class**: Type specified at instantiation
- **Method**: Type can be inferred from arguments

```csharp
Box<int> b = new Box<int>();  // Class generic
int max = Max<int>(5, 10);      // Method generic
int max2 = Max(5, 10);          // Type inferred
```

---

### Q9: Can you inherit generic class?
**A:** Yes!

```csharp
public class Box<T> { }
public class IntBox : Box<int> { }
```

---

### Q10: Why is List<T> better than ArrayList?
**A:** Type-safe, no boxing/unboxing, compile-time checks.

```csharp
List<int> list = new List<int>();
list.Add(5);           // ✅ OK
// list.Add("text");  // ❌ Compile error
```

## 🚀 Key Takeaways

✅ Generics = Reusable code for any type  
✅ `<T>` = Type parameter  
✅ Constraints limit what types can be used  
✅ Type-safe at compile time  
✅ Better performance than non-generic  
✅ Use generics instead of casting  
✅ .NET collections are generic (List<T>, Dictionary<K,V>)
