## ЁЯза What are Collections?

Collections store **groups of related data**. C# has many types тАФ each built for a different job. Picking the right one matters for performance and clarity.

```csharp
// You already know the basics
var list = new List<int> { 1, 2, 3 };

// But there's a whole world beyond List тЬЕ
Dictionary<string, int> scores = new();
HashSet<string> uniqueTags = new();
Queue<string> printQueue = new();
Stack<string> undoStack = new();
```

## ЁЯУЛ List\<T\> тАФ The Workhorse

```csharp
var fruits = new List<string> { "Apple", "Banana", "Mango" };

// Add / Remove
fruits.Add("Orange");
fruits.AddRange(new[] { "Grape", "Papaya" });
fruits.Remove("Banana");
fruits.RemoveAt(0);

// Access
Console.WriteLine(fruits[0]);          // first item
Console.WriteLine(fruits.Count);       // count
Console.WriteLine(fruits.Contains("Mango")); // True

// Search
int idx = fruits.IndexOf("Mango");    // position
var longNames = fruits.FindAll(f => f.Length > 5);

// Sort
fruits.Sort();
fruits.Sort((a, b) => b.CompareTo(a));   // descending

// Convert
string[] arr = fruits.ToArray();
```

## ЁЯЧВя╕П Dictionary\<TKey, TValue\> тАФ Key-Value Lookup

```csharp
var scores = new Dictionary<string, int>
{
    ["Asha"]  = 95,
    ["Ravi"]  = 88,
    ["Priya"] = 92,
};

// Add / Update
scores["Kiran"] = 78;
scores["Asha"]  = 97;   // update existing

// Safe read тАФ never throws
if (scores.TryGetValue("Ravi", out int rScore))
    Console.WriteLine($"Ravi: {rScore}");   // Ravi: 88

// Check existence
bool hasAsha = scores.ContainsKey("Asha");   // True

// Iterate
foreach (var (name, score) in scores)
    Console.WriteLine($"{name}: {score}");

// Keys and Values
List<string> names    = scores.Keys.ToList();
List<int>    allScores = scores.Values.ToList();

// Remove
scores.Remove("Kiran");
```

## ЁЯФ╡ HashSet\<T\> тАФ Unique Values Only

```csharp
var tags = new HashSet<string> { "csharp", "dotnet", "api" };

// Add тАФ duplicates silently ignored тЬЕ
tags.Add("csharp");    // already exists тАФ nothing happens
tags.Add("azure");     // new тАФ added

Console.WriteLine(tags.Count);          // 4
Console.WriteLine(tags.Contains("api")); // True

// Set operations
var set1 = new HashSet<int> { 1, 2, 3, 4 };
var set2 = new HashSet<int> { 3, 4, 5, 6 };

set1.IntersectWith(set2);   // { 3, 4 }    тАФ common items
set1.UnionWith(set2);       // { 1,2,3,4,5,6 } тАФ all items
set1.ExceptWith(set2);      // { 1, 2 }    тАФ in set1 but not set2

// Use HashSet when: you need unique values, or fast Contains checks
```

## ЁЯОл Queue\<T\> тАФ First In, First Out (FIFO)

```csharp
var printQueue = new Queue<string>();

// Enqueue тАФ add to back
printQueue.Enqueue("Document1.pdf");
printQueue.Enqueue("Document2.pdf");
printQueue.Enqueue("Document3.pdf");

// Dequeue тАФ take from front
string next = printQueue.Dequeue();   // "Document1.pdf"
Console.WriteLine(next);

// Peek тАФ look without removing
string upcoming = printQueue.Peek();  // "Document2.pdf"

Console.WriteLine(printQueue.Count);  // 2

// Use Queue for: print jobs, task processing, BFS algorithms
```

## ЁЯУЪ Stack\<T\> тАФ Last In, First Out (LIFO)

```csharp
var undoStack = new Stack<string>();

// Push тАФ add to top
undoStack.Push("Type 'Hello'");
undoStack.Push("Bold text");
undoStack.Push("Change color");

// Pop тАФ take from top (last added = first out)
string lastAction = undoStack.Pop();   // "Change color"
Console.WriteLine($"Undoing: {lastAction}");

// Peek тАФ look without removing
Console.WriteLine(undoStack.Peek());   // "Bold text"

// Use Stack for: undo/redo, browser history, expression parsing
```

## ЁЯФЧ LinkedList\<T\> тАФ Fast Insert/Delete

```csharp
var playlist = new LinkedList<string>();

playlist.AddLast("Song A");
playlist.AddLast("Song B");
playlist.AddLast("Song C");
playlist.AddFirst("Intro");   // add to beginning

// Insert in middle
var nodeB = playlist.Find("Song B");
playlist.AddAfter(nodeB!, "Song B2");

// Remove
playlist.Remove("Song A");

// Iterate
foreach (string song in playlist)
    Console.WriteLine(song);

// Use LinkedList when: frequent insert/delete in middle (faster than List)
```

## ЁЯПОя╕П Performance Comparison

| Collection | Add | Remove | Search | Best for |
|---|---|---|---|---|
| `List<T>` | O(1) | O(n) | O(n) | General purpose |
| `Dictionary` | O(1) | O(1) | O(1) | Key-value lookup |
| `HashSet<T>` | O(1) | O(1) | O(1) | Unique values |
| `Queue<T>` | O(1) | O(1) | O(n) | FIFO processing |
| `Stack<T>` | O(1) | O(1) | O(n) | LIFO / undo |
| `LinkedList` | O(1) | O(1) | O(n) | Frequent middle insert |

## ЁЯМН Real-World Example

```csharp
public class StudentTracker
{
    // Fast name lookup
    private Dictionary<int, string> _students = new();

    // Track unique subjects
    private HashSet<string> _subjects = new();

    // Pending registrations
    private Queue<string> _waitlist = new();

    public void AddStudent(int id, string name) => _students[id] = name;
    public void AddSubject(string subject) => _subjects.Add(subject);
    public void AddToWaitlist(string name) => _waitlist.Enqueue(name);

    public string? ProcessNext()
        => _waitlist.Count > 0 ? _waitlist.Dequeue() : null;

    public void PrintAll()
    {
        Console.WriteLine("=== Students ===");
        foreach (var (id, name) in _students)
            Console.WriteLine($"  {id}: {name}");

        Console.WriteLine("=== Subjects ===");
        foreach (var subject in _subjects)
            Console.WriteLine($"  {subject}");
    }
}
```

## ЁЯОд Interview Q&A

**Q1: What is the difference between `List<T>` and `Dictionary<TKey, TValue>`?**
A: `List<T>` stores items by index тАФ search is O(n). `Dictionary` stores by key тАФ lookup is O(1).
```csharp
var list = new List<string> { "Asha", "Ravi" };
var dict = new Dictionary<int, string> { [1] = "Asha", [2] = "Ravi" };
Console.WriteLine(dict[1]);   // O(1) тАФ instant тЬЕ
```

**Q2: When would you use `HashSet` over `List`?**
A: When you need unique values or fast O(1) `Contains` checks. `List.Contains` is O(n); `HashSet.Contains` is O(1).

**Q3: What is the difference between `Queue` and `Stack`?**
A: Queue = FIFO (first in, first out). Stack = LIFO (last in, first out).
```csharp
Queue<int> q = new(); q.Enqueue(1); q.Enqueue(2); q.Dequeue(); // 1
Stack<int> s = new(); s.Push(1);    s.Push(2);    s.Pop();     // 2
```

**Q4: How do you safely get a value from a Dictionary?**
A: Use `TryGetValue` тАФ no exception if key missing.
```csharp
if (dict.TryGetValue("key", out int val))
    Console.WriteLine(val);
```

**Q5: What is the difference between `Array` and `List<T>`?**
A: Array = fixed size. `List<T>` = dynamic, grows as needed. Use array when size is known and fixed.

## ЁЯЪА Key Takeaways

тЬЕ `List<T>` тАФ general purpose, ordered, allows duplicates  
тЬЕ `Dictionary<K,V>` тАФ key-value, O(1) lookup  
тЬЕ `HashSet<T>` тАФ unique values only, O(1) contains  
тЬЕ `Queue<T>` тАФ FIFO, first in first out  
тЬЕ `Stack<T>` тАФ LIFO, last in first out  
тЬЕ `TryGetValue` тАФ safe dictionary access, no exceptions  
тЬЕ Pick the right collection тАФ it affects performance significantly
