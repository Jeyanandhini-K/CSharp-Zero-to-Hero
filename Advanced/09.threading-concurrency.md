# 🧵 Threading & Concurrency in C#

---

## 🧠 What is a Thread?

A **thread** is the smallest unit of execution. Your program starts with one thread — the **main thread**. Threading lets you run multiple things at the same time.

```csharp
// Single thread — one thing at a time
Console.WriteLine("Step 1");
Console.WriteLine("Step 2");   // waits for Step 1
Console.WriteLine("Step 3");   // waits for Step 2

// Multi-thread — things run simultaneously
Thread t1 = new Thread(() => Console.WriteLine("Running on Thread 1"));
Thread t2 = new Thread(() => Console.WriteLine("Running on Thread 2"));
t1.Start();   // runs in parallel
t2.Start();   // runs in parallel
```

> 💡 Think of threads like workers. One worker does one thing at a time. More workers = more things happening at once. 👷👷👷

---

## 🔑 Three Ways to Run Code Concurrently

```csharp
// 1️⃣ Thread — low level, full control
Thread t = new Thread(() => DoWork());
t.Start();

// 2️⃣ ThreadPool — reuses threads, more efficient
ThreadPool.QueueUserWorkItem(_ => DoWork());

// 3️⃣ Task (recommended ✅) — modern, works with async/await
Task.Run(() => DoWork());
```

> ✅ **Always prefer `Task.Run`** in modern C#. It uses the ThreadPool automatically and works with `async/await`.

---

## ⚡ Task.Run — The Modern Way

```csharp
// Run heavy work on a background thread
await Task.Run(() =>
{
    // CPU-heavy work — doesn't block the main thread
    for (int i = 0; i < 1_000_000; i++) { }
    Console.WriteLine("Heavy work done!");
});

Console.WriteLine("Main thread is free while that runs!");
```

> 💡 Use `Task.Run` for **CPU-heavy** work (calculations, image processing).
> Use `await` directly for **I/O work** (files, network, database) — no `Task.Run` needed.

## 🚨 Race Conditions — The Classic Bug

When two threads read/write the same variable at the same time, you get unpredictable results:

```csharp
int counter = 0;

// ❌ Race condition — both threads modify counter at same time
Thread t1 = new Thread(() => { for (int i = 0; i < 1000; i++) counter++; });
Thread t2 = new Thread(() => { for (int i = 0; i < 1000; i++) counter++; });

t1.Start(); t2.Start();
t1.Join();  t2.Join();

Console.WriteLine(counter);   // ❌ NOT 2000 — could be anything!
```

---

## 🔒 Lock — The Fix for Race Conditions

`lock` ensures only **one thread at a time** can run a block of code:

```csharp
int counter = 0;
object lockObj = new object();   // the "key" to the lock

Thread t1 = new Thread(() =>
{
    for (int i = 0; i < 1000; i++)
        lock (lockObj) { counter++; }   // only 1 thread enters at a time
});

Thread t2 = new Thread(() =>
{
    for (int i = 0; i < 1000; i++)
        lock (lockObj) { counter++; }
});

t1.Start(); t2.Start();
t1.Join();  t2.Join();

Console.WriteLine(counter);   // ✅ Always 2000
```

---

## ⚛️ Interlocked — Atomic Operations

For simple counter operations, `Interlocked` is faster than `lock`:

```csharp
int counter = 0;

Thread t1 = new Thread(() =>
{
    for (int i = 0; i < 1000; i++)
        Interlocked.Increment(ref counter);   // ✅ thread-safe, no lock needed
});

Thread t2 = new Thread(() =>
{
    for (int i = 0; i < 1000; i++)
        Interlocked.Increment(ref counter);
});

t1.Start(); t2.Start();
t1.Join();  t2.Join();

Console.WriteLine(counter);   // ✅ Always 2000
```

## 🧵 Thread.Join — Wait for a Thread to Finish

```csharp
Thread worker = new Thread(() =>
{
    Console.WriteLine("Worker started...");
    Thread.Sleep(2000);
    Console.WriteLine("Worker done!");
});

worker.Start();
worker.Join();   // main thread waits here until worker finishes

Console.WriteLine("Main thread continues");
```

## 🏊 Thread-Safe Collections

Regular collections like `List<T>` are NOT thread-safe. Use these instead:

```csharp
using System.Collections.Concurrent;

// ✅ Thread-safe versions
ConcurrentQueue<int>      queue   = new();
ConcurrentStack<int>      stack   = new();
ConcurrentDictionary<string, int> dict = new();
ConcurrentBag<int>        bag     = new();   // unordered, fast

// ConcurrentDictionary example
var scores = new ConcurrentDictionary<string, int>();
scores["Asha"] = 100;
scores.TryAdd("Ravi", 90);
scores.AddOrUpdate("Asha", 100, (key, old) => old + 10);   // 110
```

## 🔐 Mutex vs Monitor vs SemaphoreSlim

```csharp
// SemaphoreSlim — limit how many threads run at once
SemaphoreSlim sem = new SemaphoreSlim(3);   // max 3 threads at a time

async Task ProcessAsync(int id)
{
    await sem.WaitAsync();   // wait for a slot
    try
    {
        Console.WriteLine($"Thread {id} working...");
        await Task.Delay(1000);
    }
    finally
    {
        sem.Release();   // always release, even on error
    }
}

// Useful for: rate limiting, connection pools, throttling requests
```

## 🌍 Real-World Example: Parallel Data Processing

```csharp
public class DataProcessor
{
    private readonly object _lock = new();
    private readonly List<string> _results = new();

    public async Task ProcessAllAsync(string[] items)
    {
        // Process all items in parallel
        await Parallel.ForEachAsync(items,
            new ParallelOptions { MaxDegreeOfParallelism = 4 },
            async (item, _) =>
            {
                string result = await ProcessItemAsync(item);

                lock (_lock)   // protect the shared list
                {
                    _results.Add(result);
                }
            });

        Console.WriteLine($"Processed {_results.Count} items");
    }

    private async Task<string> ProcessItemAsync(string item)
    {
        await Task.Delay(100);   // simulate work
        return $"Processed: {item}";
    }
}
```

## ⚠️ Deadlock — The Danger Zone

A deadlock happens when two threads each wait for the other to release a lock — forever:

```csharp
object lock1 = new(), lock2 = new();

// Thread 1: locks lock1, then tries lock2
Thread t1 = new Thread(() =>
{
    lock (lock1) { Thread.Sleep(100); lock (lock2) { } }
});

// Thread 2: locks lock2, then tries lock1
Thread t2 = new Thread(() =>
{
    lock (lock2) { Thread.Sleep(100); lock (lock1) { } }  // ❌ DEADLOCK!
});
```

> 🛡️ **Avoid deadlocks:** always acquire locks in the **same order**, use `async/await` instead of blocking calls, keep lock sections **short**.

## 📋 Summary Table

| Concept | Tool | Use when |
|---|---|---|
| Run background work | `Task.Run()` | CPU-heavy operations |
| Prevent race conditions | `lock` | Multiple threads share data |
| Fast atomic ops | `Interlocked` | Simple counter/flag updates |
| Wait for thread | `Thread.Join()` | Need thread to finish |
| Limit concurrency | `SemaphoreSlim` | Rate limiting, throttling |
| Thread-safe list | `ConcurrentBag<T>` | Multi-thread collections |
| Thread-safe dict | `ConcurrentDictionary` | Multi-thread lookups |
| Parallel loops | `Parallel.ForEach` | Process collections fast |

## 🎤 Interview Q&A

**Q1: What is a thread?**
A: The smallest unit of execution. A program starts with one main thread; you can create more to run code simultaneously.

**Q2: What is a race condition?**
A: When two threads access shared data at the same time and produce unpredictable results.
```csharp
// Both threads read/write counter at same time — result is random
counter++;
```

**Q3: How do you fix a race condition?**
A: Use `lock` to ensure only one thread accesses the data at a time.
```csharp
lock (lockObj) { counter++; }
```

**Q4: What is `Task.Run`?**
A: Runs code on a ThreadPool thread. Best for CPU-heavy work.
```csharp
await Task.Run(() => HeavyCalculation());
```

**Q5: What is the difference between `Task.Run` and `await`?**
A: `await` is for I/O-bound work (network, files). `Task.Run` is for CPU-bound work (calculations). Don't wrap I/O in `Task.Run`.

**Q6: What is a deadlock?**
A: Two threads each waiting for the other's lock — both frozen forever. Avoid by always locking in the same order.

**Q7: What is `Interlocked`?**
A: Provides atomic (thread-safe) operations without a lock. Faster for simple increments/decrements.
```csharp
Interlocked.Increment(ref counter);
```

**Q8: What is `SemaphoreSlim`?**
A: Limits how many threads can run a section of code simultaneously. Great for throttling.
```csharp
SemaphoreSlim sem = new SemaphoreSlim(3);   // max 3 at once
```

**Q9: What thread-safe collections exist in C#?**
A: `ConcurrentDictionary`, `ConcurrentQueue`, `ConcurrentStack`, `ConcurrentBag` — all in `System.Collections.Concurrent`.

**Q10: Should you use `Thread` directly in modern C#?**
A: Rarely. Prefer `Task.Run` and `async/await` — they use the ThreadPool efficiently and are easier to manage.

## 🚀 Key Takeaways

✅ Threads let your program do multiple things at once  
✅ Use `Task.Run` for CPU-heavy work — not raw `Thread`  
✅ Race conditions happen when threads share data unsafely  
✅ `lock` prevents race conditions — keep lock sections short  
✅ `Interlocked` is faster than `lock` for simple operations  
✅ Use `ConcurrentDictionary` / `ConcurrentBag` for thread-safe collections  
✅ `SemaphoreSlim` limits how many threads run at once  
✅ Deadlocks = two threads waiting on each other — always lock in the same order  
✅ Prefer `async/await` over threads for I/O work
