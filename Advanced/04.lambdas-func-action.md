# 🎯 Lambdas, Func, Action & Predicate in C#

---

## 🧠 What is a Lambda?

A **lambda** is an anonymous (no-name) function you write inline. Instead of declaring a full method, you write the logic right where you need it.

```csharp
// Full method
int Add(int a, int b) { return a + b; }

// Same thing as a lambda ✅
var add = (int a, int b) => a + b;

Console.WriteLine(add(3, 4));   // 7
```

> 💡 `=>` is the lambda operator — read it as **"goes to"**. `(a, b) => a + b` means "a and b go to a + b".

---

## 🔑 Lambda Syntax

```csharp
// No params
() => Console.WriteLine("Hello!")

// One param (parens optional)
x => x * x
(x) => x * x

// Multiple params
(a, b) => a + b

// Multiple lines — use braces
(a, b) =>
{
    int result = a + b;
    Console.WriteLine(result);
    return result;
}
```

## 📦 Func\<T\> — Returns a Value

`Func` holds a method that **returns something**.

```csharp
// Func<input, output>
Func<int, int>     square  = x => x * x;
Func<int, int, int> add    = (a, b) => a + b;
Func<string, bool> isLong  = s => s.Length > 5;
Func<string, string> shout = s => s.ToUpper() + "!";

Console.WriteLine(square(5));          // 25
Console.WriteLine(add(3, 4));          // 7
Console.WriteLine(isLong("Hi"));       // False
Console.WriteLine(shout("hello"));     // HELLO!

// Func<T1, T2, ..., TResult>
// Last type parameter is ALWAYS the return type
Func<int, int, int, int> sum3 = (a, b, c) => a + b + c;
Console.WriteLine(sum3(1, 2, 3));      // 6
```

## 📢 Action\<T\> — Returns Nothing (void)

`Action` holds a method that **does something but returns nothing**.

```csharp
Action greet              = () => Console.WriteLine("Hello!");
Action<string> greetName  = name => Console.WriteLine($"Hello, {name}!");
Action<string, int> info  = (name, age) => Console.WriteLine($"{name} is {age}");

greet();                  // Hello!
greetName("Asha");        // Hello, Asha!
info("Ravi", 25);         // Ravi is 25

// Common use — callbacks and event handlers
void ExecuteWithLog(Action action, string label)
{
    Console.WriteLine($"Starting {label}...");
    action();
    Console.WriteLine($"Done {label}!");
}

ExecuteWithLog(() => Console.WriteLine("Working..."), "Task");
// Starting Task...
// Working...
// Done Task!
```

## ✅ Predicate\<T\> — Returns bool

`Predicate<T>` is shorthand for `Func<T, bool>` — tests a condition.

```csharp
Predicate<int>    isEven    = n => n % 2 == 0;
Predicate<string> isLong    = s => s.Length > 5;
Predicate<int>    isPositive = n => n > 0;

Console.WriteLine(isEven(4));          // True
Console.WriteLine(isLong("Hello"));    // False
Console.WriteLine(isPositive(-1));     // False

// Used in List methods
var numbers = new List<int> { 1, 2, 3, 4, 5, 6 };
var evens   = numbers.FindAll(isEven);   // [2, 4, 6]
numbers.RemoveAll(n => n < 3);           // removes 1, 2
```

## 🔗 Lambdas + LINQ — The Dream Team

Lambdas are the engine behind LINQ:

```csharp
var students = new List<Student>
{
    new("Asha",  95),
    new("Ravi",  72),
    new("Priya", 88),
    new("Kiran", 65),
};

// Every LINQ method takes a lambda ✅
var topStudents = students
    .Where(s => s.Score >= 80)          // Func<Student, bool>
    .OrderByDescending(s => s.Score)    // Func<Student, int>
    .Select(s => $"{s.Name}: {s.Score}") // Func<Student, string>
    .ToList();

topStudents.ForEach(Console.WriteLine);
// Asha: 95
// Priya: 88

record Student(string Name, int Score);
```

## 🎭 Storing & Passing Lambdas

```csharp
// Store in a variable
Func<int, bool> isEven = n => n % 2 == 0;

// Pass as argument
void Filter(List<int> list, Func<int, bool> predicate)
{
    foreach (int n in list)
        if (predicate(n)) Console.Write(n + " ");
}

Filter(new List<int> { 1,2,3,4,5,6 }, n => n % 2 == 0);  // 2 4 6
Filter(new List<int> { 1,2,3,4,5,6 }, n => n > 3);        // 4 5 6

// Return a lambda from a method
Func<int, int> Multiplier(int factor) => n => n * factor;

var triple = Multiplier(3);
Console.WriteLine(triple(5));   // 15
Console.WriteLine(triple(10));  // 30
```

## 🌍 Real-World Example

```csharp
public class OrderProcessor
{
    private readonly List<Order> _orders;

    public OrderProcessor(List<Order> orders) => _orders = orders;

    // Accept a filter as Func — caller decides the condition
    public List<Order> GetOrders(Func<Order, bool> filter)
        => _orders.Where(filter).ToList();

    // Accept a transform as Func
    public List<T> Transform<T>(Func<Order, T> selector)
        => _orders.Select(selector).ToList();

    // Accept an action to run on each
    public void ForEach(Action<Order> action)
        => _orders.ForEach(action);
}

var processor = new OrderProcessor(orders);

// Caller provides the logic — no changes to OrderProcessor ✅
var bigOrders    = processor.GetOrders(o => o.Total > 1000);
var emails       = processor.Transform(o => o.CustomerEmail);
processor.ForEach(o => Console.WriteLine($"Order #{o.Id}: ₹{o.Total}"));
```

## 🎤 Interview Q&A

**Q1: What is a lambda expression?**
A: An anonymous inline function using `=>` syntax.
```csharp
Func<int, int> square = x => x * x;
Console.WriteLine(square(5));   // 25
```

**Q2: What is the difference between `Func` and `Action`?**
A: `Func` returns a value. `Action` returns nothing (void).
```csharp
Func<int, int>   double_ = x => x * 2;   // returns int
Action<string>   print   = s => Console.WriteLine(s); // returns void
```

**Q3: What is `Predicate<T>`?**
A: Shorthand for `Func<T, bool>` — tests a condition and returns true or false.
```csharp
Predicate<int> isEven = n => n % 2 == 0;
```

**Q4: How do lambdas relate to LINQ?**
A: Every LINQ method (`.Where`, `.Select`, `.OrderBy`) takes a lambda as its argument. Lambdas ARE the query logic in LINQ.

**Q5: What is a closure?**
A: A lambda that captures variables from its surrounding scope.
```csharp
int min = 5;
Func<int, bool> aboveMin = n => n > min;   // 'min' captured from outer scope
Console.WriteLine(aboveMin(10));   // True
min = 15;
Console.WriteLine(aboveMin(10));   // False — uses updated 'min'!
```

**Q6: What does `=>` mean?**
A: The lambda operator — "goes to". Left side = parameters, right side = body.

**Q7: Can lambdas have multiple lines?**
A: Yes — use curly braces and explicit `return`.
```csharp
Func<int, int> process = x =>
{
    int doubled = x * 2;
    return doubled + 1;
};
```

## 🚀 Key Takeaways

✅ Lambda = anonymous inline function using `=>`  
✅ `Func<T, TResult>` — takes input, returns output  
✅ `Action<T>` — takes input, returns nothing  
✅ `Predicate<T>` — takes input, returns bool  
✅ Pass lambdas as arguments — makes methods flexible  
✅ LINQ is built entirely on `Func` + lambdas  
✅ Closures capture outer variables — be careful with mutations
