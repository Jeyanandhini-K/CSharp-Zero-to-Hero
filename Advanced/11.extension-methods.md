# 🔌 Extension Methods in C#

---

## 🧠 What are Extension Methods?

Extension methods let you **add new methods to an existing type** — without modifying the original class, without inheritance, without access to the source code.

```csharp
// Imagine string had no IsNullOrEmpty helper...
string name = "";

// ❌ Old way — verbose
if (string.IsNullOrEmpty(name)) { }

// ✅ Extension method — reads naturally
if (name.IsNullOrEmpty()) { }
```

> 💡 You're not changing the `string` class. You're just teaching C# a new trick for calling your method on it. Magic? Almost. 🪄

---

## 🔑 How to Write One

Three rules:
1. Must be in a **`static` class**
2. Method must be **`static`**
3. First parameter has the **`this`** keyword — that's the type you're extending

```csharp
// ✅ Extension method — all three rules followed
public static class StringExtensions
{
    public static bool IsNullOrEmpty(this string value)
    {
        return string.IsNullOrEmpty(value);
    }

    public static string Capitalize(this string value)
    {
        if (string.IsNullOrEmpty(value)) return value;
        return char.ToUpper(value[0]) + value[1..];
    }
}

// Using them — looks like built-in string methods!
string name = "asha";
Console.WriteLine(name.IsNullOrEmpty());   // False
Console.WriteLine(name.Capitalize());      // Asha
```

## 🔗 Chaining Extensions

Extensions chain naturally — just like LINQ:

```csharp
public static class StringExtensions
{
    public static string Capitalize(this string s)
        => string.IsNullOrEmpty(s) ? s : char.ToUpper(s[0]) + s[1..];

    public static string Truncate(this string s, int maxLength)
        => s.Length <= maxLength ? s : s[..maxLength] + "...";

    public static string RemoveSpaces(this string s)
        => s.Replace(" ", "");
}

string result = "  hello world  "
    .Trim()             // built-in
    .Capitalize()       // extension
    .Truncate(8);       // extension

Console.WriteLine(result);   // Hello wo...
```

## 📦 Extending Any Type

You can extend classes, structs, interfaces — anything:

```csharp
// Extend int
public static class IntExtensions
{
    public static bool IsEven(this int n) => n % 2 == 0;
    public static bool IsPositive(this int n) => n > 0;
    public static int Square(this int n) => n * n;
}

int num = 4;
Console.WriteLine(num.IsEven());     // True
Console.WriteLine(num.Square());     // 16

// Extend IEnumerable<T>
public static class CollectionExtensions
{
    public static bool IsEmpty<T>(this IEnumerable<T> source)
        => !source.Any();

    public static void ForEach<T>(this IEnumerable<T> source, Action<T> action)
    {
        foreach (var item in source) action(item);
    }
}

var list = new List<int> { 1, 2, 3 };
Console.WriteLine(list.IsEmpty());          // False
list.ForEach(x => Console.WriteLine(x));    // 1, 2, 3
```

## 🌍 Real-World Examples

### ✅ DateTime helpers

```csharp
public static class DateExtensions
{
    public static bool IsWeekend(this DateTime date)
        => date.DayOfWeek is DayOfWeek.Saturday or DayOfWeek.Sunday;

    public static string ToFriendly(this DateTime date)
    {
        var diff = DateTime.Now - date;
        return diff.TotalMinutes < 1   ? "just now"
             : diff.TotalHours   < 1   ? $"{(int)diff.TotalMinutes}m ago"
             : diff.TotalDays    < 1   ? $"{(int)diff.TotalHours}h ago"
             : date.ToString("MMM dd, yyyy");
    }
}

var posted = DateTime.Now.AddHours(-2);
Console.WriteLine(posted.ToFriendly());     // 2h ago
Console.WriteLine(DateTime.Now.IsWeekend()); // True/False
```

### ✅ Validation helpers

```csharp
public static class ValidationExtensions
{
    public static bool IsValidEmail(this string email)
        => email.Contains('@') && email.Contains('.');

    public static bool IsValidAge(this int age)
        => age is >= 0 and <= 150;

    public static bool HasMinLength(this string s, int min)
        => s?.Length >= min;
}

string email = "asha@example.com";
Console.WriteLine(email.IsValidEmail());        // True
Console.WriteLine(25.IsValidAge());             // True
Console.WriteLine("hello".HasMinLength(3));     // True
```

### ✅ Null safety helpers

```csharp
public static class NullExtensions
{
    public static bool IsNull<T>(this T obj) where T : class
        => obj is null;

    public static T OrDefault<T>(this T? obj, T defaultValue) where T : class
        => obj ?? defaultValue;
}

string? name = null;
Console.WriteLine(name.IsNull());               // True
Console.WriteLine(name.OrDefault("Anonymous")); // Anonymous
```

## ⚠️ Rules & Limitations

```csharp
// ❌ Can't access private members of the class you're extending
public static string GetSecret(this MyClass obj)
{
    return obj._privateField;   // ❌ compile error!
}

// ❌ Extension methods don't override existing methods
// If string already has .Contains(), your extension won't replace it

// ✅ They DO show up in IntelliSense — just like built-in methods
// ✅ They work on interfaces — extending IEnumerable is very common
// ✅ They can be generic
public static T FirstOrDefault<T>(this IEnumerable<T> source, T fallback)
    => source.Any() ? source.First() : fallback;
```

## 🧩 Where You Already Use Extension Methods

LINQ is entirely built on extension methods! Every `.Where()`, `.Select()`, `.OrderBy()` is an extension on `IEnumerable<T>`:

```csharp
var names = new List<string> { "Asha", "Ravi", "Priya" };

// These are ALL extension methods from System.Linq!
var result = names
    .Where(n => n.Length > 3)      // extension
    .OrderBy(n => n)               // extension
    .Select(n => n.ToUpper())      // extension
    .ToList();                     // extension

Console.WriteLine(string.Join(", ", result));   // ASHA, PRIYA, RAVI
```

| Rule | Detail |
|---|---|
| Class must be | `static` |

| First parameter | `this TypeName paramName` |
| Access level | Only `public` members of the type |
| Can extend | Classes, structs, interfaces, enums |

## 🎤 Interview Q&A

**Q1: What is an extension method?**
A: A static method that adds functionality to an existing type without modifying it or using inheritance.

**Q2: What are the three requirements for an extension method?**
A: (1) Must be in a `static` class. (2) Must be a `static` method. (3) First parameter must use the `this` keyword.
```csharp
public static class MyExtensions
{
    public static bool IsEven(this int n) => n % 2 == 0;
}
```

**Q3: Can you extend a class you don't have the source code for?**
A: Yes! That's the whole point. You can extend `string`, `int`, `DateTime`, even third-party library types.

**Q4: Can extension methods access private members?**
A: No. They can only access `public` (and `internal` in the same assembly) members.

**Q5: Do extension methods override existing methods?**
A: No. If the class already has a method with the same signature, the class's own method wins.

**Q6: What is LINQ and how does it relate to extension methods?**
A: LINQ (Language Integrated Query) is entirely built on extension methods on `IEnumerable<T>`. Every `.Where()`, `.Select()`, `.OrderBy()` is an extension method.

**Q7: Can you add extension methods to an interface?**
A: Yes — and it's very useful! Adding extensions to `IEnumerable<T>` applies to `List<T>`, arrays, and anything else that implements the interface.

**Q8: Where do you put extension methods?**
A: In a `static` class, usually in a file named after what you're extending — e.g. `StringExtensions.cs`, `DateExtensions.cs`.

**Q9: How does the compiler find extension methods?**
A: By the `using` directive for the namespace where the static class lives. No `using` = extension method is invisible.

**Q10: Can extension methods be generic?**
A: Yes!
```csharp
public static bool IsNull<T>(this T obj) where T : class => obj is null;
```

## 🚀 Key Takeaways

✅ Extension methods add functionality to types you don't own  
✅ Three rules: `static` class, `static` method, `this` on first param  
✅ Call them like regular methods — IntelliSense shows them too  
✅ Can't access private members or override existing methods  
✅ Work on classes, structs, interfaces, enums  
✅ LINQ is 100% built on extension methods  
✅ Chain them for clean, readable fluent APIs  
✅ Add `using YourNamespace;` or they won't be visible
