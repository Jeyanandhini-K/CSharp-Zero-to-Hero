# ⚡ Async & Await in C#

---

## 🧠 What is Async Programming?

By default, C# runs code **line by line** — each line waits for the previous one to finish. That's **synchronous**.

**Asynchronous** code says: *"Start this task, and while it's working — go do something else. Come back when it's done."*

```csharp
// ❌ Synchronous — freezes everything while waiting
string data = DownloadData();       // UI freezes here for 3 seconds!
Console.WriteLine(data);

// ✅ Asynchronous — keeps moving while waiting
string data = await DownloadDataAsync();   // frees the thread while waiting
Console.WriteLine(data);
```

> 💡 Think of it like ordering food at a restaurant. You don't stand frozen at the counter waiting — you sit down and do other things. The waiter comes back when it's ready. 🍕

---

## 🔑 The Three Keywords

| Keyword | What it does |
|---|---|
| `async` | Marks a method as asynchronous |
| `await` | Pauses *this* method until the task finishes — without blocking the thread |
| `Task` | Represents an operation that will complete in the future |

```csharp
// async goes on the METHOD
// await goes INSIDE the method
public async Task<string> GetDataAsync()
{
    string result = await FetchFromServerAsync();   // wait here, don't block
    return result;
}
```

---

## 📦 Task — The Return Type

| Return type | Meaning |
|---|---|
| `Task` | Async method that returns nothing (like `void`) |
| `Task<T>` | Async method that returns a value of type `T` |
| `ValueTask<T>` | Like `Task<T>` but lighter — use when result is often already ready |

```csharp
// Returns nothing
public async Task SaveFileAsync(string content) { ... }

// Returns a string
public async Task<string> ReadFileAsync(string path) { ... }

// Returns an int
public async Task<int> CountRecordsAsync() { ... }
```

## ✅ Your First Async Method

```csharp
using System.Net.Http;

public async Task<string> GetWebPageAsync(string url)
{
    HttpClient client = new HttpClient();
    string html = await client.GetStringAsync(url);   // wait for download
    return html;
}

// Calling it
string page = await GetWebPageAsync("https://example.com");
Console.WriteLine(page.Length);
```

---

## 🔄 How It Actually Works (Step by Step)

```
1. GetWebPageAsync() starts
2. Hits await → "pause here, release the thread"
3. Thread goes and does other work (handles other requests, updates UI...)
4. Download finishes → method resumes right where it left off
5. Returns the result
```

> ⚠️ `await` does NOT create a new thread. It **releases** the current thread while waiting, then picks up again when the result is ready.

## ⛓️ Async Chains — await All the Way Up

When you `await` something, the caller also needs to `await` you:

```csharp
// Bottom — does the real work
public async Task<string> ReadFileAsync(string path)
{
    return await File.ReadAllTextAsync(path);
}

// Middle — awaits the bottom
public async Task<int> CountLinesAsync(string path)
{
    string content = await ReadFileAsync(path);
    return content.Split('\n').Length;
}

// Top — awaits the middle
public async Task PrintLineCountAsync(string path)
{
    int count = await CountLinesAsync(path);
    Console.WriteLine($"Lines: {count}");
}
```

> 💡 Async is like a chain — once you go async, everything calling it should be async too. Don't break the chain!

---

## 🚀 Running Multiple Tasks at Once

Don't `await` one by one if the tasks don't depend on each other — run them **in parallel**!

```csharp
// ❌ Slow — waits 3s then 3s = 6 seconds total
string page1 = await GetPageAsync("url1");   // wait 3s
string page2 = await GetPageAsync("url2");   // wait 3s

// ✅ Fast — both run at the same time = ~3 seconds total
Task<string> task1 = GetPageAsync("url1");   // start — don't await yet
Task<string> task2 = GetPageAsync("url2");   // start — don't await yet

string[] results = await Task.WhenAll(task1, task2);   // wait for BOTH
Console.WriteLine(results[0]);
Console.WriteLine(results[1]);
```

---

## 🕐 Task.Delay — Async Waiting

`Task.Delay` is the async version of `Thread.Sleep` — waits without blocking:

```csharp
// ❌ Blocks the thread completely
Thread.Sleep(2000);

// ✅ Releases the thread while waiting
await Task.Delay(2000);
Console.WriteLine("2 seconds later!");
```

## 🌍 Real-World Example: Fetch & Process Data

```csharp
public class WeatherService
{
    private readonly HttpClient _client = new();

    public async Task<string> GetWeatherAsync(string city)
    {
        Console.WriteLine($"Fetching weather for {city}...");

        string url = $"https://api.weather.com/{city}";
        string response = await _client.GetStringAsync(url);

        Console.WriteLine("Got the data!");
        return response;
    }

    public async Task ShowMultipleCitiesAsync()
    {
        // Fetch all cities at the same time
        Task<string> chennai  = GetWeatherAsync("Chennai");
        Task<string> mumbai   = GetWeatherAsync("Mumbai");
        Task<string> delhi    = GetWeatherAsync("Delhi");

        string[] results = await Task.WhenAll(chennai, mumbai, delhi);

        Console.WriteLine($"Chennai: {results[0]}");
        Console.WriteLine($"Mumbai:  {results[1]}");
        Console.WriteLine($"Delhi:   {results[2]}");
    }
}
```

## 🚨 Common Mistakes

### ❌ Mistake 1: async void (don't do it!)
```csharp
// ❌ Bad — exceptions can't be caught, can't be awaited
public async void LoadData() { ... }

// ✅ Good — always use Task
public async Task LoadData() { ... }
```
> Exception: event handlers (like button clicks in UI) are the only valid use of `async void`.

### ❌ Mistake 2: Blocking with .Result or .Wait()
```csharp
// ❌ Deadlock danger! Blocks the thread
string data = GetDataAsync().Result;
GetDataAsync().Wait();

// ✅ Always await properly
string data = await GetDataAsync();
```

### ❌ Mistake 3: Awaiting one by one when you could parallelize
```csharp
// ❌ Slow — sequential
var a = await TaskA();
var b = await TaskB();

// ✅ Fast — parallel
var (a, b) = await (TaskA(), TaskB());   // or use Task.WhenAll
```

---

## 🔥 async Main — Entry Point

Since C# 7.1, `Main` can be async:

```csharp
// ✅ Modern entry point
static async Task Main(string[] args)
{
    await DoSomethingAsync();
    Console.WriteLine("Done!");
}
```

## 🎤 Interview Q&A

**Q1: What is async/await in C#?**
A: A way to write asynchronous code that looks synchronous. `async` marks the method, `await` pauses it without blocking the thread.

**Q2: What does `await` actually do?**
A: It pauses the current method and **releases the thread** until the task finishes — then resumes where it left off.
```csharp
string data = await GetDataAsync();   // pause here, don't block
```

**Q3: What is a `Task`?**
A: Represents an operation that will complete in the future. `Task` = no return value. `Task<T>` = returns T.

**Q4: What's the difference between `Task` and `Task<T>`?**
A: `Task` = async void equivalent (safe). `Task<T>` = async method that returns a value of type T.

**Q5: Why should you avoid `async void`?**
A: Exceptions can't be caught, and the method can't be awaited. Only acceptable for event handlers.
```csharp
// ❌ async void — dangerous
// ✅ async Task — always prefer this
```

**Q6: What is `Task.WhenAll`?**
A: Runs multiple tasks in parallel and waits for ALL of them to finish.
```csharp
await Task.WhenAll(task1, task2, task3);
```

**Q7: What is `Task.WhenAny`?**
A: Waits for the **first** task to finish (useful for timeouts or racing).
```csharp
Task finished = await Task.WhenAny(task1, task2);
```

**Q8: Why is `.Result` or `.Wait()` dangerous?**
A: It blocks the thread synchronously and can cause **deadlocks**, especially in UI or ASP.NET apps.
```csharp
// ❌ Deadlock risk
string data = GetDataAsync().Result;
// ✅ Always await
string data = await GetDataAsync();
```

**Q9: What is the naming convention for async methods?**
A: Always end with `Async` — e.g. `GetDataAsync()`, `ReadFileAsync()`, `SaveAsync()`.

**Q10: Does `await` create a new thread?**
A: No! It **releases** the current thread while waiting. No new thread is created.

**Q11: What is `Task.Delay` vs `Thread.Sleep`?**
A: `Thread.Sleep` blocks the thread. `Task.Delay` releases it — always use `Task.Delay` in async code.
```csharp
await Task.Delay(1000);   // ✅ async-friendly
Thread.Sleep(1000);       // ❌ blocks
```

**Q12: Can `Main` be async?**
A: Yes, since C# 7.1.
```csharp
static async Task Main(string[] args) { await DoWorkAsync(); }
```

## 🚀 Key Takeaways

✅ `async` marks a method as asynchronous  
✅ `await` pauses the method without blocking the thread  
✅ Always return `Task` or `Task<T>` — never `async void`  
✅ Use `Task.WhenAll` to run tasks in parallel  
✅ Use `Task.WhenAny` to wait for the first to finish  
✅ Never use `.Result` or `.Wait()` — causes deadlocks  
✅ `await Task.Delay()` not `Thread.Sleep()` in async code  
✅ Name async methods with the `Async` suffix  
✅ `await` does NOT create a new thread — it releases one
