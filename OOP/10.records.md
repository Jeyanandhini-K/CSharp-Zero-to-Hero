# ЁЯУЭ Records in C#

---

## ЁЯза What is a Record?

A `record` is a special type built for **data**. One line replaces a constructor, properties, `ToString`, and equality тАФ all generated for you automatically.

```csharp
// тЭМ Old way тАФ a whole class just to hold data
public class Person
{
    public string Name { get; init; }
    public int Age { get; init; }
    public Person(string name, int age) { Name = name; Age = age; }
    public override string ToString() => $"Person {{ Name = {Name}, Age = {Age} }}";
    // + Equals, GetHashCode...
}

// тЬЕ New way тАФ one line does ALL of the above
public record Person(string Name, int Age);
```

---

## тЪб What You Get for Free

```csharp
var p = new Person("Asha", 30);

Console.WriteLine(p);           // Person { Name = Asha, Age = 30 }   тЖР ToString тЬЕ
var (name, age) = p;            // Deconstruction тЬЕ
Console.WriteLine(name);        // Asha
```

All of this тАФ **zero extra code**.

---

## ЁЯОп The Headline Feature: Value Equality

Two records are equal when their **data** matches тАФ not their memory address:

```csharp
var a = new Person("Asha", 30);
var b = new Person("Asha", 30);

Console.WriteLine(a == b);      // тЬЕ True  тАФ same data!

// Compare with a CLASS:
// a == b would be FALSE тАФ two different objects in memory
```

> ЁЯТб Records compare by **value**. Classes compare by **reference**. This is the biggest practical difference.

---

## ЁЯФД `with` Expressions тАФ Safe Copying

Records are **immutable** (can't change after creation). To "update" one, make a modified copy with `with`:

```csharp
var p = new Person("Asha", 30);
var older = p with { Age = 31 };

Console.WriteLine(p.Age);       // 30  тЖР original untouched тЬЕ
Console.WriteLine(older.Age);   // 31  тЖР new copy тЬЕ
```

> This is called **non-destructive mutation** тАФ you never break the original.

## ЁЯПЧя╕П Records Can Have Behavior Too

```csharp
public record Circle(double Radius)
{
    public double Area => Math.PI * Radius * Radius;

    public string Describe() => $"Circle with radius {Radius}, area {Area:F2}";
}

var c = new Circle(5);
Console.WriteLine(c.Area);        // 78.54
Console.WriteLine(c.Describe());  // Circle with radius 5, area 78.54
```

## ЁЯЖЪ record vs record struct

```csharp
// record (class) тАФ reference type, value equality
public record Person(string Name, int Age);

// record struct тАФ value type, value equality
public record struct Coordinate(int X, int Y);
```

| | `record` | `record struct` |
|---|---|---|
| Type | Reference (heap) | Value (stack) |
| Assignment | Shares reference | Copies |
| Equality | By value | By value |
| Immutable | тЬЕ by default | тЬЕ by default |

> ЁЯТб Use plain `record` for most data. Use `record struct` when you specifically want copy-on-assign behavior.

---

## тЪая╕П One Subtle Point

A `record` is still a **reference type** тАФ `b = a` shares the same object, just like any class. It doesn't cause problems because records are **immutable** тАФ nobody can change the shared data under you, and `with` always makes a fresh copy.

---

## ЁЯЖЪ Record vs Class vs Struct

| | `class` | `struct` | `record` |
|---|---|---|---|
| Type | Reference | Value | Reference |
| Equality | Reference | Value | **Value** тЬЕ |
| Immutable | тЭМ manual | тЭМ manual | тЬЕ default |
| Inheritance | тЬЕ | тЭМ | тЬЕ |
| Best for | Behavior + state | Tiny values | **Pure data** тЬЕ |

## ЁЯМН Real-World Example

```csharp
public record Product(string Name, decimal Price, string Category);

var laptop = new Product("Dell XPS", 999.99m, "Electronics");
var discounted = laptop with { Price = 799.99m };

Console.WriteLine(laptop);      // Product { Name = Dell XPS, Price = 999.99, Category = Electronics }
Console.WriteLine(discounted);  // Product { Name = Dell XPS, Price = 799.99, Category = Electronics }
Console.WriteLine(laptop == discounted);  // False тАФ different Price
```

## ЁЯОд Interview Q&A

**Q1: What is a record in C#?**
A: A concise, immutable reference type for data. Auto-generates constructor, properties, ToString, and equality.
```csharp
public record Person(string Name, int Age);
```

**Q2: What makes records special vs classes?**
A: **Value-based equality** тАФ two records with the same data are equal.
```csharp
var a = new Person("Asha", 30);
var b = new Person("Asha", 30);
Console.WriteLine(a == b);  // True тЬЕ (class would be False)
```

**Q3: What is a `with` expression?**
A: Creates a copy of a record with some properties changed. The original is untouched.
```csharp
var older = p with { Age = 31 };
```

**Q4: Are records mutable?**
A: By default, no тАФ properties are `init`-only. You can make mutable records but it's not recommended.

**Q5: What is the difference between `record` and `record struct`?**
A: `record` = reference type (heap). `record struct` = value type (stack, copied on assign). Both have value equality.

**Q6: Can records inherit?**
A: Yes! Records can inherit from other records.
```csharp
public record Animal(string Name);
public record Dog(string Name, string Breed) : Animal(Name);
```

**Q7: When should you use a record?**
A: For immutable data тАФ API models, DTOs, config, anything you compare by value.

**Q8: What does a record's `ToString()` look like?**
A: Automatically prints all property values.
```csharp
var p = new Person("Asha", 30);
Console.WriteLine(p);  // Person { Name = Asha, Age = 30 }
```

## ЁЯЪА Key Takeaways

тЬЕ `record` = concise data type тАФ one line does everything  
тЬЕ Auto-generates constructor, properties, `ToString`, equality  
тЬЕ **Value equality** тАФ equal data = equal records  
тЬЕ **Immutable** by default тАФ use `with` to make modified copies  
тЬЕ `with` = non-destructive mutation тАФ original stays safe  
тЬЕ `record struct` = value type + value equality  
тЬЕ Use records for **data**, classes for **behavior + identity**
