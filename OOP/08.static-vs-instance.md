# ⚙️ Static vs Instance Members in C#

## 🧠 What is Static vs Instance?

**Instance members** = Each object has its own copy  
**Static members** = Shared across all objects of the class

---

## 📌 Instance Members (Normal)

Each object has its own copy of the data.

```csharp
public class Car
{
    public string Brand { get; set; }
    public string Color { get; set; }

    public void Drive()
    {
        Console.WriteLine($"{Brand} {Color} is driving.");
    }
}

Car car1 = new Car { Brand = "Toyota", Color = "Red" };
Car car2 = new Car { Brand = "BMW", Color = "Blue" };

car1.Drive();  // Toyota Red is driving.
car2.Drive();  // BMW Blue is driving.

// Each object has its own Brand and Color
```

---

## 🔗 Static Members (Shared)

All objects share the same data.

```csharp
public class Counter
{
    public static int TotalCreated { get; set; }  // Shared
    public string Name { get; set; }               // Per object

    public Counter(string name)
    {
        Name = name;
        TotalCreated++;  // Increment shared counter
    }
}

Counter c1 = new Counter("First");
Counter c2 = new Counter("Second");
Counter c3 = new Counter("Third");

Console.WriteLine(Counter.TotalCreated);  // 3 (shared value)
Console.WriteLine(c1.Name);               // First
Console.WriteLine(c2.Name);               // Second
```

**Note:** Access static members using **ClassName.Member**, not object reference.

---

## 🛠️ Static Methods

Methods that don't need an object to call.

```csharp
public class MathHelper
{
    // Static method - no object needed
    public static int Add(int a, int b)
    {
        return a + b;
    }

    // Instance method - needs object
    public void PrintMessage(string msg)
    {
        Console.WriteLine(msg);
    }
}

// Call static method - no object needed
int result = MathHelper.Add(5, 10);  // 15

// Call instance method - needs object
MathHelper helper = new MathHelper();
helper.PrintMessage("Hello");
```

## 🏗️ Real-World Example: Database Connection

```csharp
public class Database
{
    // Static - shared connection string
    private static string connectionString = "Server=localhost;Database=MyDB";
    
    // Static - tracks total connections
    public static int TotalConnections { get; private set; }

    // Instance - each user has their own session
    public string SessionId { get; private set; }
    public DateTime ConnectedAt { get; private set; }

    public Database()
    {
        SessionId = Guid.NewGuid().ToString();
        ConnectedAt = DateTime.Now;
        TotalConnections++;  // Increment shared counter
    }

    // Static method
    public static void SetConnectionString(string newConnection)
    {
        connectionString = newConnection;
    }

    // Instance method
    public void ExecuteQuery(string query)
    {
        Console.WriteLine($"[{SessionId}] Executing: {query}");
    }
}

// Usage
Database.SetConnectionString("Server=newserver;Database=NewDB");

Database db1 = new Database();
Database db2 = new Database();
Database db3 = new Database();

db1.ExecuteQuery("SELECT * FROM Users");

Console.WriteLine(Database.TotalConnections);  // 3 (shared across all)
Console.WriteLine(db1.SessionId);              // Unique per object
```

## 📊 Static Constructor

Runs once when the class is first used, not for each object.

```csharp
public class Settings
{
    public static string AppName { get; private set; }
    public static string Version { get; private set; }

    // Static constructor - runs once
    static Settings()
    {
        AppName = "MyApp";
        Version = "1.0.0";
        Console.WriteLine("Settings initialized once!");
    }

    public Settings()
    {
        // Instance constructor - runs for each object
        Console.WriteLine("Object created");
    }
}

Settings s1 = new Settings();  // "Settings initialized once!" then "Object created"
Settings s2 = new Settings();  // "Object created" only (static constructor already ran)
```

## 🎯 Common Use Cases

### Case 1: Utility Methods

```csharp
public class StringUtils
{
    // Static method - doesn't need state
    public static string ReverseString(string input)
    {
        return new string(input.Reverse().ToArray());
    }

    public static bool IsValidEmail(string email)
    {
        return email.Contains("@");
    }
}

// Use without object
Console.WriteLine(StringUtils.ReverseString("Hello"));  // olleH
Console.WriteLine(StringUtils.IsValidEmail("a@b.com")); // True
```

### Case 2: Singleton Pattern

```csharp
public class Logger
{
    private static Logger instance;

    private Logger()
    {
        // Private constructor - cannot create new instances
    }

    // Static method to get the single instance
    public static Logger GetInstance()
    {
        if (instance == null)
            instance = new Logger();
        return instance;
    }

    public void Log(string message)
    {
        Console.WriteLine($"[LOG] {message}");
    }
}

// Usage - always the same instance
Logger logger1 = Logger.GetInstance();
Logger logger2 = Logger.GetInstance();
// logger1 and logger2 are the same object!
```

### Case 3: Factory Method

```csharp
public class UserFactory
{
    // Static factory method
    public static User CreateAdmin(string name)
    {
        return new User { Name = name, Role = "Admin" };
    }

    public static User CreateGuest(string name)
    {
        return new User { Name = name, Role = "Guest" };
    }
}

public class User
{
    public string Name { get; set; }
    public string Role { get; set; }
}

// Usage
User admin = UserFactory.CreateAdmin("Ali");
User guest = UserFactory.CreateGuest("Guest");
```

## 📋 Comparison Table

| Feature | Instance | Static |
|---|---|---|
| **Accessed via** | Object | Class name |
| **Shared across objects** | ❌ No | ✅ Yes |
| **Created when** | Each new object | First use |
| **Memory** | Each object gets copy | Shared memory |
| **Example** | `car.Color` | `Counter.Total` |

## ⚠️ When NOT to Use Static

```csharp
// ❌ Bad - using static for data that varies per object
public class Person
{
    public static string Name;  // WRONG! Shared by all people
}

// ✅ Good - instance member for per-object data
public class Person
{
    public string Name { get; set; }  // Each person has their own
}
```

## 🎤 Interview Q&A

### Q1: What is the difference between static and instance members?
**A:**
- **Static** = Shared by all objects, access via class name
- **Instance** = Each object has its own, access via object

```csharp
public class Counter
{
    public static int Total { get; set; }  // Static - shared
    public int Id { get; set; }            // Instance - per object
}

Counter.Total = 10;      // Static
new Counter().Id = 1;    // Instance
```

---

### Q2: When should you use static members?
**A:** When data/behavior is shared across all objects:
- Counters
- Utility methods
- Configuration settings
- Singleton instances

```csharp
public static int TotalUsersCreated { get; set; }
public static void Log(string msg) { }
```

---

### Q3: Can you access static members from instance methods?
**A:** Yes!

```csharp
public class Person
{
    public static int TotalPeople { get; set; }

    public Person(string name)
    {
        TotalPeople++;  // Access static from instance method
    }
}
```

---

### Q4: Can you access instance members from static methods?
**A:** No! Static methods don't have `this` context.

```csharp
public class Person
{
    public string Name { get; set; }

    public static void PrintName()
    {
        // Console.WriteLine(Name);  // ❌ Error
        // Must pass object as parameter
    }
}
```

---

### Q5: What is a static constructor?
**A:** Runs once when the class is first used, before any instance is created.

```csharp
public class Settings
{
    static Settings()
    {
        Console.WriteLine("Class initialized once");
    }

    public Settings()
    {
        Console.WriteLine("Instance created");
    }
}

Settings s1 = new Settings();  // Both lines print
Settings s2 = new Settings();  // Only instance constructor
```

---

### Q6: Can you have both static and instance constructors?
**A:** Yes!

```csharp
public class MyClass
{
    static MyClass()      // Runs once
    {
        Console.WriteLine("Static");
    }

    public MyClass()      // Runs each time
    {
        Console.WriteLine("Instance");
    }
}
```

---

### Q7: What is the singleton pattern?
**A:** A class with only one instance, accessible via static method.

```csharp
public class Singleton
{
    private static Singleton instance;

    private Singleton() { }

    public static Singleton GetInstance()
    {
        if (instance == null)
            instance = new Singleton();
        return instance;
    }
}

Singleton s1 = Singleton.GetInstance();
Singleton s2 = Singleton.GetInstance();
// s1 and s2 are the same object
```

---

### Q8: What are utility classes?
**A:** Classes with only static methods, never instantiated.

```csharp
public static class MathHelper
{
    public static int Add(int a, int b) { return a + b; }
    public static int Multiply(int a, int b) { return a * b; }
}

int result = MathHelper.Add(5, 10);
```

---

### Q9: Can you inherit static members?
**A:** No! Static members are not inherited, but can be accessed through derived class.

```csharp
public class Parent
{
    public static void Greet() { }
}

public class Child : Parent { }

Parent.Greet();   // ✅ Works
Child.Greet();    // ✅ Works (but calls Parent's version)
```

---

### Q10: What happens if you modify a static variable?
**A:** The change affects all objects.

```csharp
public class Counter
{
    public static int Count { get; set; }
}

Counter.Count = 5;

var c1 = new Counter();
var c2 = new Counter();

// Both see the same value
Console.WriteLine(Counter.Count);  // 5
```

---

## 🚀 Key Takeaways

✅ Static = Shared by all objects  
✅ Instance = Each object has own copy  
✅ Static accessed via class name: `ClassName.Member`  
✅ Use static for shared utilities and counters  
✅ Static constructor runs once at start  
✅ Cannot access instance from static without parameter  
✅ Singleton pattern = One instance only
