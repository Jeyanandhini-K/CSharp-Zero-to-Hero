# 🧱 Structs in C#

---

## 🧠 What is a Struct?

A `struct` is a **value type** you define yourself. It looks like a class — but behaves like `int` or `bool`. When you assign it, it gets **copied**, not shared.

```csharp
struct Point
{
    public int X;
    public int Y;

    public Point(int x, int y)
    {
        X = x;
        Y = y;
    }
}
```

---

## ⚡ The Big Difference: Copy vs Share

```csharp
// With a STRUCT (value type) ✅
Point a = new Point(1, 2);
Point b = a;       // COPY — completely independent
b.X = 99;

Console.WriteLine(a.X);   // 1  ← unchanged!
Console.WriteLine(b.X);   // 99

// With a CLASS (reference type) ⚠️
// b = a would make both point to the SAME object
// changing b.X would also change a.X!
```

> 💡 Structs **copy** on assignment. Classes **share** on assignment. This is the value vs reference type difference — but for types *you* create.

---

## 📦 Structs Can Have Methods Too

```csharp
struct Point
{
    public int X;
    public int Y;

    public Point(int x, int y) { X = x; Y = y; }

    public double DistanceToOrigin() => Math.Sqrt(X * X + Y * Y);

    public override string ToString() => $"({X}, {Y})";
}

var p = new Point(3, 4);
Console.WriteLine(p.DistanceToOrigin());  // 5
Console.WriteLine(p);                     // (3, 4)
```

## 🔒 `readonly struct` — Immutable by Design

Mutable structs cause sneaky bugs. Lock them down with `readonly`:

```csharp
readonly struct Money
{
    public decimal Amount { get; }
    public string Currency { get; }

    public Money(decimal amount, string currency)
    {
        Amount = amount;
        Currency = currency;
    }
}

var price = new Money(9.99m, "USD");
// price.Amount = 5m;  ❌ compiler error — can't change it
```

> ✅ Best practice: **always make structs immutable** to avoid surprises.

---

## 🚨 Common Pitfall: Mutating a Copy

```csharp
var points = new List<Point> { new Point(1, 1) };

points[0].X = 5;   // ❌ You're mutating a COPY, not the stored value!

// ✅ Do this instead — replace the whole value:
points[0] = new Point(5, 1);
```

> This trips up almost every beginner. Remember: list indexing gives you a **copy** of the struct, not the original.

---

## 🏗️ What Structs Can and Can't Do

| ✅ Can | ❌ Can't |
|---|---|
| Have fields, properties, methods | Inherit from a class or another struct |
| Have constructors | Be `null` (unless `Nullable<T>`) |
| Implement interfaces | Use parameterless constructor (before C# 10) |
| Be `readonly` | Have a finalizer |

---

## 🧱 Built-in Types That Are Structs

You've been using structs all along!

```csharp
int, double, bool, char    // all structs
DateTime, TimeSpan, Guid   // structs
(int, string)              // tuples are structs too!
```

That's why `var d2 = d1;` on a `DateTime` gives you an independent copy.

---

## 🆚 Struct vs Class — When to Use Which?

| Use a `struct` when... | Use a `class` when... |
|---|---|
| Data is small and simple | Object has identity |
| It represents a single value | Data is large or complex |
| It's immutable | You need inheritance |
| Copying is fine | You want to share the object |
| e.g. `Point`, `Color`, `Money` | e.g. `User`, `Order`, `BankAccount` |

> 💡 **When in doubt → use a class.** Reach for a struct only when you have a clear, value-like reason.

## 🎤 Interview Q&A

**Q1: What is a struct in C#?**
A: A value type you define. Copied on assignment, stored on the stack.
```csharp
struct Point { public int X; public int Y; }
```

**Q2: How is a struct different from a class?**
A: Struct = value type (copied). Class = reference type (shared).
```csharp
Point a = new Point(1, 2);
Point b = a;   // copy — independent
b.X = 99;
Console.WriteLine(a.X);  // 1 — unchanged!
```

**Q3: Can structs have methods?**
A: Yes! Constructors, methods, properties — just like a class.

**Q4: Can structs inherit?**
A: No. Structs can't inherit from classes or other structs, but they can implement interfaces.

**Q5: What is `readonly struct`?**
A: A struct where no fields can be changed after construction. Prevents bugs.
```csharp
readonly struct Money { public decimal Amount { get; } }
```

**Q6: Can a struct be null?**
A: Not by default. Use `Point?` (Nullable<Point>) to allow null.

**Q7: What built-in types are structs?**
A: `int`, `double`, `bool`, `char`, `DateTime`, `TimeSpan`, `Guid`, and tuples.

**Q8: What is the most common struct pitfall?**
A: Mutating a copy thinking you're changing the original.
```csharp
list[0].X = 5;   // ❌ mutates a copy!
list[0] = new Point(5, 1);  // ✅ replace it
```

---

## 🚀 Key Takeaways

✅ `struct` = value type — **copied** on assignment  
✅ `class` = reference type — **shared** on assignment  
✅ Structs can have fields, properties, methods, constructors  
✅ Structs **cannot** inherit  
✅ Always prefer `readonly struct` — avoids mutation bugs  
✅ Use structs for small, simple, value-like data  
✅ Default to `class` when in doubt  
✅ `int`, `bool`, `DateTime` are all structs under the hood
