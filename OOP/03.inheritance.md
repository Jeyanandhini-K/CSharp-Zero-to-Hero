# 📚 Inheritance in C#

## 🧠 What is Inheritance?

**Inheritance** allows a class to **inherit** properties and methods from another class. It's used to avoid code duplication and create a hierarchy.

> Parent class (base) → Child class (derived)

---

## 📌 Basic Inheritance

```csharp
// Parent class (base class)
public class Animal
{
    public string Name { get; set; }
    public int Age { get; set; }

    public void Eat()
    {
        Console.WriteLine($"{Name} is eating.");
    }

    public void Sleep()
    {
        Console.WriteLine($"{Name} is sleeping.");
    }
}

// Child class (derived class)
public class Dog : Animal
{
    public void Bark()
    {
        Console.WriteLine($"{Name} is barking: Woof!");
    }
}

// Usage
Dog dog = new Dog();
dog.Name = "Buddy";
dog.Age = 3;
dog.Eat();    // Inherited from Animal
dog.Sleep();  // Inherited from Animal
dog.Bark();   // Own method
```

**Dog inherits everything from Animal:**
- ✅ Properties: `Name`, `Age`
- ✅ Methods: `Eat()`, `Sleep()`
- ✅ Plus its own: `Bark()`

---

## 🔗 Constructor in Inheritance

The child class must initialize the parent class using `base()`.

```csharp
public class Animal
{
    public string Name { get; set; }
    public int Age { get; set; }

    public Animal(string name, int age)
    {
        Name = name;
        Age = age;
    }
}

public class Dog : Animal
{
    public string Breed { get; set; }

    // Call parent constructor with base()
    public Dog(string name, int age, string breed) : base(name, age)
    {
        Breed = breed;
    }
}

Dog dog = new Dog("Buddy", 3, "Labrador");
Console.WriteLine($"{dog.Name}, {dog.Age}, {dog.Breed}");
// Buddy, 3, Labrador
```

## 🔄 Method Overriding

A child class can **override** a parent method to change its behavior using `virtual` and `override`.

```csharp
public class Animal
{
    public string Name { get; set; }

    // Mark as virtual to allow overriding
    public virtual void Speak()
    {
        Console.WriteLine($"{Name} makes a sound.");
    }
}

public class Dog : Animal
{
    // Override the parent method
    public override void Speak()
    {
        Console.WriteLine($"{Name} barks: Woof!");
    }
}

public class Cat : Animal
{
    public override void Speak()
    {
        Console.WriteLine($"{Name} meows: Meow!");
    }
}

Animal a1 = new Dog();
a1.Name = "Buddy";
a1.Speak();  // Buddy barks: Woof!

Animal a2 = new Cat();
a2.Name = "Whiskers";
a2.Speak();  // Whiskers meows: Meow!
```

**Important keywords:**
- `virtual` = "This method can be overridden in child classes"
- `override` = "I'm providing a new version of this method"

---

## 🆔 Understanding `base`

`base` refers to the **parent class**. Use it to call parent methods or properties.

```csharp
public class Animal
{
    public string Name { get; set; }

    public virtual void Speak()
    {
        Console.WriteLine($"{Name} makes a sound.");
    }
}

public class Dog : Animal
{
    public override void Speak()
    {
        base.Speak();  // Call parent's Speak() first
        Console.WriteLine("Woof!");
    }
}

Dog dog = new Dog();
dog.Name = "Buddy";
dog.Speak();
// Output:
// Buddy makes a sound.
// Woof!
```

## 🎯 Multi-level Inheritance

A class can inherit from another inherited class.

```csharp
public class Animal
{
    public string Name { get; set; }
    public virtual void Speak() { }
}

public class Mammal : Animal
{
    public void Nurse() { }
}

public class Dog : Mammal
{
    public void Fetch() { }
}

Dog dog = new Dog();
dog.Name = "Buddy";
dog.Speak();    // From Animal
dog.Nurse();    // From Mammal
dog.Fetch();    // Own method
```

## 🚫 Access Modifiers in Inheritance

- `public` = Child can access
- `protected` = Child can access, outside cannot
- `private` = Only parent class itself can access

```csharp
public class Animal
{
    public string Name { get; set; }           // Child can access
    protected int Age { get; set; }            // Child can access, outside cannot
    private string Id { get; set; }            // Only Animal class can access
}

public class Dog : Animal
{
    public void PrintInfo()
    {
        Console.WriteLine(Name);   // ✅ OK - public
        Console.WriteLine(Age);    // ✅ OK - protected
        // Console.WriteLine(Id);  // ❌ ERROR - private
    }
}
```

## 📊 Real-World Example

```csharp
public class Vehicle
{
    public string Brand { get; set; }
    public int Year { get; set; }

    public Vehicle(string brand, int year)
    {
        Brand = brand;
        Year = year;
    }

    public virtual void Drive()
    {
        Console.WriteLine($"{Brand} is driving.");
    }
}

public class Car : Vehicle
{
    public int Doors { get; set; }

    public Car(string brand, int year, int doors) : base(brand, year)
    {
        Doors = doors;
    }

    public override void Drive()
    {
        Console.WriteLine($"{Brand} car with {Doors} doors is driving.");
    }
}

public class Motorcycle : Vehicle
{
    public bool HasSidecar { get; set; }

    public Motorcycle(string brand, int year, bool hasSidecar) : base(brand, year)
    {
        HasSidecar = hasSidecar;
    }

    public override void Drive()
    {
        Console.WriteLine($"{Brand} motorcycle is driving.");
    }
}

Car car = new Car("Toyota", 2020, 4);
car.Drive();  // Toyota car with 4 doors is driving.

Motorcycle bike = new Motorcycle("Harley", 2021, false);
bike.Drive();  // Harley motorcycle is driving.
```

## 📋 Summary Table

| Concept | Meaning | Example |
|---|---|---|
| **Inheritance** | Child gets parent's members | `class Dog : Animal` |
| **Parent/Base** | Class being inherited from | `public class Animal` |
| **Child/Derived** | Class inheriting | `public class Dog : Animal` |
| **virtual** | Can be overridden | `public virtual void Speak()` |
| **override** | New version in child | `public override void Speak()` |
| **base** | Refer to parent | `base.Speak()` |
| **protected** | Parent and child only | `protected int Age` |

## 🎤 Interview Q&A

### Q1: What is inheritance?
**A:** Inheritance allows a child class to inherit properties and methods from a parent class, avoiding code duplication.

```csharp
public class Animal
{
    public void Eat() { }
}

public class Dog : Animal  // Inherits from Animal
{
    public void Bark() { }
}

Dog d = new Dog();
d.Eat();   // Inherited
d.Bark();  // Own
```

---

### Q2: What is the difference between a base class and a derived class?
**A:**
- **Base class** (parent) = Class being inherited from
- **Derived class** (child) = Class inheriting from the base

```csharp
public class Animal { }        // Base class
public class Dog : Animal { }  // Derived class
```

---

### Q3: How do you call a parent class constructor?
**A:** Using `base()` in the child's constructor.

```csharp
public class Animal
{
    public Animal(string name) { }
}

public class Dog : Animal
{
    public Dog(string name) : base(name)  // Calls Animal's constructor
    {
    }
}
```

---

### Q4: What is method overriding?
**A:** A child class provides a new implementation of a parent method. Parent must use `virtual`, child uses `override`.

```csharp
public class Animal
{
    public virtual void Speak()  // Can be overridden
    {
        Console.WriteLine("Sound");
    }
}

public class Dog : Animal
{
    public override void Speak()  // New implementation
    {
        Console.WriteLine("Woof!");
    }
}
```

---

### Q5: What is the difference between method overriding and method overloading?
**A:**
- **Overriding** = Child provides new implementation of parent's method
- **Overloading** = Multiple methods with same name, different parameters

```csharp
// Overriding (same signature, different implementation)
public class Animal
{
    public virtual void Speak() { }
}
public class Dog : Animal
{
    public override void Speak() { }
}

// Overloading (same name, different parameters)
public void Speak(string word) { }
public void Speak(int times) { }
```

---

### Q6: What is the `base` keyword?
**A:** `base` refers to the parent class. Use it to call parent methods or constructors.

```csharp
public class Dog : Animal
{
    public override void Speak()
    {
        base.Speak();  // Call parent's Speak()
        Console.WriteLine("Woof!");
    }

    public Dog(string name) : base(name)  // Call parent's constructor
    {
    }
}
```

---

### Q7: What is the difference between `public`, `protected`, and `private`?
**A:**
- `public` = Accessible everywhere
- `protected` = Accessible in class and derived classes
- `private` = Accessible only in the class itself

```csharp
public class Animal
{
    public string Name { get; set; }         // Accessible everywhere
    protected int Age { get; set; }          // Child classes can access
    private string Id { get; set; }          // Only Animal can access
}

public class Dog : Animal
{
    public void Test()
    {
        Console.WriteLine(Name);   // ✅ OK
        Console.WriteLine(Age);    // ✅ OK
        // Console.WriteLine(Id);  // ❌ ERROR
    }
}
```

---

### Q8: Can you inherit from multiple classes?
**A:** No! C# supports single inheritance (one parent class only). Use interfaces for multiple inheritance (covered in Interfaces topic).

```csharp
public class Animal { }
public class Pet { }

// ❌ ERROR: Cannot inherit from multiple classes
// public class Dog : Animal, Pet { }

// ✅ Correct: Single inheritance
public class Dog : Animal { }
```

---

### Q9: What is multi-level inheritance?
**A:** A class inherits from another class which itself inherits from another class.

```csharp
public class Animal { }
public class Mammal : Animal { }
public class Dog : Mammal { }

Dog d = new Dog();
// Dog inherits from Mammal, which inherits from Animal
```

---

### Q10: What happens if child and parent both define a method with same name?
**A:** The child's version is used. To call parent's version, use `base.MethodName()`.

```csharp
public class Animal
{
    public virtual void Speak()
    {
        Console.WriteLine("Sound");
    }
}

public class Dog : Animal
{
    public override void Speak()
    {
        base.Speak();           // Parent's version
        Console.WriteLine("Woof!");  // Child's version
    }
}
```

## 🚀 Key Takeaways

✅ Inheritance reduces code duplication  
✅ Child inherits parent's properties and methods  
✅ `virtual` allows overriding, `override` implements new version  
✅ `base()` calls parent constructor  
✅ `base.Method()` calls parent method  
✅ `protected` is for parent-child access  
✅ C# supports single inheritance only
