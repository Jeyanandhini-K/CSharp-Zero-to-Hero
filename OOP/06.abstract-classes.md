# 🎲 Abstract Classes in C#

## 🧠 What is an Abstract Class?

An **abstract class** is a class that **cannot be instantiated**. It's designed to be a **base class** that provides common functionality and defines methods that child classes must implement.

> Abstract class = "Blueprint with some predefined behavior"

---

## 📌 Basic Abstract Class

```csharp
// Abstract class
public abstract class Animal
{
    public string Name { get; set; }

    // Regular method - has implementation
    public void Sleep()
    {
        Console.WriteLine($"{Name} is sleeping.");
    }

    // Abstract method - no implementation (must override)
    public abstract void Speak();

    // Another abstract method
    public abstract void Eat();
}

// Concrete class (implementation of abstract class)
public class Dog : Animal
{
    public override void Speak()
    {
        Console.WriteLine($"{Name} barks: Woof!");
    }

    public override void Eat()
    {
        Console.WriteLine($"{Name} is eating dog food.");
    }
}

// Usage
// ❌ Cannot do: new Animal();  // Abstract class
// ✅ Can do: new Dog();

Dog dog = new Dog();
dog.Name = "Buddy";
dog.Sleep();   // From Animal
dog.Speak();   // Overridden in Dog
dog.Eat();     // Overridden in Dog
```

## 🔄 Abstract Class with Constructor

Abstract classes can have constructors to initialize base properties.

```csharp
public abstract class Vehicle
{
    public string Brand { get; set; }
    public int Year { get; set; }

    // Constructor in abstract class
    public Vehicle(string brand, int year)
    {
        Brand = brand;
        Year = year;
    }

    // Concrete method
    public void ShowInfo()
    {
        Console.WriteLine($"{Brand} - {Year}");
    }

    // Abstract method
    public abstract void Drive();
}

public class Car : Vehicle
{
    public int Doors { get; set; }

    // Must call parent constructor
    public Car(string brand, int year, int doors) : base(brand, year)
    {
        Doors = doors;
    }

    public override void Drive()
    {
        Console.WriteLine($"{Brand} car is driving with {Doors} doors.");
    }
}

Car car = new Car("Toyota", 2020, 4);
car.ShowInfo();  // Toyota - 2020
car.Drive();     // Toyota car is driving with 4 doors.
```

## 🏗️ Real-World Example: Payment Processing

```csharp
public abstract class PaymentProcessor
{
    public string TransactionId { get; set; }
    public abstract string ProcessorName { get; }

    // Concrete method - same for all
    public void LogTransaction(double amount)
    {
        Console.WriteLine($"[{ProcessorName}] Transaction: ${amount}");
    }

    // Abstract method - each implements differently
    public abstract bool ValidatePayment(double amount);
    public abstract void ProcessPayment(double amount);
}

public class CreditCardProcessor : PaymentProcessor
{
    public override string ProcessorName => "Credit Card";

    public override bool ValidatePayment(double amount)
    {
        return amount > 0 && amount < 10000;  // Validation logic
    }

    public override void ProcessPayment(double amount)
    {
        if (ValidatePayment(amount))
        {
            Console.WriteLine($"Processing ${amount} via Credit Card...");
            TransactionId = Guid.NewGuid().ToString();
        }
    }
}

public class PayPalProcessor : PaymentProcessor
{
    public override string ProcessorName => "PayPal";

    public override bool ValidatePayment(double amount)
    {
        return amount > 0 && amount < 5000;
    }

    public override void ProcessPayment(double amount)
    {
        if (ValidatePayment(amount))
        {
            Console.WriteLine($"Processing ${amount} via PayPal...");
            TransactionId = Guid.NewGuid().ToString();
        }
    }
}

PaymentProcessor processor1 = new CreditCardProcessor();
processor1.LogTransaction(100);
processor1.ProcessPayment(100);

PaymentProcessor processor2 = new PayPalProcessor();
processor2.LogTransaction(100);
processor2.ProcessPayment(100);
```

## 🔗 Abstract Properties

Abstract classes can define properties that child classes must implement.

```csharp
public abstract class Employee
{
    public string Name { get; set; }
    
    // Abstract property
    public abstract double Salary { get; set; }
    
    // Abstract method
    public abstract void CalculateBonus();
}

public class Manager : Employee
{
    private double salary;
    
    public override double Salary
    {
        get { return salary; }
        set { salary = value; }
    }

    public override void CalculateBonus()
    {
        Console.WriteLine($"Manager bonus: ${Salary * 0.2}");
    }
}

Manager mgr = new Manager();
mgr.Name = "Ali";
mgr.Salary = 5000;
mgr.CalculateBonus();  // Manager bonus: $1000
```

## 🔄 Abstract Class vs Interface

| Feature | Abstract Class | Interface |
|---|---|---|
| Purpose | Base functionality + contract | Contract only |
| Implementation | Can have | Cannot have (before C# 8) |
| Constructor | ✅ Yes | ❌ No |
| Multiple inheritance | ❌ No | ✅ Yes |
| Access modifiers | ✅ Yes | ❌ No |
| Abstract members | ✅ Yes | ✅ Yes |

```csharp
// Abstract class - provides base functionality
public abstract class Shape
{
    protected string Color;  // Can be protected
    public abstract double GetArea();
    public void SetColor(string c) { Color = c; }  // Implementation
}

// Interface - defines contract only
public interface IDrawable
{
    void Draw();
}
```

## 📋 Summary Table

| Concept | Meaning | Example |
|---|---|---|
| **Abstract class** | Cannot instantiate | `public abstract class Animal` |
| **Abstract method** | Must be overridden | `public abstract void Speak();` |
| **Abstract property** | Must be implemented | `public abstract double Salary { get; set; }` |
| **override** | Implement abstract member | `public override void Speak()` |
| **Concrete class** | Can be instantiated | `public class Dog : Animal` |

## 🎤 Interview Q&A

### Q1: What is an abstract class?
**A:** A class that cannot be instantiated. It serves as a base class providing common functionality and defining abstract methods that child classes must implement.

```csharp
public abstract class Animal { }

// ❌ Cannot do
new Animal();

// ✅ Can do
public class Dog : Animal { }
new Dog();
```

---

### Q2: What is an abstract method?
**A:** A method with no implementation. Child classes must provide the implementation using `override`.

```csharp
public abstract class Animal
{
    public abstract void Speak();  // No body
}

public class Dog : Animal
{
    public override void Speak()
    {
        Console.WriteLine("Woof!");
    }
}
```

---

### Q3: Can an abstract class have concrete methods?
**A:** Yes! Abstract classes can have both abstract (must override) and concrete (ready to use) methods.

```csharp
public abstract class Animal
{
    public abstract void Speak();      // Abstract
    public void Sleep() { }            // Concrete
}

public class Dog : Animal
{
    public override void Speak() { }   // Must implement
    // Sleep() inherited as-is
}
```

---

### Q4: Can an abstract class have a constructor?
**A:** Yes! Used to initialize base properties.

```csharp
public abstract class Vehicle
{
    public string Brand { get; set; }

    public Vehicle(string brand)
    {
        Brand = brand;
    }
}

public class Car : Vehicle
{
    public Car(string brand) : base(brand) { }
}
```

---

### Q5: What is the difference between `abstract` and `virtual`?
**A:**
- `abstract` = Must be overridden (no implementation)
- `virtual` = Can be overridden (has implementation)

```csharp
public abstract class Animal
{
    public abstract void Speak();      // Must override
    public virtual void Sleep() { }    // Can override
}
```

---

### Q6: Can you have multiple abstract methods?
**A:** Yes! A child class must override all of them.

```csharp
public abstract class Animal
{
    public abstract void Speak();
    public abstract void Eat();
    public abstract void Move();
}

public class Dog : Animal
{
    public override void Speak() { }
    public override void Eat() { }
    public override void Move() { }
}
```

---

### Q7: What if a child class doesn't implement all abstract methods?
**A:** The child class must also be declared `abstract`.

```csharp
public abstract class Animal
{
    public abstract void Speak();
    public abstract void Eat();
}

// Incomplete implementation
public abstract class Mammal : Animal
{
    public override void Eat() { }  // Implements one
    // Speak() still abstract
}

// Complete implementation
public class Dog : Mammal
{
    public override void Speak() { }  // Implements remaining
}
```

---

### Q8: Can an abstract class inherit from another abstract class?
**A:** Yes!

```csharp
public abstract class Animal { }
public abstract class Mammal : Animal { }
public class Dog : Mammal { }
```

---

### Q9: When should you use abstract class vs interface?
**A:**
- Use **abstract class** when you have shared implementation/state
- Use **interface** when you're just defining a contract

```csharp
// Abstract class - provides common functionality
public abstract class Vehicle
{
    public void Refuel() { }  // Same for all vehicles
    public abstract void Drive();
}

// Interface - just defines behavior
public interface IPayable
{
    void ProcessPayment();
}
```

---

### Q10: Can abstract class implement an interface?
**A:** Yes!

```csharp
public interface ILogger
{
    void Log(string message);
}

public abstract class DataProcessor : ILogger
{
    public void Log(string message) { }
    public abstract void Process();
}
```

---

## 🚀 Key Takeaways

✅ Abstract class = Cannot instantiate  
✅ Abstract method = Must be overridden  
✅ Can mix abstract and concrete methods  
✅ Can have constructor and properties  
✅ Use when sharing implementation  
✅ Better than interface when state needed  
✅ All abstract members must be overridden
