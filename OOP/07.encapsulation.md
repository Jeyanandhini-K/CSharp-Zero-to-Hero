# 🔒 Encapsulation in C#

## 🧠 What is Encapsulation?

**Encapsulation** is the practice of **hiding internal data** and providing **controlled access** through properties and methods. It protects your object's state from misuse.

> Encapsulation = "Hide implementation, expose interface"

---

## 📌 Basic Encapsulation with Properties

```csharp
public class BankAccount
{
    private double balance;  // Hidden from outside

    public double Balance
    {
        get { return balance; }
        set
        {
            if (value >= 0)
                balance = value;
            else
                Console.WriteLine("Balance cannot be negative!");
        }
    }
}

BankAccount account = new BankAccount();
account.Balance = 1000;      // Valid ✅
account.Balance = -500;      // Invalid ❌ (rejected by setter)
```

**Key benefits:**
- ✅ Data validation
- ✅ Control how data is accessed
- ✅ Can change implementation without breaking external code

## 🔐 Access Modifiers

```csharp
public class Person
{
    public string Name { get; set; }                    // Anyone can access
    protected int Age { get; set; }                     // Class and derived classes
    private string SocialSecurityNumber { get; set; }   // Only this class
    internal string EmployeeId { get; set; }            // Only this project
}
```

| Modifier | Accessible From | Usage |
|---|---|---|
| `public` | Anywhere | Properties users need |
| `protected` | This class + derived classes | Parent-child shared data |
| `private` | Only this class | Internal data |
| `internal` | Same project/assembly | Project-wide data |

## 💡 Practical Example: Student Class

```csharp
public class Student
{
    // Hidden fields
    private string id;
    private double gpa;
    private List<double> grades;

    // Property with validation
    public string Name { get; set; }

    public double GPA
    {
        get { return gpa; }
        set
        {
            if (value >= 0 && value <= 4.0)
                gpa = value;
            else
                throw new ArgumentException("GPA must be between 0 and 4.0");
        }
    }

    public string Id
    {
        get { return id; }
        private set { id = value; }  // Only setter is private
    }

    public List<double> Grades
    {
        get { return new List<double>(grades); }  // Return copy, not original
    }

    // Constructor
    public Student(string name)
    {
        Name = name;
        Id = Guid.NewGuid().ToString();
        grades = new List<double>();
        GPA = 0;
    }

    // Public method
    public void AddGrade(double grade)
    {
        if (grade >= 0 && grade <= 100)
        {
            grades.Add(grade);
            CalculateGPA();
        }
    }

    // Private method (internal logic)
    private void CalculateGPA()
    {
        if (grades.Count == 0)
            GPA = 0;
        else
            GPA = grades.Average() / 25;  // Convert to 4.0 scale
    }
}

// Usage
Student student = new Student("Ali");
student.Name = "Ali";
student.AddGrade(90);
student.AddGrade(85);
Console.WriteLine(student.GPA);  // Calculated automatically

// Cannot access private members
// student.grades;  // ❌ Error
// student.CalculateGPA();  // ❌ Error
```

## 🎯 Property Patterns in C#

### Pattern 1: Auto-Property

```csharp
public string Name { get; set; }
public int Age { get; set; }
```

### Pattern 2: Read-Only Property

```csharp
private string id;
public string Id
{
    get { return id; }
}
```

Or shorter syntax:

```csharp
public string Id { get; private set; }
```

### Pattern 3: Computed Property

```csharp
public double CircleArea
{
    get { return Math.PI * radius * radius; }
}
```

### Pattern 4: Property with Validation

```csharp
private int age;
public int Age
{
    get { return age; }
    set
    {
        if (value >= 0 && value <= 150)
            age = value;
    }
}
```

### Pattern 5: Init-Only Property (C# 9+)

```csharp
public class Person
{
    public string Name { get; init; }  // Can only set during initialization
}

Person p = new Person { Name = "Ali" };
p.Name = "Sara";  // ❌ Error
```

## 🛡️ Defensive Copying

Expose copies of collections, not originals.

```csharp
public class Team
{
    private List<string> members;

    public Team()
    {
        members = new List<string>();
    }

    // ❌ Bad - returns original list
    public List<string> GetMembers_Bad()
    {
        return members;
    }

    // ✅ Good - returns copy
    public List<string> GetMembers()
    {
        return new List<string>(members);
    }

    public void AddMember(string name)
    {
        members.Add(name);
    }
}

Team team = new Team();
team.AddMember("Ali");
team.AddMember("Fatima");

List<string> membersCopy = team.GetMembers();
membersCopy.Add("Hacker");  // Modifies copy, not original
```

## 📊 Real-World Example: Bank Account

```csharp
public class BankAccount
{
    private string accountNumber;
    private double balance;
    private List<string> transactions;

    public string AccountNumber
    {
        get { return accountNumber; }
        private set { accountNumber = value; }
    }

    public double Balance
    {
        get { return balance; }
        private set { balance = value; }  // Only internal methods can set
    }

    public BankAccount(double initialBalance)
    {
        AccountNumber = Guid.NewGuid().ToString();
        Balance = initialBalance;
        transactions = new List<string>();
        transactions.Add($"Initial deposit: ${initialBalance}");
    }

    public void Deposit(double amount)
    {
        if (amount <= 0)
            throw new ArgumentException("Amount must be positive");

        Balance += amount;
        transactions.Add($"Deposit: +${amount}");
    }

    public void Withdraw(double amount)
    {
        if (amount <= 0)
            throw new ArgumentException("Amount must be positive");

        if (amount > Balance)
            throw new InvalidOperationException("Insufficient funds");

        Balance -= amount;
        transactions.Add($"Withdrawal: -${amount}");
    }

    public List<string> GetTransactionHistory()
    {
        return new List<string>(transactions);  // Defensive copy
    }
}

// Usage
BankAccount account = new BankAccount(1000);
account.Deposit(500);
account.Withdraw(200);

Console.WriteLine($"Balance: ${account.Balance}");
foreach (string transaction in account.GetTransactionHistory())
{
    Console.WriteLine(transaction);
}

// Cannot access private members
// account.balance = -1000;  // ❌ Error
// account.Balance = -1000;  // ❌ Error (setter is private)
```

## 📋 Summary Table

| Concept | Meaning | Example |
|---|---|---|
| **Encapsulation** | Hide data, expose interface | Private fields, public properties |
| **Private field** | Hidden from outside | `private double balance;` |
| **Public property** | Controlled access | `public double Balance { get; set; }` |
| **Validation** | Check in setter | `if (value >= 0) balance = value;` |
| **Read-only** | Cannot change after set | `public string Id { get; private set; }` |
| **Defensive copy** | Return copy, not original | `return new List<>(items);` |

## 🎤 Interview Q&A

### Q1: What is encapsulation?
**A:** Hiding internal data and providing controlled access through properties and methods. Protects object state from misuse.

```csharp
public class BankAccount
{
    private double balance;  // Hidden
    
    public double Balance
    {
        get { return balance; }
        set { if (value >= 0) balance = value; }
    }
}
```

---

### Q2: Why is encapsulation important?
**A:**
- ✅ Data validation - ensure valid state
- ✅ Control access - prevent misuse
- ✅ Change implementation later without breaking code
- ✅ Maintainability - clear interface

---

### Q3: What is the difference between a field and a property?
**A:**
- **Field** = Simple variable (rarely used in professional code)
- **Property** = Controlled access with getters/setters

```csharp
// ❌ Field (avoid)
public double Balance;

// ✅ Property (use this)
public double Balance { get; set; }
```

---

### Q4: What are access modifiers?
**A:** Keywords controlling who can access class members:
- `public` = Anyone
- `protected` = Class + derived classes
- `private` = Class only
- `internal` = Same project

---

### Q5: What is a property with a private setter?
**A:** The property can be read publicly but only set from within the class.

```csharp
public class Person
{
    public string Id { get; private set; }
}

Person p = new Person();
Console.WriteLine(p.Id);  // ✅ Can read
// p.Id = "123";  // ❌ Cannot set
```

---

### Q6: How do you validate in a property setter?
**A:** Use an `if` statement to check the value.

```csharp
public int Age
{
    get { return age; }
    set
    {
        if (value >= 0 && value <= 150)
            age = value;
        else
            throw new ArgumentException("Invalid age");
    }
}
```

---

### Q7: What is defensive copying?
**A:** Returning a copy of internal data, not the original. Prevents external modification.

```csharp
private List<string> items;

// ❌ Bad - returns original
public List<string> GetItems() { return items; }

// ✅ Good - returns copy
public List<string> GetItems() { return new List<string>(items); }
```

---

### Q8: Can you have a property without a private field?
**A:** Yes! Auto-properties don't need explicit backing fields.

```csharp
public string Name { get; set; }  // C# creates backing field automatically
```

---

### Q9: What is an init-only property?
**A:** Can be set only during initialization, not changed afterward.

```csharp
public class Person
{
    public string Name { get; init; }
}

Person p = new Person { Name = "Ali" };
// p.Name = "Sara";  // ❌ Error
```

---

### Q10: How does encapsulation help with maintenance?
**A:** You can change internal implementation without affecting external code.

```csharp
// Original
public double Salary { get; set; }

// Later change - still works!
private double salary;
public double Salary
{
    get { return salary * 1.05; }  // Apply tax
    set { salary = value; }
}

// External code doesn't break
employee.Salary = 5000;
```
