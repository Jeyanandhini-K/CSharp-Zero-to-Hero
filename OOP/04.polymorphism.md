# 🎭 Polymorphism in C#

## 🧠 What is Polymorphism?

**Polymorphism** means "many forms". It allows you to use a **parent class reference** to point to **child class objects**, and call the correct method based on the actual object type.

---

## 📌 Compile-Time Polymorphism (Method Overloading)

Same method name, different parameters.

```csharp
public class Calculator
{
    public int Add(int a, int b)
    {
        return a + b;
    }

    public double Add(double a, double b)
    {
        return a + b;
    }

    public int Add(int a, int b, int c)
    {
        return a + b + c;
    }
}

Calculator calc = new Calculator();
Console.WriteLine(calc.Add(5, 10));           // 15
Console.WriteLine(calc.Add(5.5, 10.5));       // 16.0
Console.WriteLine(calc.Add(5, 10, 15));       // 30
```

The correct method is determined **at compile time** based on parameters.

---

## 🔄 Runtime Polymorphism (Method Overriding)

Parent class reference pointing to child class object.

```csharp
public class Animal
{
    public virtual void Speak()
    {
        Console.WriteLine("Animal makes a sound.");
    }
}

public class Dog : Animal
{
    public override void Speak()
    {
        Console.WriteLine("Dog barks: Woof!");
    }
}

public class Cat : Animal
{
    public override void Speak()
    {
        Console.WriteLine("Cat meows: Meow!");
    }
}

// **Key**: Parent class reference, child class objects
Animal animal1 = new Dog();   // Dog object, Animal reference
Animal animal2 = new Cat();   // Cat object, Animal reference

animal1.Speak();  // "Dog barks: Woof!" (Dog's version called)
animal2.Speak();  // "Cat meows: Meow!" (Cat's version called)
```

**The correct method is determined at runtime** based on the actual object type.

## 🎯 Real-World Example: Payment Processing

```csharp
public abstract class PaymentMethod
{
    public abstract void ProcessPayment(double amount);
}

public class CreditCard : PaymentMethod
{
    public override void ProcessPayment(double amount)
    {
        Console.WriteLine($"Processing ${amount} via Credit Card...");
    }
}

public class PayPal : PaymentMethod
{
    public override void ProcessPayment(double amount)
    {
        Console.WriteLine($"Processing ${amount} via PayPal...");
    }
}

public class Bitcoin : PaymentMethod
{
    public override void ProcessPayment(double amount)
    {
        Console.WriteLine($"Processing ${amount} via Bitcoin...");
    }
}

// Process payments polymorphically
public class ShoppingCart
{
    public void Checkout(PaymentMethod payment, double total)
    {
        payment.ProcessPayment(total);  // Calls correct ProcessPayment
    }
}

ShoppingCart cart = new ShoppingCart();

PaymentMethod method1 = new CreditCard();
cart.Checkout(method1, 100);  // Credit Card payment

PaymentMethod method2 = new PayPal();
cart.Checkout(method2, 100);  // PayPal payment

PaymentMethod method3 = new Bitcoin();
cart.Checkout(method3, 100);  // Bitcoin payment
```

## 📊 Polymorphism with Collections

Store multiple types in one collection using parent class reference.

```csharp
public class Animal
{
    public string Name { get; set; }
    public virtual void Speak() { }
}

public class Dog : Animal
{
    public override void Speak()
    {
        Console.WriteLine($"{Name} barks.");
    }
}

public class Cat : Animal
{
    public override void Speak()
    {
        Console.WriteLine($"{Name} meows.");
    }
}

// Collection of parent type, containing child objects
List<Animal> animals = new List<Animal>
{
    new Dog { Name = "Buddy" },
    new Cat { Name = "Whiskers" },
    new Dog { Name = "Max" },
    new Cat { Name = "Fluffy" }
};

// Each calls its own Speak() method
foreach (Animal animal in animals)
{
    animal.Speak();
}

// Output:
// Buddy barks.
// Whiskers meows.
// Max barks.
// Fluffy meows.
```

## 🔍 Type Checking with `is` and `as`

Check or convert object types at runtime.

```csharp
Animal animal1 = new Dog();
Animal animal2 = new Cat();

// Using 'is' to check type
if (animal1 is Dog)
{
    Console.WriteLine("animal1 is a Dog");
}

// Using 'as' to cast safely
Dog dog = animal1 as Dog;
if (dog != null)
{
    Console.WriteLine("Successfully cast to Dog");
}

Cat cat = animal2 as Cat;
if (cat != null)
{
    Console.WriteLine("Successfully cast to Cat");
}
```

## 📋 Summary Table

| Concept | Type | When Determined | Example |
|---|---|---|---|
| **Method Overloading** | Compile-time | At compile | Different parameters |
| **Method Overriding** | Runtime | At runtime | Parent ref, child object |
| **virtual** | Enables overriding | Parent method | Can be overridden |
| **override** | Implements new version | Child method | New implementation |
| **abstract** | Must be overridden | Parent method | No implementation |

## 🎤 Interview Q&A

### Q1: What is polymorphism?
**A:** Polymorphism allows you to use a parent class reference to call methods on child class objects, with the correct method being determined based on the actual object type.

```csharp
Animal animal = new Dog();
animal.Speak();  // Calls Dog's Speak(), not Animal's
```

---

### Q2: What is the difference between compile-time and runtime polymorphism?
**A:**
- **Compile-time** (Method Overloading) = Correct method chosen when code is compiled
- **Runtime** (Method Overriding) = Correct method chosen when code runs

```csharp
// Compile-time: Method overloading
public void Print(int x) { }
public void Print(string x) { }

// Runtime: Method overriding
public virtual void Speak() { }
public override void Speak() { }
```

---

### Q3: What is method overloading?
**A:** Same method name with different parameters. The compiler chooses the correct method based on argument types.

```csharp
public int Add(int a, int b) { return a + b; }
public double Add(double a, double b) { return a + b; }

Add(5, 10);        // Calls int version
Add(5.5, 10.5);    // Calls double version
```

---

### Q4: What is method overriding?
**A:** Child class provides a new implementation of a parent's method. Parent uses `virtual`, child uses `override`.

```csharp
public class Animal
{
    public virtual void Speak() { }
}

public class Dog : Animal
{
    public override void Speak()  // New implementation
    {
        Console.WriteLine("Woof!");
    }
}
```

---

### Q5: Why use polymorphism?
**A:** To write flexible, maintainable code:
- Same code works with different object types
- Easy to add new types without changing existing code
- More readable and object-oriented

---

### Q6: Can you store multiple object types in one collection?
**A:** Yes! Using parent class reference:

```csharp
List<Animal> animals = new List<Animal>
{
    new Dog(),
    new Cat(),
    new Dog()
};

foreach (Animal animal in animals)
{
    animal.Speak();  // Each calls its own version
}
```

---

### Q7: What is the `is` operator?
**A:** Checks if an object is of a specific type or inherits from it.

```csharp
Animal animal = new Dog();

if (animal is Dog)
{
    Console.WriteLine("It's a Dog");
}

if (animal is Animal)
{
    Console.WriteLine("It's an Animal");
}
```

---

### Q8: What is the `as` operator?
**A:** Safely casts an object to a type. Returns null if cast fails.

```csharp
Animal animal = new Dog();

Dog dog = animal as Dog;
if (dog != null)
{
    Console.WriteLine("Cast successful");
}

Cat cat = animal as Cat;
if (cat == null)
{
    Console.WriteLine("Cast failed");
}
```

---

### Q9: What is abstract method and abstract class?
**A:** An abstract method must be overridden in child classes. An abstract class cannot be instantiated.

```csharp
public abstract class Animal
{
    public abstract void Speak();  // Must be overridden
}

public class Dog : Animal
{
    public override void Speak()
    {
        Console.WriteLine("Woof!");
    }
}

// ❌ Cannot do: new Animal();
// ✅ Can do: new Dog();
```

---

### Q10: How is polymorphism useful in real applications?
**A:** It allows you to handle different types uniformly. Example: Payment processing.

```csharp
public class Store
{
    public void ProcessPayment(PaymentMethod payment, double amount)
    {
        payment.ProcessPayment(amount);  // Works with any payment type
    }
}

store.ProcessPayment(new CreditCard(), 100);
store.ProcessPayment(new PayPal(), 100);
store.ProcessPayment(new Bitcoin(), 100);
```
