# 🏗️ Constructors in C#

## 📌 What is a Constructor?

A constructor is a **special method** that runs automatically when you create an object. It initializes the object's properties to valid starting values.

---

## ✍️ Basic Constructor

```csharp
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }

    // Constructor
    public Person(string name, int age)
    {
        Name = name;
        Age = age;
    }
}

Person p = new Person("Ali", 25);
Console.WriteLine($"{p.Name}, {p.Age}");  // Ali, 25
```

---

## 🔄 Constructor Overloading

You can have **multiple constructors** with different parameters. C# will call the correct one based on what you pass.

```csharp
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
    public string Email { get; set; }

    // Constructor 1 - all parameters
    public Person(string name, int age, string email)
    {
        Name = name;
        Age = age;
        Email = email;
    }

    // Constructor 2 - only name and age
    public Person(string name, int age)
    {
        Name = name;
        Age = age;
        Email = "N/A";
    }

    // Constructor 3 - only name
    public Person(string name)
    {
        Name = name;
        Age = 0;
        Email = "N/A";
    }

    // Constructor 4 - no parameters
    public Person()
    {
        Name = "Unknown";
        Age = 0;
        Email = "N/A";
    }
}

Person p1 = new Person("Ali", 25, "ali@email.com");
Person p2 = new Person("Fatima", 22);
Person p3 = new Person("Sara");
Person p4 = new Person();
```

## 🎯 Constructor Chaining with `this()`

Instead of repeating code, one constructor can call another using `this()`.

```csharp
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
    public string Email { get; set; }

    public Person(string name, int age, string email)
    {
        Name = name;
        Age = age;
        Email = email;
    }

    // This constructor calls the above one
    public Person(string name, int age) : this(name, age, "N/A")
    {
    }

    // This constructor also chains
    public Person(string name) : this(name, 0, "N/A")
    {
    }

    public Person() : this("Unknown", 0, "N/A")
    {
    }
}

Person p1 = new Person("Ali", 25, "ali@email.com");
Person p2 = new Person("Fatima", 22);  // Calls Person(string, int)
Person p3 = new Person("Sara");         // Calls Person(string)
Person p4 = new Person();               // Calls Person()
```

**Benefits of constructor chaining:**
- ✅ No code duplication
- ✅ Easier to maintain
- ✅ Default values in one place

## 📦 Optional Parameters

Instead of multiple constructors, you can use optional parameters with default values.

```csharp
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
    public string Email { get; set; }

    // One constructor handles all cases
    public Person(string name, int age = 0, string email = "N/A")
    {
        Name = name;
        Age = age;
        Email = email;
    }
}

Person p1 = new Person("Ali", 25, "ali@email.com");
Person p2 = new Person("Fatima", 22);      // Email defaults to "N/A"
Person p3 = new Person("Sara");            // Age=0, Email="N/A"
```

**Rules for optional parameters:**
- Must come **after** required parameters
- Must have a default value assigned
- Can be omitted when calling

## 🛠️ Using `this.` in Constructor

`this.` clarifies which variable is the property (when parameter has same name).

```csharp
public class Student
{
    public string Name { get; set; }

    public Student(string name)
    {
        this.Name = name;  // "this.Name" = property, "name" = parameter
    }
}
```

## ⚡ Common Constructor Patterns

### Pattern 1: Validation in Constructor

```csharp
public class BankAccount
{
    public string AccountNumber { get; set; }
    public double Balance { get; set; }

    public BankAccount(string accountNumber, double initialBalance)
    {
        if (string.IsNullOrEmpty(accountNumber))
            throw new ArgumentException("Account number cannot be empty");
        
        if (initialBalance < 0)
            throw new ArgumentException("Balance cannot be negative");

        AccountNumber = accountNumber;
        Balance = initialBalance;
    }
}

// ✅ Valid
BankAccount acc1 = new BankAccount("12345", 1000);

// ❌ Throws error
BankAccount acc2 = new BankAccount("", 1000);
BankAccount acc3 = new BankAccount("12345", -500);
```

### Pattern 2: Private Fields with Constructor

```csharp
public class Circle
{
    private double radius;

    public Circle(double r)
    {
        if (r <= 0)
            throw new ArgumentException("Radius must be positive");
        
        radius = r;
    }

    public double GetArea()
    {
        return Math.PI * radius * radius;
    }

    public double GetCircumference()
    {
        return 2 * Math.PI * radius;
    }
}

Circle c = new Circle(5);
Console.WriteLine(c.GetArea());          // 78.54
Console.WriteLine(c.GetCircumference()); // 31.42
```

## 🎤 Interview Q&A

### Q1: What is a constructor?
**A:** A special method that runs automatically when creating an object. It initializes properties to valid starting values.

```csharp
public class Person
{
    public Person(string name)
    {
        Name = name;  // Initialized immediately
    }
}

Person p = new Person("Ali");
```

---

### Q2: What is constructor overloading?
**A:** Having multiple constructors with different parameters. C# calls the correct one based on what you pass.

```csharp
public Person(string name, int age) { }
public Person(string name) { }
public Person() { }

new Person("Ali", 25);  // Calls first constructor
new Person("Ali");      // Calls second constructor
new Person();           // Calls third constructor
```

---

### Q3: What is constructor chaining?
**A:** One constructor calling another using `this()`. Avoids code duplication.

```csharp
public Person(string name, int age, string email)
{
    Name = name;
    Age = age;
    Email = email;
}

public Person(string name, int age) : this(name, age, "N/A")  // Calls above
{
}
```

---

### Q4: What are optional parameters?
**A:** Parameters with default values. They can be omitted when calling the method/constructor.

```csharp
public Person(string name, int age = 0, string email = "N/A")
{
    Name = name;
    Age = age;
    Email = email;
}

new Person("Ali", 25, "ali@email.com");
new Person("Ali", 25);      // Email = "N/A"
new Person("Ali");          // Age = 0, Email = "N/A"
```

---

### Q5: What is the difference between overloading and optional parameters?
**A:**
- **Overloading** = Multiple methods with different signatures
- **Optional parameters** = One method with default values

Both achieve similar results, but optional parameters are often cleaner:

```csharp
// Overloading
public Person(string name) { }
public Person(string name, int age) { }

// Optional parameters (cleaner)
public Person(string name, int age = 0) { }
```

---

### Q6: Can you have a constructor with no parameters?
**A:** Yes! A parameterless constructor (also called default constructor) is useful for default initialization.

```csharp
public class Person
{
    public Person()
    {
        Name = "Unknown";
        Age = 0;
    }
}

Person p = new Person();
```

---

### Q7: What happens if you define a constructor?
**A:** The default parameterless constructor disappears. If you want it, you must define it explicitly.

```csharp
public class Person
{
    public Person(string name)
    {
        Name = name;
    }
}

// ❌ ERROR: Person() doesn't exist anymore
Person p = new Person();

// ✅ CORRECT
Person p = new Person("Ali");
```

---

### Q8: How do you validate in a constructor?
**A:** Use `if` statements and throw exceptions if data is invalid.

```csharp
public BankAccount(string accountNum, double balance)
{
    if (string.IsNullOrEmpty(accountNum))
        throw new ArgumentException("Account number required");
    
    if (balance < 0)
        throw new ArgumentException("Balance cannot be negative");

    AccountNumber = accountNum;
    Balance = balance;
}
```

---

### Q9: What is `this()` in constructor chaining?
**A:** It calls another constructor in the same class, passing parameters to it.

```csharp
public Person(string name, int age, string email)
{
    Name = name;
    Age = age;
    Email = email;
}

public Person(string name, int age) : this(name, age, "N/A")
{
    // Calls the above constructor automatically
}
```

---

### Q10: Can you call a parent class constructor?
**A:** Yes! Using `base()` (covered in Inheritance topic).

```csharp
public class Animal
{
    public Animal(string name) { }
}

public class Dog : Animal
{
    public Dog(string name) : base(name)  // Calls Animal's constructor
    {
    }
}
```

✅ Constructor initializes objects automatically  
✅ Overload constructors for flexibility  
✅ Chain constructors to avoid duplication  
✅ Use optional parameters for simplicity  
✅ Validate input in constructors  
✅ Default constructor disappears if you define one
