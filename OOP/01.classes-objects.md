# 🏗️ Classes and Objects in C#

## 🧠 What is OOP?

**Object-Oriented Programming (OOP)** is a way of writing code by organizing it into **real-world things**. Instead of writing a list of instructions, you create **blueprints** (classes) and build **real things** (objects) from them.

---

## 📌 What is a Class?

A **class** is a **blueprint**. An **object** is an instance of that class.

> 🍪 Cookie cutter (blueprint) → Cookies (objects)

---

## ✍️ How to Define a Class

```csharp
public class Student
{
    public string Name { get; set; }
    public int Age { get; set; }
    public double GPA { get; set; }
}
```

---

## 🔨 How to Create Objects

```csharp
Student student1 = new Student();
student1.Name = "Ali";
student1.Age = 20;
student1.GPA = 3.8;

Console.WriteLine(student1.Name);  // Ali
```

**Multiple independent objects:**

```csharp
Student s1 = new Student();
s1.Name = "Ali";
s1.Age = 20;

Student s2 = new Student();
s2.Name = "Fatima";
s2.Age = 22;
```

---

## 🏗️ Constructors

A constructor runs automatically when you create an object.

```csharp
public class Student
{
    public string Name { get; set; }
    public int Age { get; set; }
    public double GPA { get; set; }

    public Student(string name, int age, double gpa)
    {
        Name = name;
        Age = age;
        GPA = gpa;
    }
}

Student s1 = new Student("Ali", 20, 3.8);
Student s2 = new Student("Fatima", 22, 3.9);
```

---

## ⚙️ Methods

Methods define what objects can do.

```csharp
public class Student
{
    public string Name { get; set; }
    public int Age { get; set; }
    public double GPA { get; set; }

    public Student(string name, int age, double gpa)
    {
        Name = name;
        Age = age;
        GPA = gpa;
    }

    public void Introduce()
    {
        Console.WriteLine($"Hi! I'm {Name}, I'm {Age}, GPA: {GPA}");
    }

    public bool IsHonorRoll()
    {
        return GPA >= 3.5;
    }
}

Student s = new Student("Ali", 20, 3.8);
s.Introduce();  // Hi! I'm Ali, I'm 20, GPA: 3.8
Console.WriteLine(s.IsHonorRoll());  // True
```

## 🔑 The `this` Keyword

`this` refers to the current object.

```csharp
public class Student
{
    public string Name { get; set; }
    public int Age { get; set; }

    public Student(string name, int age)
    {
        this.Name = name;  // "this.Name" = property
        this.Age = age;    // "name" = parameter
    }
}
```

---

## 🧪 Complete Example

```csharp
public class Car
{
    public string Brand { get; set; }
    public string Color { get; set; }
    public int Speed { get; set; }

    public Car(string brand, string color)
    {
        this.Brand = brand;
        this.Color = color;
        this.Speed = 0;
    }

    public void Accelerate(int amount)
    {
        Speed += amount;
        Console.WriteLine($"{Brand} is now going {Speed} km/h.");
    }

    public void Brake()
    {
        Speed = 0;
        Console.WriteLine($"{Brand} has stopped.");
    }

    public void ShowInfo()
    {
        Console.WriteLine($"Brand: {Brand} | Color: {Color} | Speed: {Speed}");
    }
}

Car car1 = new Car("Toyota", "Red");
car1.ShowInfo();       // Brand: Toyota | Color: Red | Speed: 0
car1.Accelerate(60);   // Toyota is now going 60 km/h
car1.Brake();          // Toyota has stopped
```

## 🎤 Interview Q&A

### Q1: What is the difference between a class and an object?
**A:** A class is a blueprint (template), an object is an actual instance created from that blueprint. One class can create many objects.

```csharp
public class Dog { }  // Class = blueprint

Dog dog1 = new Dog();  // Object 1
Dog dog2 = new Dog();  // Object 2
```

---

### Q2: What is a constructor and why do we need it?
**A:** A constructor is a special method that runs automatically when an object is created. We use it to initialize properties so the object starts in a valid state.

```csharp
public class Person
{
    public string Name { get; set; }
    
    public Person(string name)
    {
        Name = name;  // Initialized immediately
    }
}

Person p = new Person("Ali");  // One line!
```

---

### Q3: What is the difference between a field and a property?
**A:** 
- **Field**: Simple variable (old way, not recommended)
- **Property**: Controlled access with `{ get; set; }` (modern, professional C#)

```csharp
// ❌ Field
public string Name;

// ✅ Property
public string Name { get; set; }
```

---

### Q4: What does `this` mean?
**A:** `this` refers to the current object. It's useful when a parameter has the same name as a property.

```csharp
public Student(string name)
{
    this.Name = name;  // "this.Name" = property, "name" = parameter
}
```

---

### Q5: Can a class have multiple constructors?
**A:** Yes! This is called **constructor overloading**. Each can take different parameters.

```csharp
public class Person
{
    public string Name { get; set; }
    public int Age { get; set; }
    
    public Person(string name, int age)  // Constructor 1
    {
        Name = name;
        Age = age;
    }
    
    public Person(string name)  // Constructor 2
    {
        Name = name;
        Age = 0;
    }
    
    public Person()  // Constructor 3
    {
        Name = "Unknown";
        Age = 0;
    }
}

Person p1 = new Person("Ali", 25);
Person p2 = new Person("Fatima");
Person p3 = new Person();
```

---

### Q6: What is the difference between a method and a property?
**A:**
- **Property**: Stores data, uses `{ get; set; }`
- **Method**: Performs action, uses `()`

```csharp
public string Name { get; set; }  // Property
public void Introduce() { }        // Method
```

---

### Q7: What does `public` mean?
**A:** `public` means anyone can access it. Other access modifiers:
- `private` = Only the class itself
- `protected` = Class + inherited classes
- `internal` = Only this project

```csharp
public class BankAccount
{
    public string AccountNumber { get; set; }  // Anyone
    private double balance;                    // Only this class
}
```

---

### Q8: What happens if you don't define a constructor?
**A:** C# creates a default parameterless constructor. But once you define your own, the default disappears — you must use your constructor.

```csharp
public class Person
{
    public Person(string name)
    {
        // Now you MUST provide a name
        // Person p = new Person(); // ❌ Error!
    }
}
```

---

### Q9: Can a class inherit from another class?
**A:** Yes! A class can inherit properties and methods from another class (covered in Inheritance topic).

```csharp
public class Animal
{
    public string Name { get; set; }
    public void Eat() { }
}

public class Dog : Animal  // Inherits from Animal
{
    public void Bark() { }
}

Dog d = new Dog();
d.Name = "Buddy";  // From Animal
d.Eat();           // From Animal
d.Bark();          // Own method
```

---

### Q10: What is encapsulation?
**A:** Hiding internal data and allowing controlled access through properties and methods. This protects your object's state.

```csharp
public class BankAccount
{
    private double balance;  // Hidden
    
    public double Balance
    {
        get { return balance; }
        set { if (value >= 0) balance = value; }  // Validation!
    }
}

BankAccount acc = new BankAccount();
acc.Balance = 1000;   // Valid ✅
acc.Balance = -500;   // Invalid, rejected ❌
```
