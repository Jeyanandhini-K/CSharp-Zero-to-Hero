# What Is .NET (and C#)?

Before you write any code, it helps to understand *what* you're actually working with. This page answers the two questions every beginner has — "What is C#?" and "What is .NET?" — and how they fit together.

---

## C# vs .NET — what's the difference?

Short version:

- **C#** (pronounced "C-sharp") is a **programming language** — the words and grammar you write.
- **.NET** (pronounced "dot net") is a **platform** — the engine that runs your C# code, plus a huge library of pre-built tools you get for free.

An analogy: C# is like a language you speak. .NET is everything around it that makes the language *useful* — the people who understand it, the books already written in it, and the stage you stand on to be heard.

You write C#. .NET runs it. You'll use them together every day, which is why people often blur the two.

---

## What can you build with C# and .NET?

A lot. C# is general-purpose, and .NET runs almost everywhere:

- **Web apps and APIs** — with ASP.NET Core
- **Desktop apps** — Windows (WPF, WinUI) and cross-platform
- **Mobile apps** — iOS and Android, with .NET MAUI
- **Games** — Unity, one of the biggest game engines, uses C#
- **Cloud services and microservices**
- **Console / command-line tools**
- **AI, IoT, and more**

Learn the language once, and these doors all open.

---

## How does C# code actually run?

This is the part that makes you *understand* rather than just memorize. When you run a C# program, it goes through a few steps:

1. **You write source code** — plain text in `.cs` files.
2. **The compiler (Roslyn) translates it** — but *not* straight into machine code. It compiles your C# into **IL** (Intermediate Language), a low-level, CPU-independent set of instructions, packaged into a `.dll` or `.exe`.
3. **The runtime (CLR) executes it** — when you run the program, the **Common Language Runtime (CLR)** takes over. Its **JIT (Just-In-Time) compiler** converts the IL into actual machine code for *your* specific computer, right as it's needed.
4. **The CLR manages memory** — through **garbage collection**, it automatically frees memory you're no longer using, so you rarely have to think about it.

```
Your C# code (.cs)
      │  compiled by Roslyn
      ▼
Intermediate Language (IL)  ──►  packaged in .dll / .exe
      │  JIT-compiled at runtime by the CLR
      ▼
Machine code  ──►  runs on your CPU
```

Why the two-step (IL, then machine code)? Because IL is **platform-independent**. The same compiled program can run on Windows, macOS, or Linux — the CLR on each platform handles the final translation. That's how .NET became "write once, run almost anywhere."

---

## Key terms you'll keep hearing

- **CLR (Common Language Runtime)** — the engine that runs your code and manages memory.
- **BCL (Base Class Library)** — thousands of ready-made classes for everyday tasks (text, files, dates, collections, networking…). You don't reinvent the wheel.
- **SDK (Software Development Kit)** — everything you need to *build* .NET apps: the compiler, the CLI tools, and the runtime. (You'll install this next.)
- **Runtime** — just enough to *run* a finished .NET app, without the build tools. End users install this; developers install the SDK.

## A note on the "many .NETs" confusion

If you Google around, you'll see confusing names. Here's the map so you're never lost:

- **.NET Framework** (versions 1.0–4.8) — the *original*, **Windows-only**, released back in 2002. Still around for old apps, but **not** what you should learn today.
- **.NET Core** (versions 1–3.1) — a cross-platform, open-source rewrite. This name is now retired.
- **.NET 5, 6, 7, 8, 9, 10…** — the **modern, unified** platform. Microsoft dropped the word "Core" and just calls it **.NET**. Cross-platform, fast, and the present and future of the ecosystem.

**The rule for beginners:** ignore .NET Framework and .NET Core as *targets*. Learn **modern .NET** — currently **.NET 10**, the latest long-term-support (LTS) release. Everything in this repo uses it.

> **Tip:** Major .NET versions ship every November. Even-numbered versions (6, 8, 10) are **LTS** (supported ~3 years); odd-numbered ones (7, 9, 11) are **STS** (~2 years). For learning and most projects, pick the latest LTS.

---

## Recap

- **C#** is the language; **.NET** is the platform that runs it.
- Your code is compiled to **IL**, then JIT-compiled to **machine code** by the **CLR** at runtime.
- The **CLR** also handles memory for you via **garbage collection**.
- Use **modern .NET** (currently **.NET 10 LTS**) — not the old .NET Framework.
