# Installing & Setting Up .NET

Time to get your machine ready. By the end of this page you'll have the .NET SDK installed, an editor set up, and a way to confirm everything works. This takes about 10–15 minutes.

---

## Step 1 — Install the .NET SDK

You need the **SDK** (not just the runtime), because the SDK includes the compiler and command-line tools you'll use to build programs.

1. Go to the official download page: **https://dotnet.microsoft.com/download**
2. Download the **latest LTS version** — currently **.NET 10**. The page highlights the recommended version automatically.
3. Run the installer for your operating system:
   - **Windows** — run the `.exe` installer and click through.
   - **macOS** — run the `.pkg` installer. On Apple Silicon (M1/M2/M3…) choose the **Arm64** build; on older Intel Macs choose **x64**.
   - **Linux** — install via your package manager (`apt`, `dnf`, etc.) or the official install script. The download page lists the exact commands per distro.

> Always download from `dotnet.microsoft.com`. Don't grab .NET from random third-party sites.

---

## Step 2 — Verify the installation

Open a **terminal** (Command Prompt or PowerShell on Windows; Terminal on macOS/Linux) and run:

```bash
dotnet --version
```

You should see a version number, something like:

```
10.0.9
```

The exact patch number will differ — what matters is that a number appears and you don't get a "command not found" error.

For more detail, try:

```bash
dotnet --info
```

This shows your installed SDK(s), runtime(s), and OS info — handy when troubleshooting later.

> **"command not found"?** The installer usually adds `dotnet` to your PATH automatically. If it didn't, close and reopen your terminal first. If it still fails, restart your computer, or re-run the installer.

---

## Step 3 — Choose a code editor

You *can* write C# in Notepad, but a real editor gives you autocomplete, error highlighting, and debugging. Pick one:

- **Visual Studio Code (VS Code)** — free, lightweight, runs on Windows/macOS/Linux. **Recommended for beginners** and used throughout this guide. You'll add the C# extension below.
- **Visual Studio** (currently **Visual Studio 2026**) — a full-featured IDE for Windows and Mac. Powerful but heavier. Great once you're building larger apps.
- **JetBrains Rider** — an excellent paid IDE (free for students and open source), cross-platform. A favorite among professionals.

If you're unsure, start with **VS Code**.

---

## Step 4 — Set up VS Code for C#

1. Install **VS Code**: https://code.visualstudio.com
2. Open VS Code and click the **Extensions** icon in the left sidebar (or press `Ctrl+Shift+X` / `Cmd+Shift+X`).
3. Search for **"C# Dev Kit"** (published by Microsoft) and click **Install**. This bundles C# language support, IntelliSense (autocomplete), debugging, and project tools.
4. That's it — VS Code now understands C# files and will help as you type.

> The C# Dev Kit pulls in the base **C#** extension automatically, so you don't need to install that separately.

---

## Recap

- Install the **.NET SDK** (latest LTS — **.NET 10**) from `dotnet.microsoft.com`.
- Confirm it works with **`dotnet --version`**.
- Use **VS Code** + the **C# Dev Kit** extension (recommended), or Visual Studio / Rider.
