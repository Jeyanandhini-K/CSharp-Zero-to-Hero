# Your First C# Program

You've got .NET installed and an editor ready. Let's write and run actual C# ‚Äî starting with the famous "Hello, World!" and then making it a little more interesting.

---

## Two ways to begin

Modern .NET gives you two paths. We'll mainly use the project approach (the standard one), but it's worth knowing both.

### Option A ‚Äî A single file (quickest, newest)

Since C# 14 (shipped with .NET 10), you can run a **single `.cs` file** like a script ‚Äî no project needed. Create a file called `hello.cs`:

```csharp
Console.WriteLine("Hello, World!");
```

Then run it from the terminal:

```bash
dotnet run hello.cs
```

That's the whole program. Great for quick experiments.

### Option B ‚Äî A console project (the standard way)

This is what you'll use most, and what the rest of this guide assumes. In your terminal:

```bash
dotnet new console -o HelloWorld
cd HelloWorld
dotnet run
```

- `dotnet new console` creates a new console (text-based) app.
- `-o HelloWorld` puts it in a folder named `HelloWorld`.
- `dotnet run` builds and runs it.

You should see:

```
Hello, World!
```

Congratulations ‚Äî you just built and ran a C# program.

---

## Looking at the code

Open the project folder in VS Code (`code .` from inside the folder, or File ‚Üí Open Folder), then open **`Program.cs`**. On modern .NET it's surprisingly short:

```csharp
// See https://aka.ms/new-console-template for more information
Console.WriteLine("Hello, World!");
```

Just one real line. This uses **top-level statements** ‚Äî a modern feature that lets you write code directly without ceremony. `Console.WriteLine(...)` prints a line of text to the screen.

---

## What's actually happening (the "classic" version)

Older tutorials ‚Äî and larger programs ‚Äî show a longer form. It's the *same* program written out fully:

```csharp
using System;

namespace HelloWorld
{
    class Program
    {
        static void Main(string[] args)
        {
            Console.WriteLine("Hello, World!");
        }
    }
}
```

It helps to know each piece, because you'll see it everywhere:

- **`using System;`** ‚Äî imports a namespace so you can use its types (like `Console`) without typing the full name.
- **`namespace HelloWorld`** ‚Äî a container that groups related code and prevents naming clashes.
- **`class Program`** ‚Äî a class; in C#, code lives inside classes.
- **`static void Main(string[] args)`** ‚Äî the **entry point**. Every C# program starts running here. `args` holds any command-line arguments.
- **`Console.WriteLine(...)`** ‚Äî prints text followed by a new line.

Top-level statements (the short version) are just the compiler letting you skip the `namespace` / `class` / `Main` boilerplate for simple programs ‚Äî it generates them behind the scenes. Both forms do exactly the same thing.

## Make it interactive

Printing is fun, but let's read input too. Replace the contents of `Program.cs` with:

```csharp
Console.Write("What's your name? ");
string? name = Console.ReadLine();
Console.WriteLine($"Hello, {name}! Welcome to C#.");
```

Run it with `dotnet run` and type your name when asked.

What's new here:

- **`Console.Write`** ‚Äî like `WriteLine`, but without adding a new line after.
- **`Console.ReadLine()`** ‚Äî waits for the user to type a line and press Enter, then gives back what they typed as text (a `string`). The `?` means it *might* be empty/null ‚Äî you'll learn all about that in the null-handling lesson.
- **`$"...{name}..."`** ‚Äî **string interpolation**: the `$` lets you drop variables straight into text using `{ }`. Cleaner than gluing strings together with `+`.

## Comments

Notes for humans that the compiler ignores:

```csharp
// This is a single-line comment

/* This is a
   multi-line comment */
```

Use them to explain *why* your code does something.

## Exercises

1. Change the greeting to also ask for the user's favorite programming language, then print one sentence using both their name and the language.
2. Print three separate lines using three `Console.WriteLine` calls. Then rewrite it as a *single* call using `\n` (the newline character) inside the string.
3. Create a second project with `dotnet new console -o Calculator` that asks for two numbers and prints their sum. *Hint:* `Console.ReadLine()` gives text, so you'll need to convert it to a number with `int.Parse(...)`. (We'll cover conversion properly in the type-casting lesson.)

## Recap

- Run a single file with **`dotnet run file.cs`**, or create a project with **`dotnet new console`** + **`dotnet run`**.
- Modern C# uses **top-level statements**; the classic form spells out **`namespace` ‚Üí `class` ‚Üí `Main`**.
- **`Console.WriteLine` / `Write`** print; **`Console.ReadLine`** reads input.
- **String interpolation** (`$"...{x}..."`) drops variables into text.
