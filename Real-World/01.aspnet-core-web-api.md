# 🌐 ASP.NET Core Web API in C#

---

## 🧠 What is ASP.NET Core?

**ASP.NET Core** is Microsoft's framework for building web applications and APIs. A **Web API** receives HTTP requests and returns data (usually JSON) — it's the backend that powers mobile apps, web frontends, and other services.

```
Client (React / Mobile / Postman)
        │  HTTP Request (GET /api/products)
        ▼
   ASP.NET Core Web API
        │  Returns JSON
        ▼
   { "id": 1, "name": "Laptop", "price": 999.99 }
```

> 💡 Everything you've learned — classes, async/await, DI, SOLID, patterns — comes together here. This is where C# runs in the real world. 🚀

## 🎮 Your First Controller

```csharp
using Microsoft.AspNetCore.Mvc;

[ApiController]                    // marks this as an API controller
[Route("api/[controller]")]        // route: api/products
public class ProductsController : ControllerBase
{
    private static readonly List<Product> _products = new()
    {
        new Product { Id = 1, Name = "Laptop",  Price = 999.99m },
        new Product { Id = 2, Name = "Phone",   Price = 599.99m },
        new Product { Id = 3, Name = "Headset", Price = 149.99m },
    };

    // GET api/products
    [HttpGet]
    public ActionResult<List<Product>> GetAll()
        => Ok(_products);

    // GET api/products/1
    [HttpGet("{id}")]
    public ActionResult<Product> GetById(int id)
    {
        var product = _products.FirstOrDefault(p => p.Id == id);
        if (product is null) return NotFound();   // 404
        return Ok(product);                       // 200
    }

    // POST api/products
    [HttpPost]
    public ActionResult<Product> Create(Product product)
    {
        product.Id = _products.Max(p => p.Id) + 1;
        _products.Add(product);
        return CreatedAtAction(nameof(GetById), new { id = product.Id }, product);  // 201
    }

    // PUT api/products/1
    [HttpPut("{id}")]
    public IActionResult Update(int id, Product updated)
    {
        var product = _products.FirstOrDefault(p => p.Id == id);
        if (product is null) return NotFound();
        product.Name  = updated.Name;
        product.Price = updated.Price;
        return NoContent();   // 204
    }

    // DELETE api/products/1
    [HttpDelete("{id}")]
    public IActionResult Delete(int id)
    {
        var product = _products.FirstOrDefault(p => p.Id == id);
        if (product is null) return NotFound();
        _products.Remove(product);
        return NoContent();   // 204
    }
}

public class Product
{
    public int     Id    { get; set; }
    public string  Name  { get; set; } = "";
    public decimal Price { get; set; }
}
```

## 📡 HTTP Methods & Status Codes

| Method | Use | Common codes |
|---|---|---|
| `GET` | Read data | 200 OK, 404 Not Found |
| `POST` | Create | 201 Created, 400 Bad Request |
| `PUT` | Replace | 204 No Content, 404 Not Found |
| `PATCH` | Partial update | 204 No Content |
| `DELETE` | Remove | 204 No Content, 404 Not Found |

## 💉 Dependency Injection in Controllers

```csharp
[ApiController]
[Route("api/[controller]")]
public class OrdersController : ControllerBase
{
    private readonly IOrderService _orderService;
    private readonly ILogger<OrdersController> _logger;

    // DI — injected automatically ✅
    public OrdersController(IOrderService orderService, ILogger<OrdersController> logger)
    {
        _orderService = orderService;
        _logger = logger;
    }

    [HttpGet]
    public async Task<ActionResult<List<Order>>> GetAll()
    {
        _logger.LogInformation("Getting all orders");
        var orders = await _orderService.GetAllAsync();
        return Ok(orders);
    }

    [HttpPost]
    public async Task<IActionResult> Place(Order order)
    {
        bool success = await _orderService.PlaceOrderAsync(order);
        if (!success) return BadRequest("Order failed");
        return CreatedAtAction(nameof(GetAll), order);
    }
}
```

## 🔍 Route Parameters & Query Strings

```csharp
// Route parameter: GET api/products/42
[HttpGet("{id}")]
public ActionResult<Product> GetById(int id) { ... }

// Query string: GET api/products?category=electronics&maxPrice=500
[HttpGet]
public ActionResult<List<Product>> Search(
    [FromQuery] string? category,
    [FromQuery] decimal? maxPrice)
{
    var results = _products.AsQueryable();
    if (category is not null) results = results.Where(p => p.Category == category);
    if (maxPrice is not null)  results = results.Where(p => p.Price <= maxPrice);
    return Ok(results.ToList());
}

// Request body: POST api/products
[HttpPost]
public IActionResult Create([FromBody] Product product) { ... }

// Header: custom header value
[HttpGet("secure")]
public IActionResult Secure([FromHeader(Name = "X-Api-Key")] string apiKey) { ... }
```

## ✅ Model Validation — Auto-Validated

```csharp
using System.ComponentModel.DataAnnotations;

public class CreateProductRequest
{
    [Required]
    [StringLength(100, MinimumLength = 2)]
    public string Name { get; set; } = "";

    [Range(0.01, 1_000_000)]
    public decimal Price { get; set; }

    [Required]
    [EmailAddress]
    public string ContactEmail { get; set; } = "";
}

// [ApiController] automatically returns 400 if validation fails — no extra code! ✅
[HttpPost]
public IActionResult Create(CreateProductRequest request)
{
    // If we reach here, request is already valid ✅
    var product = new Product { Name = request.Name, Price = request.Price };
    return CreatedAtAction(nameof(GetById), new { id = product.Id }, product);
}
```

## 🛡️ Middleware — The Request Pipeline

Every request flows through a **pipeline of middleware** in order:

```csharp
// Program.cs — order matters!
var app = builder.Build();

app.UseHttpsRedirection();    // redirect HTTP → HTTPS
app.UseAuthentication();      // who are you?
app.UseAuthorization();       // what are you allowed to do?
app.UseRateLimiter();         // throttle requests
app.MapControllers();         // route to controllers
```

### Custom Middleware

```csharp
// Log every request and how long it took
app.Use(async (context, next) =>
{
    var watch = Stopwatch.StartNew();
    var path  = context.Request.Path;

    await next();   // call the rest of the pipeline

    watch.Stop();
    Console.WriteLine($"{context.Request.Method} {path} → {context.Response.StatusCode} ({watch.ElapsedMilliseconds}ms)");
});
```

## 🌍 Full Real-World Program.cs

```csharp
var builder = WebApplication.CreateBuilder(args);

// Services
builder.Services.AddControllers();
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

// Your app services
builder.Services.AddScoped<IProductRepository, SqlProductRepository>();
builder.Services.AddScoped<IProductService,    ProductService>();
builder.Services.AddScoped<IOrderRepository,   SqlOrderRepository>();
builder.Services.AddScoped<IOrderService,      OrderService>();

// Database
builder.Services.AddDbContext<AppDbContext>(opt =>
    opt.UseSqlServer(builder.Configuration.GetConnectionString("Default")));

var app = builder.Build();

// Middleware pipeline
if (app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}

app.UseHttpsRedirection();
app.UseAuthorization();
app.MapControllers();
app.Run();
```

## ⚡ Minimal APIs (C# 10+)

For simple endpoints, skip the controller entirely:

```csharp
var app = WebApplication.Create(args);

app.MapGet("/api/hello", () => "Hello, World!");

app.MapGet("/api/products/{id}", async (int id, IProductService svc) =>
{
    var product = await svc.GetByIdAsync(id);
    return product is null ? Results.NotFound() : Results.Ok(product);
});

app.MapPost("/api/products", async (Product product, IProductService svc) =>
{
    await svc.CreateAsync(product);
    return Results.Created($"/api/products/{product.Id}", product);
});

app.Run();
```

## 🎤 Interview Q&A

**Q1: What is ASP.NET Core?**
A: A cross-platform framework for building web apps, APIs, and services. Fast, open-source, and built on .NET.

**Q2: What does `[ApiController]` do?**
A: Enables automatic model validation (400 on invalid), binding source inference, and problem detail responses.

**Q3: What is the difference between `Ok()`, `NotFound()`, and `NoContent()`?**
A: `Ok()` = 200 with data. `NotFound()` = 404. `NoContent()` = 204, no body — used after PUT/DELETE.

**Q4: What is middleware?**
A: Components in the request pipeline that process requests/responses in order. Examples: auth, HTTPS redirect, logging, rate limiting.

**Q5: What is `[FromQuery]` vs `[FromBody]` vs `[FromRoute]`?**
A: Source of the parameter. `[FromRoute]` = URL path. `[FromQuery]` = query string. `[FromBody]` = request body (JSON).

**Q6: What is Swagger?**
A: Auto-generated interactive API documentation. Add `AddSwaggerGen()` and visit `/swagger` to test endpoints.

**Q7: What is the difference between a controller and minimal API?**
A: Controllers = class-based, structured, great for large APIs. Minimal APIs = function-based, less boilerplate, great for small APIs.

**Q8: How does DI work in controllers?**
A: Constructor injection — the DI container resolves and injects all dependencies registered in `Program.cs`.

**Q9: What is `CreatedAtAction`?**
A: Returns 201 Created with a `Location` header pointing to the new resource's GET endpoint — the correct REST response for POST.

**Q10: What is the request pipeline?**
A: The ordered chain of middleware that every HTTP request flows through before reaching a controller, and every response flows back through.

## 🚀 Key Takeaways

✅ `[ApiController]` + `ControllerBase` = the foundation of every controller  
✅ `[HttpGet]`, `[HttpPost]`, `[HttpPut]`, `[HttpDelete]` map HTTP methods  
✅ Return `Ok()`, `NotFound()`, `CreatedAtAction()`, `NoContent()` — not raw data  
✅ `[ApiController]` auto-validates models — returns 400 if invalid  
✅ DI works in controllers via constructor injection  
✅ Middleware runs in order — auth before controllers, always  
✅ Swagger is auto-generated — use it to test your API instantly  
✅ Minimal APIs = less code for simple endpoints  
✅ `Program.cs` = where services are registered and middleware is configured
