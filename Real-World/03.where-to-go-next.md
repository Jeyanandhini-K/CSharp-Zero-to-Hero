## 📚 Topics to Explore Next

### 🔐 Authentication & Authorization
```bash
dotnet add package Microsoft.AspNetCore.Authentication.JwtBearer
```
- JWT tokens — how login works in APIs
- `[Authorize]` attribute — protect endpoints
- Roles and policies

### 🧪 Integration & End-to-End Testing
```bash
dotnet add package Microsoft.AspNetCore.Mvc.Testing
```
- Test your API with real HTTP requests
- `WebApplicationFactory` — spin up a test server
- Test the full stack without mocking

### ☁️ Cloud & Deployment
- **Azure** — Microsoft's cloud, perfect for .NET
- **Docker** — containerize your app (`dotnet publish` → Docker image)
- **GitHub Actions** — CI/CD, auto-deploy on every push

### 📡 Real-Time with SignalR
```bash
dotnet add package Microsoft.AspNetCore.SignalR
```
- WebSockets in C# — live chat, notifications, dashboards

### 🏎️ Performance
- `Span<T>` and `Memory<T>` — zero-allocation code
- `IAsyncEnumerable<T>` — stream large datasets
- Benchmarking with **BenchmarkDotNet**

### 🧩 Architecture Patterns
- **Clean Architecture** — layers that don't depend inward
- **CQRS** — separate reads (queries) from writes (commands)
- **MediatR** — decouple handlers from controllers

## 💼 Getting Job-Ready

### Build your portfolio
- Push every project to GitHub with a clear README
- Deploy at least one project publicly (Azure Free Tier, Railway, Fly.io)
- Write clean code — interviewers look at your repos

### Practice interview questions
- Your `Interview-QA/` folder has 100+ questions — review them weekly
- LeetCode Easy/Medium for DSA (focus on arrays, strings, hashmaps)
- Practice explaining your code out loud — interviewers care as much about HOW you think

### What companies look for in a C# developer
```
✅ OOP and SOLID principles
✅ async/await — you must understand this cold
✅ Entity Framework Core or Dapper
✅ REST APIs with ASP.NET Core
✅ Unit testing with xUnit/Moq
✅ Git — branching, PRs, commit messages
✅ SQL basics — JOIN, GROUP BY, indexes
```
