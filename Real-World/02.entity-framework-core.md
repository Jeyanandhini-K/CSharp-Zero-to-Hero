# 🗄️ Entity Framework Core in C#

---

## 🧠 What is Entity Framework Core?

**Entity Framework Core (EF Core)** is an ORM — **Object-Relational Mapper**. It lets you talk to a database using C# classes and LINQ — no raw SQL needed.

```csharp
// ❌ Without EF Core — raw SQL everywhere
string sql = "SELECT * FROM Products WHERE Price < 500";
// Open connection, create command, read data, map to objects... 😩

// ✅ With EF Core — pure C#
var cheapProducts = await context.Products
    .Where(p => p.Price < 500)
    .ToListAsync();   // EF generates the SQL for you ✅
```

> 💡 EF Core translates your C# LINQ queries into SQL, executes them, and maps the results back to C# objects automatically.

## 🏗️ Define Your Models

```csharp
// Each class = one database table
public class Product
{
    public int     Id       { get; set; }   // auto primary key ✅
    public string  Name     { get; set; } = "";
    public decimal Price    { get; set; }
    public int     Stock    { get; set; }
    public string  Category { get; set; } = "";

    // Navigation property — relationship to another table
    public List<OrderItem> OrderItems { get; set; } = new();
}

public class Order
{
    public int      Id            { get; set; }
    public DateTime PlacedAt      { get; set; } = DateTime.UtcNow;
    public string   CustomerEmail { get; set; } = "";
    public decimal  Total         { get; set; }

    public List<OrderItem> Items { get; set; } = new();
}

public class OrderItem
{
    public int     Id        { get; set; }
    public int     OrderId   { get; set; }   // foreign key
    public int     ProductId { get; set; }   // foreign key
    public int     Quantity  { get; set; }
    public decimal UnitPrice { get; set; }

    // Navigation properties
    public Order   Order   { get; set; } = null!;
    public Product Product { get; set; } = null!;
}
```

## 🔌 DbContext — The Gateway to Your Database

```csharp
public class AppDbContext : DbContext
{
    // Each DbSet = one table
    public DbSet<Product>   Products   => Set<Product>();
    public DbSet<Order>     Orders     => Set<Order>();
    public DbSet<OrderItem> OrderItems => Set<OrderItem>();

    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { }

    // Optional: fine-tune the schema
    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        modelBuilder.Entity<Product>(e =>
        {
            e.Property(p => p.Name).HasMaxLength(200).IsRequired();
            e.Property(p => p.Price).HasColumnType("decimal(18,2)");
        });

        modelBuilder.Entity<OrderItem>()
            .HasOne(oi => oi.Order)
            .WithMany(o => o.Items)
            .HasForeignKey(oi => oi.OrderId);
    }
}
```

## 📦 Migrations — Sync Code to Database

```bash
# Create a migration (snapshot of your model changes)
dotnet ef migrations add InitialCreate

# Apply migration to database (creates/updates tables)
dotnet ef database update

# Add more migrations as your models evolve
dotnet ef migrations add AddProductCategory
dotnet ef database update

# Undo last migration
dotnet ef migrations remove
```

> 💡 Think of migrations like **Git commits for your database schema**. Each migration records what changed.

## ✅ CRUD Operations

```csharp
public class ProductRepository : IProductRepository
{
    private readonly AppDbContext _context;
    public ProductRepository(AppDbContext context) => _context = context;

    // CREATE
    public async Task<Product> CreateAsync(Product product)
    {
        _context.Products.Add(product);
        await _context.SaveChangesAsync();   // ← writes to DB
        return product;
    }

    // READ — all
    public async Task<List<Product>> GetAllAsync()
        => await _context.Products.ToListAsync();

    // READ — by id
    public async Task<Product?> GetByIdAsync(int id)
        => await _context.Products.FindAsync(id);

    // UPDATE
    public async Task<bool> UpdateAsync(Product updated)
    {
        var product = await _context.Products.FindAsync(updated.Id);
        if (product is null) return false;

        product.Name  = updated.Name;
        product.Price = updated.Price;

        await _context.SaveChangesAsync();
        return true;
    }

    // DELETE
    public async Task<bool> DeleteAsync(int id)
    {
        var product = await _context.Products.FindAsync(id);
        if (product is null) return false;

        _context.Products.Remove(product);
        await _context.SaveChangesAsync();
        return true;
    }
}
```

## 🔍 Querying with LINQ

```csharp
// Filter
var electronics = await context.Products
    .Where(p => p.Category == "Electronics")
    .ToListAsync();

// Sort
var byPrice = await context.Products
    .OrderBy(p => p.Price)
    .ToListAsync();

// Top N
var top5 = await context.Products
    .OrderByDescending(p => p.Price)
    .Take(5)
    .ToListAsync();

// Search
var search = await context.Products
    .Where(p => p.Name.Contains("phone"))
    .ToListAsync();

// Single item
var product = await context.Products
    .FirstOrDefaultAsync(p => p.Id == 1);

// Count / Sum / Average
int count        = await context.Products.CountAsync();
decimal total    = await context.Products.SumAsync(p => p.Price);
double avgPrice  = await context.Products.AverageAsync(p => (double)p.Price);

// Include related data (JOIN)
var ordersWithItems = await context.Orders
    .Include(o => o.Items)
        .ThenInclude(i => i.Product)
    .Where(o => o.CustomerEmail == "asha@test.com")
    .ToListAsync();
```

## 🌍 Real-World: Full Repository + Service + Controller

```csharp
// Repository
public class OrderRepository : IOrderRepository
{
    private readonly AppDbContext _context;
    public OrderRepository(AppDbContext context) => _context = context;

    public async Task<List<Order>> GetByEmailAsync(string email)
        => await _context.Orders
            .Include(o => o.Items)
            .ThenInclude(i => i.Product)
            .Where(o => o.CustomerEmail == email)
            .OrderByDescending(o => o.PlacedAt)
            .ToListAsync();

    public async Task<Order> CreateAsync(Order order)
    {
        _context.Orders.Add(order);
        await _context.SaveChangesAsync();
        return order;
    }
}

// Service
public class OrderService : IOrderService
{
    private readonly IOrderRepository _repo;
    public OrderService(IOrderRepository repo) => _repo = repo;

    public async Task<Order> PlaceAsync(string email, List<OrderItem> items)
    {
        var order = new Order
        {
            CustomerEmail = email,
            Items         = items,
            Total         = items.Sum(i => i.UnitPrice * i.Quantity),
            PlacedAt      = DateTime.UtcNow,
        };
        return await _repo.CreateAsync(order);
    }
}

// Controller
[ApiController]
[Route("api/[controller]")]
public class OrdersController : ControllerBase
{
    private readonly IOrderService _service;
    public OrdersController(IOrderService service) => _service = service;

    [HttpGet("{email}")]
    public async Task<ActionResult<List<Order>>> GetByEmail(string email)
        => Ok(await _service.GetByEmailAsync(email));  // missing method — add to service ✅

    [HttpPost]
    public async Task<ActionResult<Order>> Place(PlaceOrderRequest request)
    {
        var order = await _service.PlaceAsync(request.Email, request.Items);
        return CreatedAtAction(nameof(GetByEmail), new { email = request.Email }, order);
    }
}
```

## 🚨 Common Mistakes

```csharp
// ❌ Forgetting SaveChangesAsync — changes don't persist!
context.Products.Add(product);
// Missing: await context.SaveChangesAsync();

// ❌ Loading too much data (N+1 problem)
var orders = await context.Orders.ToListAsync();
foreach (var order in orders)
{
    var items = await context.OrderItems   // ❌ one query PER order!
        .Where(i => i.OrderId == order.Id).ToListAsync();
}

// ✅ Use Include to load related data in ONE query
var orders = await context.Orders
    .Include(o => o.Items)
    .ToListAsync();

// ❌ Reusing DbContext across requests — not thread-safe!
// ✅ Register as Scoped — one per request (the default)
builder.Services.AddDbContext<AppDbContext>(...);   // Scoped by default ✅
```

## 🎤 Interview Q&A

**Q1: What is Entity Framework Core?**
A: An ORM — maps C# classes to database tables. Write LINQ instead of SQL; EF generates and runs the SQL.

**Q2: What is a DbContext?**
A: The gateway to your database. Contains `DbSet<T>` properties (one per table) and manages connections and change tracking.

**Q3: What is a migration?**
A: A snapshot of model changes — creates/alters database tables to match your C# models. Like Git commits for schema.

**Q4: What is `SaveChangesAsync()`?**
A: Commits all tracked changes (inserts, updates, deletes) to the database in one transaction.

**Q5: What is the N+1 problem?**
A: Running one query per item in a loop instead of one query total. Fix it with `.Include()`.
```csharp
// ❌ N+1 — one DB call per order
// ✅ Include — one DB call total
context.Orders.Include(o => o.Items).ToListAsync()
```

**Q6: What is `.Include()`?**
A: Eager loading — loads related navigation properties in the same query (generates a JOIN).

**Q7: What lifetime should DbContext be registered as?**
A: Scoped — one instance per HTTP request. Never Singleton (not thread-safe) or Transient (wasteful).

**Q8: What databases does EF Core support?**
A: SQL Server, SQLite, PostgreSQL (Npgsql), MySQL, Cosmos DB, and more via providers.

**Q9: What is the difference between `Find` and `FirstOrDefault`?**
A: `Find(id)` checks the change tracker cache first (faster for primary key lookup). `FirstOrDefault` always queries the DB.

**Q10: How do you seed initial data?**
A: In `OnModelCreating` using `HasData`:
```csharp
modelBuilder.Entity<Product>().HasData(
    new Product { Id = 1, Name = "Laptop", Price = 999m });
```

## 🚀 Key Takeaways

✅ EF Core = write C# LINQ, not SQL — it generates SQL for you  
✅ Each class = one table · Each `DbSet<T>` = one table in context  
✅ Always `await SaveChangesAsync()` — nothing persists without it  
✅ Migrations = version control for your database schema  
✅ Use `.Include()` to load related data — avoids N+1 queries  
✅ Register `DbContext` as **Scoped** — one per request  
✅ Repository pattern + EF Core = clean, testable data access  
✅ `FindAsync(id)` is faster for primary key lookups
